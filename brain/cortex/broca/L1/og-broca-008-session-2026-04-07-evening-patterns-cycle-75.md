## Session 2026-04-07 Evening Patterns (cycle 75)

### Pace & Mode
- Short messages, action-oriented: 'yeah', 'great, now we need to continue working on the Dashboard'
- Opens with casual check-in: 'Brain, how are you?? Where did we leave off?'
- Frustration expressed calmly but firmly: 'so this is a recurring issue where I need to approve some commands'
- Longer messages when describing architecture/vision (dashboard redesign request = detailed multi-paragraph)

### Decision Style (this session)
- Corrects defaults when they don't match: 'Every 1hour, during 8AM-8PM' (rejected 30min option, added time window)
- Expands scope beyond options given: 'all DMs, conversations, AND starred channels' (picked hybrid, not one option)
- Adds emotional context to identity questions: 'you must learn to do these things yourself :)' — affectionate teaching tone
- Present-then-act model chosen: wants to SEE tasks before brain acts. Dashboard = review surface.

### Vocabulary (new)
- 'Umm unfortunately...' = soft opener for bad news (goodhealth down)
- 'We need to continue working on' = resume prior thread, expects brain to know context
- 'find the correct spelling' = instruction embedded in answer, expects brain to look it up
- Smiley ':)' used when teaching the brain about its own purpose — warmth signal


### Slack Message Styling — Forecast Channel
- **Inline code** (`backticks`) for: product names, YYYY-MM dates, shop names, entity names
- **Bold** (\*asterisks\*) for: key numbers (forecast totals), section headers
- **Italics** (\_underscores\_) for: period labels, metadata notes
- **Bullets** (• or -) over tables — Slack tables (code blocks) are hard to read on mobile
- **Pipe separators** (|) within bullet lines for multi-metric summaries
- **Sort by value desc** — most important/largest first
- **Group errors by type** with bold headers, blank line between sections
- **No clutter** — remove columns that don't add value, use business terms (F-YoY % not New %)
- **Emoji**: standard size, not at line start for headers. Use ✅ for pass, ❌ for fail
- **Structure**: header line → blank line → content → blank line → footer/period

### General Slack Style (all channels)
- Keep messages scannable — bullet points, not paragraphs
- Use inline code for technical identifiers (function names, file names, dates)
- Bold for action items or key takeaways
- No walls of text — if longer than ~10 lines, split into sections
