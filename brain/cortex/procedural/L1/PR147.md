# PR147: EC2 Brain/Skills Symlink Recovery

## Cue

- `claude-brain.service` failing on goodhealth.lv
- `~/.claude/brain` or `~/.claude/skills` is a broken symlink
- Hooks fail with "no such file or directory"
- After a runaway agent or partial deploy

## Routine

```bash
ssh -i ~/Desktop/good_health_machine.pem ubuntu@goodhealth.lv
cd /home/ubuntu/.claude
rm -f brain skills                       # remove broken symlinks (safe — they're symlinks, not data)
git checkout HEAD -- brain/ skills/      # restore from git (gives real directories or fresh symlinks)
ls brain/hook.py                            # verify presence
python3 brain/hook.py PreToolUse </dev/null; echo "exit=$?"   # expect exit=0
sudo systemctl restart claude-brain.service
```

## Verification

- `ls brain/hook.py` returns the file
- Hook exit code is 0 (not 2 = blocked, not 1 = error)
- `systemctl status claude-brain.service` shows active

## Notes

- `git checkout HEAD --` is safe here because brain/skills are tracked dirs, not symlinks-in-git.
- This procedure was executed by Justs himself on 2026-04-26 after a brain-state issue.
- Does NOT require sudo for the git checkout — only the service restart.
