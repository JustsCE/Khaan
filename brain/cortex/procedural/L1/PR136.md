# og Brain — MCP Server Deployment Pipeline (Self-Hosted on goodhealth.lv)

## Architecture
- Host nginx (/etc/nginx/sites-enabled/brain-viz) is the SSL terminator -- NOT Docker nginx
- Each MCP server runs as a Docker container, exposed on a unique port
- nginx routes /mcp/{name}/ -> localhost:{port}/mcp
- MCP servers ARE the Brain's hands -- sensory/motor connections to external systems

## Adding a New MCP Server

1. Create Dockerfile in viz/mcp-servers/{name}/
2. Add service to viz/docker-compose.yml with port + env vars
3. Add credentials to /opt/brain-viz/.env (gitignored)
4. Add nginx route to HOST nginx (/etc/nginx/sites-enabled/brain-viz):
   - MUST add both exact match (= /mcp/{name}) AND prefix match (/mcp/{name}/)
   - Exact match: proxy_pass to localhost:{port}/mcp
   - Prefix match: rewrite to /mcp/ path
5. rsync viz/ to /opt/brain-viz/, docker compose build + up
6. sudo nginx -t && sudo nginx -s reload
7. Register connector on claude.ai/settings -> Add custom connector
8. Add connector UUID to agent triggers via RemoteTrigger update

## Gotchas

1. HOST nginx vs Docker nginx: Real SSL proxy = /etc/nginx on host. Docker nginx is vestigial.
2. nginx-active.conf has immutable flag: sudo chattr -i before overwriting
3. AWS MCP host validation: Must set proxy_set_header Host to bind address (0.0.0.0:3003), not $host
4. Exact match routes needed: Anthropic POSTs to /mcp/{name} without trailing slash
5. supergateway listens on /mcp, not root /
6. Credentials live in /opt/brain-viz/.env (gitignored)
