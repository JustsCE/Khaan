# PR067 -- og Brain — FloatingPane Architecture Pattern
- Created: 2026-04-13
- Source: brain-retro session 2026-04-13
- Strength: 2
- Reinforced: 0
- Tags: vericity, frontend, ui, floating-pane, drag-drop
- Domain: vericity

## Cue
Creating movable/resizable UI panels overlaid on a map or full-viewport background

## Routine
1. FloatingPane component (components/ui/FloatingPane.tsx)
   - Props: id, defaultPosition, defaultSize, minSize, maxSize, title, children
   - Drag via titlebar mousedown, mousemove on document, mouseup commits to state
   - Resize via 8 invisible edge handles (n/s/e/w/ne/nw/se/sw)
   - During drag/resize: mutate DOM directly via ref (no React state until mouseup)
   - Persistence: localStorage with layout version key
   - Z-ordering: module-level counter, clicked pane goes to front
   - Viewport clamping: positions clamped to [0, viewport] on mount + resize
2. Layout version (LAYOUT_VERSION const): bump to invalidate stale positions
3. Use viewport-proportional sizing (vw/vh percentages) for default positions

## Gotchas
- localStorage positions can put panes off-screen at different resolutions. Always clamp.
- Layout version must be bumped when changing default positions
- Leaflet z-index context: panes must be SIBLINGS of the map container, not inside it
- Interactive elements in titlebar (buttons, selects) need closest() check to not trigger drag
