# PR093: Spawn New Claude Code Session for iOS App

**Cue:** User asks for a new session, wants to connect from iOS, or needs MCP tools that are not loaded in current session.

**Routine:**
1. We ARE on the EC2 (goodhealth.lv) — no SSH needed
2. Run: `nohup claude --name "<descriptive-name>" > /dev/null 2>&1 &`
3. The session appears in the Claude Code iOS app session list as Idle
4. User taps it from their phone to connect
5. MCP tools from settings.json load automatically in the new session

**Do NOT:**
- Try to SSH to goodhealth.lv (we are already on it)
- Give the user a claude.ai/code link
- Say you cannot create sessions
- Use RemoteTrigger API (that is for scheduled cron agents, not interactive sessions)

**Prerequisites:** Claude CLI installed on EC2 (it is: /usr/bin/claude)
