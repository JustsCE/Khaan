# og Brain — PR112: Fast-Path Delivery for Action-Train Iterations

## Cue
/action-train round is in progress; user asks for a refined output.

## Routine
1. Deliver the **primary format** first (the one the user originally specified — usually PDF or DOCX in the repo).
2. Do NOT chain secondary format conversions (Google Doc, DOCX, HTML preview, etc.) without explicit upfront request.
3. If user asks mid-round for an alternate format, offer to do it AFTER the primary is confirmed.
4. Context weight: base64-encoding + MCP payloads for large binaries cost conversational latency; avoid unless necessary.

## Reward
User doesn't wait. Iteration velocity matches drafting speed, not pipeline speed.

## Anti-pattern
Round-1 v4 (2026-04-22): mid-pipeline shifted from PDF to DOCX→Drive auto-convert, spent cycles on base64 encoding + tool call overhead. User asked "Why does it take so long?" — correctly pointing out the unnecessary detour. Fix: always commit the primary format first, then offer extras.
