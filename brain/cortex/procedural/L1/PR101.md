# PR101: Managing Scheduled Routine Prompts

**Cue:** Any time the user (or Brain) needs to update a claude.ai scheduled routine's prompt text — e.g. fixing a template, adding cleanup steps, tweaking mode names.

**Wrong action:** Ask the user to manually edit the routine in the claude.ai UI.
**Right action:** Invoke the `/schedule` skill (or `Skill(schedule)`) with `list` / `update` / `create` args.

**Why:** Brain has direct access to scheduled triggers via the `schedule` skill and underlying `CronCreate`/`CronDelete`/`CronList`/`RemoteTrigger` tools. Routines are scheduled remote agents — Brain manages them, doesn't hand them to the user.

**Related tools:** CronCreate, CronDelete, CronList, RemoteTrigger (deferred, fetch via ToolSearch).

**History:** 2026-04-21, sentinel cleanup session — proposed manual UI edits for 4+ routines; Justs corrected: "You always updated the agent prompts yourself...."
