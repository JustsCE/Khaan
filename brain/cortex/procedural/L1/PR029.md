### PR029: og Brain — Dashboard Conversation-Driven Task Pipeline
- **Domain:** Infrastructure / Dashboard
- **Learned:** 2026-04-08
- **Reinforced:** 0 times
- **Strength:** 1
- **Cue:** Need to extract tasks from Slack conversations for the dashboard
- **Routine:**
  1. Layer 1 (EC2 cron): viz/scripts/slack-puller.py calls Slack Web API with SLACK_BOT_TOKEN
     - conversations.list (types=im,mpim,public_channel,private_channel)
     - conversations.history per channel (oldest=max(watermark, now-14d))
     - conversations.replies for threads
     - Writes brain/sync-state/slack-raw.json, git push
  2. Layer 2 (GitHub Actions): .github/workflows/slack-task-processor.yml
     - Triggered by push to slack-raw.json
     - Claude CLI agent reads cortex (S015, S020, R016, identity.md)
     - Extracts tasks, POSTs to https://goodhealth.lv/api/dashboard/tasks/slack
  3. Cron: 0 5-17 * * 1-5 (hourly 8AM-8PM EEST, weekdays)
  4. Channels: ALL DMs + ALL group DMs + allowlisted channels
- **Gotcha:** First sync is slow (~20min) — 665 conversations at 1.5s/API call. Delta syncs fast.
- **Gotcha:** Slack rate limit (429) — must retry with Retry-After header + 1s buffer.
- **Gotcha:** xoxp- token works for DMs/groups/channels. Bot token may not see all DMs.
- **Gotcha:** conversations.list with all types returns channels first. DMs come in later pages.
- **Prerequisites:** SLACK_BOT_TOKEN on EC2, requests pip package, brain repo cloned at /brain
- **Verification:** slack-raw.json has messages, curl sync-status returns lastSync timestamp
- **Apply when:** Setting up Slack monitoring, debugging the task pipeline, adding new sources
