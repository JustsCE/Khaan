### PR161: og Brain — RemoteTrigger session_context.sources Is The Auth Boundary

- **Cue:** Designing or debugging an Anthropic RemoteTrigger that needs to read/write multiple GitHub repos. Especially: a secondary repo for agent-private memory (Teacher's own brain, future per-agent state).
- **Stage:** conscious
- **Why this exists:** From 2026-04-25 to 2026-04-28, Teacher trigger had only `claude-brain` in `session_context.sources`. The Anthropic platform auto-clones and authenticates everything in `sources`. ANY clone outside that list (e.g. `agent-setup.sh` falling back to plain `git clone https://github.com/JustsCE/teacher-brain.git`) runs un-authenticated. For a private repo, that clone fails silently and Teacher runs degraded — work-on-main commits still land in claude-brain, but Teacher's own-brain memory writes go nowhere. Bug stayed invisible because the agent does NOT exit non-zero on degraded boot.
- **Routine:**
  1. Before adding a new agent OR a new repo to an existing agent, list current trigger config: `RemoteTrigger.list` and inspect `session_context.sources` and `outcomes` for each.
  2. Every repo the agent reads from MUST be in `sources`. Use the form `{git_repository: {url: https://github.com/OWNER/REPO}}` for read-only, OR `{git_repository: {url: ..., allow_unrestricted_git_push: true}}` if the agent will push.
  3. Every repo the agent pushes to MUST also be in `outcomes`. Form: `{git_repository: {git_info: {branches: [BRANCH], repo: OWNER/REPO}}}` where BRANCH is the agent's working branch (e.g. `claude/teacher-memory`).
  4. Apply via `RemoteTrigger.update` with the full `job_config.ccr.session_context` block (partial-update of nested fields can replace siblings — send the whole block to be safe).
  5. Verify: `RemoteTrigger.get` and confirm both arrays contain the new repo.
  6. Test: `RemoteTrigger.run` to fire immediately; watch the target repo's `pushed_at` (via `gh api repos/OWNER/REPO`) for a fresh push within the agent's budget window.
- **Pitfall: the silent-degrade.** agent-setup.sh has a fallback that creates a stub directory if the secondary repo cannot be cloned (`echo "WARN: teacher-brain unavailable, stub created"`). The agent then runs work-on-main and reports success because main commits succeed. The user only notices days later when the secondary repo's commit graph stops moving. Detection: brain-cycle Phase 10 should check `pushed_at` of every repo in any agent's `outcomes` and alarm if stale > 48h.
- **Pitfall: agent-setup.sh discovery loop.** The script finds pre-mounted repos at `/home/user/REPO`, `/home/ubuntu/REPO`, etc. The Anthropic platform mounts `sources`-listed repos at `/home/user/REPO` (matching the existing first candidate). So once `sources` is correct, no script change is needed — the existing discovery will pick up the new mount.
- **Reinforced by:** E090 + E097 (Teacher death diagnosis 2026-04-28). Replaces the assumption that 'agent-setup.sh handles auth' — it does not; the platform does, via sources.
