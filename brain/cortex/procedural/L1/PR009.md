### PR009: og Brain — CI/CD Secret & Deploy Self-Management
- **Stage:** conscious
- **Learned:** 2026-04-01
- **Executions:** 1
- **Strength:** 1
- **Cue:** A GitHub Actions workflow needs a new secret, or a deploy needs credentials
- **Routine:**
  1. Check ~/.credentials.json for the required key
  2. If present: use skills/github-secrets/set_secret.py to set it on the repo via API
  3. If not present: ask user once, save to ~/.credentials.json, then set via API
  4. For EC2 SSH access: read PEM from ~/Desktop/good_health_machine.pem, set as EC2_SSH_KEY secret
  5. To monitor deploys: GET /repos/{owner}/{repo}/actions/runs, download logs from /runs/{id}/logs (ZIP)
  6. To trigger deploy: POST /actions/workflows/{name}/dispatches with ref: main
- **Reward:** Zero manual GitHub settings page visits
- **Errors:** First attempt asked user to add secrets manually — corrected by building the skill
