### PR084: og Brain — WhatsApp Bridge — Persistent JSONL Store + History Backfill
- **Domain:** Infrastructure / WhatsApp
- **Learned:** 2026-04-17
- **Strength:** 1
- **Source:** brain-retro

**Cue:** Setting up WhatsApp message persistence and historical backfill.

**Routine:**
1. Modify viz/whatsapp-bridge/src/index.ts:
   - Add JSONL append store at /app/data/messages.jsonl with dedup by message ID
   - Enable syncFullHistory: true + browser: Browsers.macOS('Desktop')
   - Handle messaging-history.set event for bulk history import
   - Add /whatsapp/messages/since endpoint for delta pulls
   - Add API key auth via WA_API_KEY env var
2. Update docker-compose.yml: add wa-data volume mounted at /app/data
3. Update Dockerfile: add git to alpine image (RUN apk add --no-cache git), create data dir
4. Deploy: copy files to EC2, docker compose build wa-bridge --no-cache && docker compose up -d wa-bridge
5. For history backfill: stop wa-bridge, docker volume rm brain-viz_wa-auth, start wa-bridge, scan QR
6. Run whatsapp-puller.py to sync messages to brain/sync-state/whatsapp-conversations.json
- **Gotcha:** Alpine image needs git installed (npm install fails without it for Baileys)
- **Gotcha:** After history sync + reconnect, sock variable goes stale — restart container to fix sends
- **Gotcha:** nginx-active.conf on EC2 is immutable (chattr +i) — add /whatsapp/ route via backend proxy instead
- **Verification:** curl https://goodhealth.lv/whatsapp/status shows persistedMessages > 0 and historySyncDone: true
- **Apply when:** Setting up or troubleshooting WhatsApp bridge
