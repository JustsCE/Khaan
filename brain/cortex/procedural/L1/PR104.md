# PR104 -- Budget Guard 3-Zone Enforcement

## Cue
Scheduled agents running out of time before cleanup/merge.

## Zones (in rule-engine.py PreToolUse + UserPromptSubmit)
- NORMAL (>780s remaining): all tools allowed
- BUDGET_GUARD (180-780s): forces brain cycle, blocks non-cycle tools
- CLEANUP_ONLY (<180s): forces cleanup only, blocks everything except Bash

## Constants
AGENT_CYCLE_THRESHOLD = 780 (13 min)
AGENT_CLEANUP_THRESHOLD = 180 (3 min)

## Key: this is mechanical (sys.exit(2) tool block), not just context injection.

## Two enforcement paths (DO NOT CONFUSE)
- **Interactive sessions**: message_counter gap >= 5 → MAINTENANCE_DUE → blocks tools until /brain-cycle
- **Agents**: Budget Guard TIME-based zones → forces cycle at remaining <= 780s
- Message counter is IRRELEVANT for agents. Agents don't have "user messages" — they have a time budget.
- Brain cycle in agent prompts is redundant — the Budget Guard handles it mechanically.
