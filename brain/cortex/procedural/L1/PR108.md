# PR108: Dashboard Resurrection Bug — Root Cause and Fix

**Cue:** Dashboard tasks/digests reappear after being cleared.

**Root cause:** `dashboard/tasks/slack.json`, `dashboard/tasks/fireflies.json`, and `brain/dashboard/latest-digest.json` were tracked in git at the repo root. The webhook handler runs `git reset --hard origin/main` inside the container, restoring these files from HEAD every time.

**Why debugging was hard:** The Node.js task ingestion code (agentSources loop, saveTasks) was the obvious suspect but was NOT the writer. The files were restored by git at the OS level before any JS code ran. Debug logging on saveTasks showed 0 calls, yet files appeared — because git reset happens in execSync before the JS filter code.

**Fix applied (2026-04-21):**
1. `git rm dashboard/tasks/slack.json dashboard/tasks/fireflies.json`
2. `git rm brain/dashboard/latest-digest.json`
3. Added both paths to `.gitignore`
4. Also added `DASHBOARD_ACTIVE_SOURCES` env var gating (defense in depth)
5. Also added `--sort=-committerdate` for branch scanning (newest-first)
6. Deleted 17 stale agent branches (awesome-cannon-*, exciting-wozniak-*)

**Files modified:**
- `~/.claude/.gitignore` — added dashboard paths
- `/opt/brain-viz/backend/src/index.ts` — ACTIVE_SOURCES filter + branch sort
- `/opt/brain-viz/docker-compose.yml` — DASHBOARD_ACTIVE_SOURCES=whatsapp env var

**Verification:** Push to git, wait for webhook, check `/api/dashboard/tasks/slack` returns `[]`.

**Lesson:** When Docker volumes overlap with git repos, `git reset --hard` can restore files that were "deleted" from the container. Always check `git show HEAD:<path>` before assuming a file isn't git-tracked.
