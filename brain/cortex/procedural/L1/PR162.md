### PR162: og Brain — Duplicate Triggers Must Stay In Sync

- **Cue:** Editing the prompt, MCP connections, sources, outcomes, model, or session_context of any agent that has more than one RemoteTrigger pointing at the same agent identity (e.g. Teacher has TWO triggers: morning at 01:00+04:00 UTC and afternoon at 14:30 UTC).
- **Stage:** conscious
- **Why this exists:** Standard 5-field cron cannot express times whose minute differs (cannot do 01:00 + 04:00 + 14:30 in one expression — minutes are 0, 0, 30 — Anthropic rejected compound cron with HTTP 400 'expected exactly 5 fields'). The workaround was to create a second trigger 'Teacher Afternoon' (`trig_01UwdFVjCt9zJFrMXnnfqdpT`) cloning the primary 'Teacher' (`trig_01LfWxRrTEdzXsEeUSQaBUaq`). This doubles the surface for config drift — exactly the bug PR161 documents.
- **Routine:** Whenever updating ANY field of a Teacher trigger:
  1. List both Teacher trigger IDs from this entry: primary `trig_01LfWxRrTEdzXsEeUSQaBUaq`, afternoon `trig_01UwdFVjCt9zJFrMXnnfqdpT`.
  2. Apply the SAME RemoteTrigger.update payload to BOTH (changing only the cron_expression and the trigger name if those should differ).
  3. Verify with RemoteTrigger.get on both. Diff them: only `cron_expression`, `name`, `id`, `created_at`, `updated_at`, `next_run_at` should differ.
- **Pitfall:** Forgetting one. Symptom: behavior changes only on certain run times. Detection: a periodic diff job in brain-cycle Phase 10 — list both triggers, diff their `job_config.ccr.session_context` and `events[0].data.message.content`, alarm on any difference.
- **Future cleanup:** If Anthropic adds compound-cron support, collapse to a single trigger and delete this rule.
- **Reinforced by:** E097 (Teacher 2nd-trigger creation 2026-04-28).
