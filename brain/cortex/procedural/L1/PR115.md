# og Brain — PR115: Chi-squared Validity — Three-Layer Check

## Cue
Running chi-squared tests on stratified/filtered data; need to determine which tests are "valid" (meaningful) vs degenerate.

## Routine — validity must check all three before running chi²:

### Layer 1: Both groups present
```python
if GROUP_A not in ct.columns or GROUP_B not in ct.columns:
    skip  # one weight group has no data in this Robson subset
```

### Layer 2: At least 2 categories × 2 groups (ct.shape check)
```python
if ct.shape[0] < 2 or ct.shape[1] < 2:
    skip  # single category = degenerate; df=0, p=1.000 (meaningless)
```
**Why critical:** `scipy.chi2_contingency` on a 1×2 table returns χ²=0, p=1.000 — looks like "identical distributions" but actually means "no variance to test". This bug cost a full iteration.

### Layer 3: Minimum cell count ≥ 5
```python
if ct.values.min() < 5:
    skip  # Cochran's rule — chi² approximation unreliable for small cells
```
Alternative for small cells: Fisher's exact test (still valid for any sample size).

### Layer 4: Run chi² WITHOUT Yates' continuity correction
```python
chi2, p, dof, _ = stats.chi2_contingency(ct, correction=False)
```
**Why:** Yates' default `correction=True` for 2×2 subtracts 0.5 from |O−E| before squaring AND clamps negatives to 0. When all deltas are < 0.5, every cell gets clamped → χ² = 0 exactly, p = 1.000. False-degenerate result.

## Gotcha summary
The combination bug: my initial implementation passed layers 1, 3, and ran chi² with Yates. Layer 2 missed meant 1×2 tables slipped through as "valid". Once fixed, Yates' correction still produced false χ²=0 on small 2×2 tables. **Both bugs presented with identical symptoms (p=1.000)** — classic hard-to-debug because different root causes produced the same surface error.

## Verification
For Robson 1 × Attiecību statuss (Justs' example):
- Raw: Virssvars 22/13, Adipozi 9/6
- |O − E| = 0.3 in every cell
- Yates clamps → χ²=0.00, p=1.000 (WRONG — fake degenerate)
- Without Yates: χ²=0.037, p=0.848 (correct — small non-significant real test)

## Reward
Validity markings match reality. No more "p=1.000" rows making it past review.
