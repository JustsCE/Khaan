### PR068: Vericity Mobile-First Map Architecture
- **Stage:** conscious
- **Learned:** 2026-04-14
- **Strength:** 1

**Cue:** Building mobile layout for map-based pages in Vericity

**Routine:**
1. Use useIsMobile() hook (768px breakpoint) to branch layout
2. Hide Layout.tsx header on mobile map pages (isMapPage + isMobile)
3. Render MobileMapHeader (logo + back + search + stats bar) as replacement
4. Map fills remaining viewport below header (absolute positioning, top={headerH})
5. Circle buttons (MobileCircleButton) stacked on left side for each pane
6. Bottom sheets (MobileBottomSheet, 60vh) for pane content, one at a time
7. Pin placement: crosshair-in-center mode (not tap — conflicts with polygons)
8. Image gallery: full-screen overlay with 15 scrollable images
9. Desktop path stays completely unchanged (if !isMobile, return desktop JSX)

**Gotchas:**
- Tap-to-place pins conflicts with district polygon click handlers — use crosshair instead
- Touch drag on bottom sheet needs startY ref tracking, dismiss if drag > 100px
- Map click events don't work reliably on mobile with Leaflet — read map.getCenter() instead
- FloatingPane has no touch support — never render it on mobile

**Prerequisites:** useIsMobile hook, MobileBottomSheet/CircleButton/MapHeader components
**Verification:** Chrome DevTools phone mode, test all circle buttons open correct sheets
