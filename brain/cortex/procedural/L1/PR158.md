## Procedure

When triaging dashboard tasks from meeting transcripts, apply these dedup signals:

1. **Same meeting source + same people + same domain = likely same task**
   - Tasks extracted from the same Fireflies transcript that reference the same people and business domain are probably sub-steps of one workflow
   - Example: "Add handover dates to Excel" + "Add Justus+Mayuri to Merge1 meetings" → both from same check-in, same people, same handover logistics

2. **Administrative sub-steps of one workflow = single task from user's perspective**
   - If tasks are sequential steps toward one outcome, group them as one task
   - The user thinks in outcomes ("integrate new team members"), not sub-steps

3. **Anti-pattern: same meeting but different workflows should NOT group**
   - A meeting might cover multiple unrelated topics
   - Only group when people + domain + goal align

## Application
- WhatsApp triage agent: apply before creating dashboard tasks
- Solo time agent: check for duplicates before acting on task list
- Recurrence detector: use canonical title from grouped tasks, not individual sub-steps
