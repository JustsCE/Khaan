### PR017: og Brain — Brain Recall Architecture Migration (v1 -> v2)
- **Stage:** conscious
- **Learned:** 2026-04-06
- **Executions:** 1
- **Strength:** 1
- **Cue:** Brain cortex files become too large for efficient recall, or recall-index needs enrichment
- **Routine:**
  1. Write a Python migration script that:
     a. Parses each monolithic .md file using regex split on ### ID: headers
     b. Extracts metadata (strength, domain, learned, cue, rule, facts, tags)
     c. Writes each entry to its own .md file (cortex/semantic/S001.md, prefrontal/R001.md, etc.)
     d. Auto-generates gists from first fact line (semantic), cue (procedural), title (episodic), rule text (prefrontal)
     e. Builds inverted_terms index by tokenizing headings + facts + cue + apply_when + tags
     f. Enriches recall-index.json: each entry ID becomes {id, path, gist, strength, learned}
     g. Renames old monoliths to .bak (not delete)
  2. Run the migration: python script.py (single call per R040)
  3. Update rule-engine.py for new recall flow:
     a. UserPromptSubmit: generate active-recall.json with scored candidates
     b. PreToolUse: track reads, clear gate only when ALL candidates read
     c. Use Agent tool if heredoc quoting blocks direct Bash (R032)
  4. Update consumers: PROMOTE checker (scan dirs), diagnostic (read_dir_entries), viz parser (readDir helper), ARCHITECTURE.md
  5. git add -A, commit, push
- **Prerequisites:** Existing monolithic cortex files, recall-index.json v1
- **Verification:** ls cortex/semantic/ shows individual files, recall-index.json has _version: 2, rule-engine.py passes ast.parse()
- **Reward:** O(1) entry lookups instead of O(n) monolith scans. Enriched recall with scoring.
- **Errors:** (1) Heredoc failed 3x on Python with single quotes - used Agent tool instead. (2) Used Write tool on brain file - triggered approval prompt, user frustrated.
