# og Brain — Docx Build from Template — End-to-End Pattern

When building a docx deliverable that must match Justs' thesis formatting (custom "Table" style, Times New Roman, "Compact" cell paragraphs):

## 1. Prepare clean template (one-time setup per session)

From Justs' v2 thesis doc, strip content + media but keep styles. Steps:
- Open docx as a zip, iterate all parts
- Delete `word/media/*` files
- Strip `<Relationship Type=".../image">` entries from `word/_rels/document.xml.rels`
- Remove `<w:drawing>...</w:drawing>` blocks from `word/document.xml`
- Rewrite as a new zip — this is `/tmp/clean_template.docx`

Result: template with all custom styles (Table, Compact, Heading 1 customized, etc.) but zero content and zero media.

## 2. Build deliverable from clean template

- shutil.copy('/tmp/clean_template.docx', OUT)
- doc = Document(OUT)
- Strip body paragraphs/tables, keep sectPr
- Add new content using:
  - style='Normal' + manual bold run for headings (NOT Heading 1/2 — see R123)
  - Tables: t.style = doc.styles['Table']
  - Cell paragraphs: pp.style = doc.styles['Compact']
- body.append(sectPr) — reattach
- doc.save(OUT)

## 3. Critical rules
- Use Times New Roman explicitly on every run via a helper that sets rFonts + lang=lv-LV.
- Never use Word's built-in Heading 1/Heading 2 styles (render as Calibri blue). Use Normal + bold run.
- Tables: apply `doc.styles['Table']` (his custom style), then re-apply center jc + header tblHeader via OxmlElement.
- Cell paragraphs: always `doc.styles['Compact']`.
- Bullets: Normal paragraph + manual bullet character prefix + hanging indent. No 'List Bullet' style.
