# PR106: Gmail MCP Attachment Download to EC2

**Cue:** Need to download email attachments (PDFs, images) from Gmail and read them on EC2.

**Problem:** The Gmail MCP container writes files to its own ephemeral filesystem. `download_attachment` reports success but files don't exist on the EC2 host.

**Routine:**
1. Add volume mount to mcp-gmail in `/opt/brain-viz/docker-compose.yml`:
   ```yaml
   volumes:
     - /home/ubuntu/.claude/downloads:/downloads
   ```
2. Create host directory: `mkdir -p /home/ubuntu/.claude/downloads`
3. Rebuild container: `cd /opt/brain-viz && docker compose up -d --no-deps --build mcp-gmail`
4. Call `download_attachment` with `savePath="/downloads"` and explicit `filename`
5. Files appear at `/home/ubuntu/.claude/downloads/<filename>` on host
6. Read with `pdftotext` (for PDFs) or Read tool directly

**Gotcha:** The MCP tool always reports "download successful" regardless of whether the host can see the file. The `:ro` flag must NOT be on this mount.
**Gotcha:** `poppler-utils` needed for PDF text extraction (`sudo apt-get install poppler-utils`).
**Gotcha:** PDFs downloaded this way are owned by root (container runs as root). Read works but write needs sudo.

**Prerequisites:** Docker compose stack running on goodhealth.lv, Gmail OAuth tokens valid.
**Verification:** `ls -la /home/ubuntu/.claude/downloads/` shows files with correct sizes.
