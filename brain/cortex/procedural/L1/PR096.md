# PR096: Brain Diagnostic Count-Parsing Bug Fix

**Cue:** Diagnostic showing wrong declared/actual counts for cortex regions (especially prefrontal showing declared=1, actual=69).

**Root cause:** Two bugs in diagnostic.py:
1. `parse_count_line()` scanned concatenated file contents for `Count: N` regex — matched false positives inside rule content (e.g., `projectOnboardingSeenCount: 1` in R097.md)
2. `count_entries()` counted `### ` headers only, but ~40 prefrontal files use frontmatter format (`---` / `# R###:`) — missed those entirely

**Fix (2026-04-18):**
- Added `count_dir_files(directory)` — counts actual .md files in a directory
- Directory-based regions (semantic, procedural, episodic, prefrontal) now use file counts
- Hippocampus still uses Count header + `### ` counting (single-file region)
- Display changed from "N declared = M actual = K thalamus" to "N files = K thalamus"
