### PR028: og Brain — EC2 Instance Recovery (goodhealth.lv)
- **Domain:** Infrastructure / Deployment
- **Learned:** 2026-04-07
- **Reinforced:** 2 time
- **Strength:** 1
- **Cue:** goodhealth.lv is down / SSH times out
- **Routine:**
  1. Check instance state via boto3 (profile='brain-viz', region='eu-north-1', instance='i-09106d98eab1f2f12')
  2. If running but SSH times out: STOP then START (not reboot) — refreshes hypervisor
  3. Wait for instance_running, then wait 60s for SSH daemon
  4. SSH with PEM: `ssh -i ~/Desktop/good_health_machine.pem ubuntu@13.62.204.81`
  5. Start containers: `cd /opt/brain-viz && docker compose up -d`
  6. If nginx fails (upstream not found): `docker compose down nginx && docker compose up -d nginx`
  7. Verify: `curl -sk https://goodhealth.lv/api/health`
- **Gotcha:** After reboot, containers may not auto-start even with restart: unless-stopped (if removed by docker compose down). Fix: systemd service brain-viz.service (oneshot, enabled) runs docker compose up -d on boot.
- **Gotcha:** nginx crashes if MCP containers are not ready. Use `resolver 127.0.0.11` + `set $var upstream:port` for optional upstreams.
- **Gotcha:** nginx-active.conf had immutable flag. Use `sudo chattr -i` before modifying.
- **Gotcha:** Reboot != stop/start. Reboot does not fix networking issues. Stop/start does.
- **Prerequisites:** brain-viz AWS profile in ~/.aws/credentials, PEM key at ~/Desktop/good_health_machine.pem
- **Verification:** curl returns 200 with cycleCount
- **Apply when:** goodhealth.lv is unreachable
