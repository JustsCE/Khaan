### PR047: CLIP Scorer Container on Azure Functions
- **Stage:** conscious
- **Strength:** 1

**Cue:** Need to deploy ML model (CLIP/torch) that is too large for Consumption remote build

**Routine:**
1. Build Docker image with model baked in (no runtime download):
   - `RUN python -c "import open_clip; open_clip.create_model_and_transforms('ViT-B-32', pretrained='openai', cache_dir='/opt/clip-model')"`
   - Set `HF_HOME=/opt/clip-model` in Dockerfile ENV
2. Push to Azure ACR: `docker push vericityacr.azurecr.io/IMAGE:TAG`
3. Create Premium EP1 plan (containers need Premium, not Consumption)
4. Set plan to scale-to-zero: `az functionapp plan update --min-instances 0 --max-burst 1`
5. Create function app with container image
6. Set app settings: SS_STORAGE_CONNECTION, CLIP_MODEL, TIME_BUDGET

**Gotchas:**
- torch is ~2GB, Consumption remote build runs out of disk space
- Cold start downloads model from HuggingFace if not baked in = 3-5min wasted
- Premium EP1 with scale-to-zero costs ~$5-10/mo (vs $150 always-on)
- Build on EC2 (has Docker), not local Windows (no Docker)
