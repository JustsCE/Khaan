### PR048: Cross-Region Latency Debugging
- **Stage:** conscious
- **Strength:** 1

**Cue:** Azure Function is slow when accessing Azure SQL or other regional services

**Routine:**
1. Check regions: `az functionapp show --query location` vs `az sql server show --query location`
2. If different regions, deploy a test function in the SAME region as the database
3. Compare timing: same code, different region
4. Example: northeurope->swedencentral SQL = 600s timeout. swedencentral->swedencentral = 95s.

**Key fact:** Azure SQL inudat.database.windows.net is in swedencentral. Function apps doing heavy DB work must also be in swedencentral.

**Verification:** `curl` the HTTP endpoint and check time_seconds in response
