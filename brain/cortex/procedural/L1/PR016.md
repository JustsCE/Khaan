### PR016: og Brain — Deploy MCP Servers to goodhealth.lv EC2
- **Stage:** conscious
- **Learned:** 2026-04-06
- **Executions:** 1
- **Strength:** 1
- **Reinforced:** 2
- **Cue:** Need to add a new MCP server for an external service (Slack, AWS, SQL, Gmail, etc.)
- **Routine:**
  1. Create Dockerfile in `viz/mcp-servers/<name>/Dockerfile`
     - For stdio-only MCP packages: wrap with `supergateway` → `CMD ["supergateway", "--stdio", "<binary>", "--outputTransport", "streamableHttp", "--port", "<port>"]`
     - For native HTTP packages (e.g., AWS): use native CMD
     - Gotcha: Check the actual installed binary name — npm package name ≠ binary name (e.g., `@gongrzhe/server-gmail-autoauth-mcp` installs as `gmail-mcp`, not `gmail-mcp-server`)
  2. Add service to `viz/docker-compose.yml` with env vars from `.env`
  3. Add nginx location block in `viz/nginx.conf`: `location /mcp/<name>/ { proxy_pass http://mcp-<name>:<port>/; }`
  4. SCP files to EC2: `scp -i $PEM <files> ubuntu@goodhealth.lv:/opt/brain-viz/`
  5. Upload `.env` with credentials to `/opt/brain-viz/.env` (never committed)
  6. Build: `docker compose build --no-cache mcp-<name>`
  7. Start: `docker compose up -d mcp-<name>`
  8. Update nginx-active.conf: `chattr -i`, copy, restart nginx container
     - Gotcha: nginx-active.conf is immutable (`chattr +i`) on EC2 — must remove flag first
     - Gotcha: nginx needs full container restart (not just reload) to pick up new Docker DNS for MCP containers
  9. Add to `settings.json` mcpServers: `{"type": "http", "url": "https://goodhealth.lv/mcp/<name>/mcp"}`
  10. Verify: `curl -X POST https://goodhealth.lv/mcp/<name>/mcp -H "Content-Type: application/json" -H "Accept: application/json, text/event-stream" -d '{"jsonrpc":"2.0","id":1,"method":"initialize",...}'`
      - 405 on GET = correct (MCP is POST-only)
      - Response should contain `serverInfo` with name/version
- **Prerequisites:** SSH access via `~/Desktop/good_health_machine.pem`, Docker on EC2, nginx running
- **Verification:** All endpoints return `serverInfo` on MCP initialize POST
- **Reward:** MCP tools available to Claude Code via HTTPS
- **Gotchas:**
  - `@connorbritain/mssql-mcp-server` needs `SERVER_NAME` + `DATABASE_NAME` env vars, NOT `MSSQL_CONNECTION_STRING`
  - Gmail autoauth needs `gcp-oauth.keys.json` in working dir (not env vars) + token in `~/.gmail-mcp/credentials.json`
  - AWS MCP (`awslabs.aws-api-mcp-server`) is Python/uvx, not npm — needs Python + uv in Docker image, wrapped with supergateway
  - supergateway `--stdio` flag takes the full command as a single string argument
