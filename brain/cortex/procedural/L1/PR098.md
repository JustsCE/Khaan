# PR098: merge_to_main MCP Auto-Recovery

**Cue:** merge_to_main fails with "unmerged files" or "dirty working tree"

**Root cause:** Previous failed merges leave unresolved conflicts in the MCP container's git repo (/brain volume). The repo stays dirty permanently — every future merge attempt fails at `git pull`.

**Fix (deployed 2026-04-20):** Auto-recovery block at start of merge_to_main():
```python
run_git("git merge --abort")
run_git("git cherry-pick --abort")
run_git("git rebase --abort")
run_git("git reset --hard HEAD")
run_git("git clean -fd")
# Then: git reset --hard origin/main (instead of git pull)
```

**Apply when:** Any MCP tool that does git operations in a persistent volume. Always assume the repo could be dirty from a previous failed run.
