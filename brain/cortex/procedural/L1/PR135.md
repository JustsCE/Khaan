# og Brain — Debugging a Failing Agent Trigger (Incremental Test Method)

**Cue:** Agent trigger fires but produces no branch, no commits, no output

**Routine:**
1. Test 1 — BARE MINIMUM: Simple echo + commit + merge. No setup, no hooks. Confirms platform can run trigger at all.
2. Test 2 — ADD SETUP: Copy settings.json, create symlinks, write .agent-mode. Confirms hooks don't crash.
3. Test 3 — ADD COMPILE-SELF: Run compile-self.py identity boot. Isolates identity boot failures.
4. Test 4 — FULL PROMPT: Restore full prompt. If this fails but test 3 passed, problem is prompt length or task complexity.
5. Test 5 — VARY SOURCES: Try single-source vs dual-source. Isolates platform issues with multiple repos.

**Root causes found 2026-04-23:**
- RECALL_GATE blocks Bash for teacher mode (Bash not in SAFE_TOOLS). Fix: exempt teacher from recall gate.
- Scheduler state inherited from repo. If MAINTENANCE_DUE is committed, agent inherits it and ALL tools blocked. Fix: auto-recover scheduler to NORMAL on agent boot.
- Dual-source config may cause platform issues. Workaround: single source, clone second repo via bash.

**Verification:** Each test should produce a branch within 3-5 minutes. No branch = failure.

**Critical lesson:** Always test incrementally. Never assume the full prompt works just because the parts do.
