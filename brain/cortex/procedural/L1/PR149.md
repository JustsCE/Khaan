# PR149: Verify Host Before Proposing SSH

## Cue
About to suggest "SSH into goodhealth.lv to run X" or include `ssh -i ~/Desktop/good_health_machine.pem ubuntu@goodhealth.lv` in any plan.

## Mandatory check (run FIRST, every time)
```
hostname
hostname -I
curl -s http://169.254.169.254/latest/meta-data/instance-id
```

If hostname starts with `ip-172-31-` and metadata returns `i-09106d98eab1f2f12`, **you ARE goodhealth.lv**. Run commands locally. Do not propose SSH.

## Why this rule exists
2026-04-26: Justs corrected this misconception 9 times in one session before demoting Brain INTERN → HIGHSCHOOLER. The brain runs as a systemd service on goodhealth.lv (per PR076 — "EC2 Remote Control Setup"). Every Bash tool call executes ON the EC2.

## Apply when
- Any AWS EC2 task on goodhealth.lv
- iptables / sysctl / systemd commands  
- File operations on /home/ubuntu/
- Reading /etc/ files
- Any time the words "SSH" or "ec2" appear in a plan
