# PR095: Fix MCP Server Containers — Binary Name + ENV Pattern

**Cue:** MCP server Docker container in restart loop with module import errors.

**Pattern:** pip-installed MCP packages (awslabs, gmail-mcp, etc.) register console_scripts entry points. The correct CMD is the binary name, NOT python -m.

**Diagnostic steps:**
1. Check logs: `docker logs <container> --tail 10`
2. If "No module named X.__main__": wrong CMD
3. Find binary: `docker run --rm --entrypoint "" <image> ls /usr/local/bin/*mcp*`
4. Fix Dockerfile CMD to use the binary name

**Known fixes:**
- gmail-mcp-server → `gmail-mcp` (not `gmail-mcp-server`)
- awslabs.aws-api-mcp-server → `awslabs.aws-api-mcp-server` (binary matches package name)

**Also check:** streamable-http transport may require `AUTH_TYPE=no-auth` env var (awslabs packages do).
