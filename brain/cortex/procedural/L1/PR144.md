# og Brain — PR144: Public Lambda via API Gateway HTTP API

## Cue
You need a public HTTPS endpoint for an AWS Lambda function on an account where
Lambda Function URLs return 403 even with correct resource policy
(FunctionURLAllowPublicAccess + lambda:InvokeFunctionUrl for principal `*`).
Symptom: `aws lambda invoke` works, public URL doesnt. Account 487042309472
specifically has this restriction (likely sandbox limit — ConcurrentExecutions: 10).

## Routine

### 1. Diagnose: confirm its an account-level block, not your config
- Direct `aws lambda invoke` works → Lambda is healthy
- Public URL `curl https://<id>.lambda-url.<region>.on.aws/` returns 403 → account block
- Spin up a brand-new test Lambda with default URL config; if THAT also 403, account-level confirmed

### 2. Delete the broken Function URL
- `aws lambda delete-function-url-config --function-name <fn>`
- `aws lambda remove-permission --function-name <fn> --statement-id FunctionURLAllowPublicAccess`

### 3. Create API Gateway HTTP API with Lambda integration (one command)
- `aws apigatewayv2 create-api --name <fn>-api --protocol-type HTTP --target <lambda-arn>`
- The --target shortcut auto-wires the $default route to the Lambda. No further route or integration config needed.
- Returns `ApiId` and `ApiEndpoint`.

### 4. Grant API Gateway permission to invoke Lambda
- `aws lambda add-permission --function-name <fn> --statement-id ApiGatewayInvoke --action lambda:InvokeFunction --principal apigateway.amazonaws.com --source-arn "arn:aws:execute-api:<region>:<account>:<ApiId>/*/*"`
- The `--function-url-auth-type` flag is NOT needed (different action: InvokeFunction not InvokeFunctionUrl).

### 5. Test
- `curl "<ApiEndpoint>/?action=status"` should return 200

## Prerequisites
- Lambda function exists and works (verified via direct aws lambda invoke)
- IAM role attached to Lambda has needed runtime permissions
- AWS credentials with apigatewayv2:CreateApi + lambda:AddPermission

## Verification
- curl returns 200 with expected JSON
- CloudWatch logs /aws/lambda/<fn> show new INVOKE entries

## Reward
Public HTTPS endpoint that works on restricted accounts. Cost: API Gateway is free for the first 1M requests/month.
