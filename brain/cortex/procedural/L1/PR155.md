# PR155 -- og Brain — Use Native Webhooks Over GitHub Actions Wrappers

**Created:** 2026-04-27
**Status:** active
**Strength:** 1
**Reinforced:** 0
**Domain:** procedural (infrastructure)

## Rule

If a GitHub Actions workflow's only job is to **HMAC-sign a payload and POST it to a URL**, replace it with a native GitHub repo webhook.

GitHub repos ship with built-in webhooks (Settings → Webhooks) that:
- POST to any URL on push
- Auto-sign with `x-hub-signature-256` using a secret you set
- Retry on failure with backoff
- Cost zero Actions minutes

The Actions wrapper form is **redundant** — anything it does, the native webhook does for free.

## Configuration via gh CLI

```bash
gh api -X POST /repos/OWNER/REPO/hooks \
  -f name=web \
  -f 'config[url]=https://target.example/api/webhook/github' \
  -f 'config[content_type]=json' \
  -f 'config[secret]=$WEBHOOK_SECRET' \
  -f 'config[insecure_ssl]=0' \
  -F active=true \
  -f 'events[]=push'
```

## When to use Actions instead

Only when the workflow does **real work** beyond signing+posting: building, testing, deploying, transforming files, calling third-party APIs with secrets that shouldn't leak to the receiver.

## Applies to

- Webhook fan-out workflows
- Notification-only workflows
- Anything POST-with-secret-and-die

## Origin

GH Actions free tier exhausted on `claude-brain` repo (2026-04-27). The brain-webhook.yml workflow was 30 lines that signed `BODY` with `WEBHOOK_SECRET` and POSTed to `goodhealth.lv/api/webhook/github` — exactly what `Settings → Webhooks` does natively.
