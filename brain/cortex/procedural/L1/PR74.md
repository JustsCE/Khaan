## Pattern: Slack slash command + symbol handling

When Slack sends slash command payloads as application/x-www-form-urlencoded, urllib.parse.parse_qs decodes + as space. This breaks any text containing literal + (e.g. US+CA, EU+RoW).

## Fix
Do not validate in the trigger/router Lambda. Let the downstream worker Lambda handle normalization. Workers should strip spaces, normalize case, and map variants to canonical forms.

## Applied
- FORECAST-review-trigger: removed strict region/shoptype validation, kept only shop-level name checks
- FORECAST-review-region: parse_region_request() already handles space-stripped normalization
