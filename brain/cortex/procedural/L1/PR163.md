### PR163: og Brain — Diagnose Silent-Degrade in Scheduled Agents

- **Cue:** A scheduled agent appears to be running but a downstream artifact is stale (e.g. teacher-brain repo last commit > 3 days, agent's own journal not growing, expected file not appearing). Activity logs may falsely show the agent as healthy.
- **Stage:** conscious
- **Why this exists:** 2026-04-28 — Justs raised 'TEACHER IS DEAD' because teacher-brain pushed_at was 3 days stale. Diagnosis took 11 minutes and proved Teacher was alive but degraded — running daily, committing claude-brain, but unable to push to teacher-brain because the repo was missing from `session_context.sources`. Agent reported success in logs because main-brain commits worked. The downstream-staleness signal was the only true tell.
- **Routine:**
  1. **Confirm staleness with authoritative source.** `gh api repos/OWNER/REPO` and inspect `pushed_at`, `updated_at`. Do NOT trust agent activity logs alone — they may only cover one phase.
  2. **Reconcile activity-log claims vs git author log.** `git log --since=DATE --format='%H | %an | %s'` and grep for the agent's commit-author identity ('Claude' for Anthropic platform agents, 'Brain (EC2)' for cron-driven host runs). If author commits exist for dates the activity log claims, the work-on-main phase is fine and the failure is in a later phase.
  3. **Inspect trigger config.** `RemoteTrigger.list` then `RemoteTrigger.get(trigger_id)`. Compare `session_context.sources` and `session_context.outcomes` to a known-healthy peer trigger (e.g. Solo Time) — every repo the agent reads from must be in `sources`; every repo it pushes to must be in `outcomes`.
  4. **Inspect agent setup script.** Read `brain/agent-setup.sh` for the agent's mode. Look for fallback clone paths that hit private repos without auth — those are silent-degrade vectors. Anthropic auth is provided ONLY for repos in `sources`.
  5. **Inspect agent prompt.** Read the trigger's `events[0].data.message.content`. Verify the prompt instructs commit + push for every repo the agent should touch.
  6. **Trace the silent-fail path.** When the platform-mounted clone is missing, agent-setup.sh falls back to `git clone https://github.com/OWNER/REPO.git $HOME/.X`. For private repos this fails (no auth), `mkdir -p` creates a stub, the agent runs degraded with empty memory.
- **Quick health-check command for any agent:**
  `gh api repos/OWNER/REPO --jq '.pushed_at,.updated_at'` — if either is > 48h old and agent is supposed to fire daily, investigate.
- **Pitfall:** activity-log entries written by the work-on-main phase mask later-phase failures. The cleanest detection is downstream artifact freshness, not agent self-report.
- **Reinforced by:** E097 (Teacher death incident).
