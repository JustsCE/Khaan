### PR005: og Brain — Brain Repo Sync Workflow
- **Stage:** conscious
- **Learned:** 2026-03-31
- **Executions:** 1
- **Strength:** 1
- **Cue:** Starting a session that will edit brain files, or brain cycle needs to persist changes
- **Routine:**
  1. Check if ~/.claude/ and project dir are in sync: `cd ~/.claude && git log --oneline -1` vs project dir
  2. If diverged: push from ~/.claude/ first, then pull into project dir
  3. Always edit brain files from within the brain project directory (permissions work there)
  4. After edits: commit + push from working directory
  5. If working from a DIFFERENT project, use Bash for brain file writes (Bash permission covers all paths)
- **Reward:** Brain files in sync, no permission prompts
- **Errors:** First sync required stash + pull + stash pop due to local CLAUDE.md changes
