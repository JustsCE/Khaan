### PR069: Vericity ACR Build on Windows
- **Stage:** conscious
- **Learned:** 2026-04-14
- **Strength:** 1

**Cue:** Building and deploying Vericity Docker image to Azure Container Apps

**Routine:**
1. az acr build crashes with Unicode error on Windows — log streaming breaks on Latvian chars
2. Pipe through cat to avoid crash: az acr build ... 2>&1 | cat | tail -5
3. If build seems stuck, check: az acr task list-runs --top 3 -o json
4. --no-logs flag does NOT reliably work — silently exits without building
5. Cancel stuck queued builds: az acr task-run cancel --run-id cgXX
6. Deploy: az containerapp update --image vericityacr.azurecr.io/vericity:vNN-tag
7. Verify: parse JSON output for container image field

**Gotchas:**
- PYTHONIOENCODING=utf-8 helps but doesnt fully fix log streaming
- Multiple retries can queue duplicate builds — check with task list-runs
- Build exit code 0 does NOT guarantee the image was pushed (Unicode crash is cosmetic but build may not have started)
- Always verify tag exists: az acr repository show-tags --top 3

**Prerequisites:** az login, correct subscription, vericityacr registry access
**Verification:** az acr repository show-tags shows new tag, containerapp update succeeds
