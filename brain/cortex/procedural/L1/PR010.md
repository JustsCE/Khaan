### PR010: og Brain — Safe Hook Deployment to settings.json
- **Stage:** conscious
- **Learned:** 2026-04-04
- **Executions:** 2
- **Strength:** 1
- **Cue:** Modifying Claude Code hooks in settings.json
- **Routine:**
  1. Test each hook script independently:
     - Shell: `bash -n "/path/to/script.sh"` (syntax) then `bash "/path/to/script.sh" < /dev/null` (runtime)
     - Python: `python "/path/to/script.py" EventType < /dev/null; echo "EXIT: $?"` — must exit 0 on empty input
  2. Verify Stop hooks can NEVER exit non-zero:
     - `grep -n "exit" script.sh` — every path must be `exit 0`
     - For rule engine: confirm no Stop rules exist in rules-engine.json that could exit 2
  3. Test error resilience — feed garbage input: `echo "not json" | python rule-engine.py PreToolUse`
  4. Write settings.json with ABSOLUTE paths only (`C:/Users/justs.ceplis/.claude/brain/...`)
     - Gotcha: Relative paths resolve from Claude's cwd (the project dir), NOT from ~/.claude/
  5. Verify: `python -c "import json; json.load(open('settings.json'))"` — valid JSON
  6. Git sync: `cd ~/.claude && git add settings.json && git commit -m "hooks: ..." && git push`
  7. Tell user to start a new session — hooks are cached at session start, no hot-reload
- **Prerequisites:** All hook scripts must exist at their absolute paths
- **Verification:** In new session, check `.message-counter` increments and no hook errors appear
- **Reward:** Working hook enforcement without bricking Claude
- **Errors:** E012 — relative paths + Stop hook feedback loop bricked Claude for 20+ minutes
- **Platform gotcha:** Linux hooks (stop-sync.sh, python3 paths) fail harmlessly on Windows. Wrap with OS detection or split into platform-specific entries.
