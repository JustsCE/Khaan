# PR054: Vericity Container App Build and Deploy

**Cue:** Frontend or backend code changed, need to deploy to vericity.ai.

**Routine:**
1. Build frontend locally first to catch errors:
   ```bash
   cd frontend && npm run build
   ```
2. Build Docker image via ACR (no local Docker needed):
   ```bash
   cd vericity-backend
   az acr build --registry vericityacr --image vericity:vN-description --file Dockerfile .
   ```
   - Gotcha: Log streaming crashes with UnicodeEncodeError on Windows (cosmetic). Check actual status:
   ```bash
   az acr task list-runs --registry vericityacr --top 2 -o json
   ```
   - Gotcha: Build takes ~4 min. Context upload is slow due to data/models dirs (~200MB)
3. Deploy to Container App:
   ```bash
   az containerapp update --name vericity-app --resource-group vericity-rg --image vericityacr.azurecr.io/vericity:vN-description
   ```
4. Verify (wait ~15-20s for revision swap):
   ```bash
   curl -s "https://vericity.ai/api/v1/health"
   curl -s "https://vericity.ai/" | grep -o 'index-[a-zA-Z0-9_-]*\.js'  # new chunk hash = new build
   ```

**Prerequisites:** `az login`, ACR access, vericity-rg resource group

**Verification:** Health returns 200, new chunk hash in index.html, page loads correctly

**Reward:** New version live on vericity.ai

**Critical version constraints:**
- @react-three/fiber MUST be v8.x (v8.18.0) — v9 requires React 19
- three.js pinned to 0.175.0
- @react-three/drei removed entirely — use native three.js
- Docker build needs --legacy-peer-deps for npm ci
