# PR103 -- Dashboard FloatingPane + Mobile Architecture

## Cue
Building the goodhealth.lv/dashboard UI with Vericity-style panels.

## Desktop (FloatingPane.tsx)
- Custom drag (titlebar mousedown/move/up) + resize (8 edge handles)
- DOM mutations during drag (no React re-render until mouseup)
- localStorage persistence (key: dashboard-layout-v1)
- Z-ordering: module-level counter, click to front
- Viewport clamping on mount + resize
- Glass style: rgba(10,15,26,0.85), blur(24px), hsl(270 80% 60% / 0.2) border

## Mobile (inline in DashboardPage.tsx)
- useIsMobile() hook, 768px breakpoint
- Circle buttons stacked on left (44px, accented ALL TASKS 52px with glow)
- Bottom sheet (65vh) with touch drag-to-dismiss (startY ref, >100px threshold)
- One sheet at a time, ALL TASKS opens by default
- Backdrop overlay click to close

## Key: FloatingPane has NO touch support — never render on mobile (PR067 gotcha)
