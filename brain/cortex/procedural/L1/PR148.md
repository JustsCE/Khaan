# PR148: Instance-NAT Setup

## Cue
- Lambda in private subnet needs internet egress
- A specific EIP must be the source IP (whitelisted by external service)
- That EIP is already attached to an EC2 in the same VPC, same AZ
- You don't want to pay $32/mo for a NAT Gateway

## Routine

### Prerequisites
- EC2 instance with EIP, in same VPC and AZ as Lambda
- Lambda must be in a DIFFERENT subnet from the EC2 (avoids routing loop)
- Both subnets in the same AZ as the EC2's ENI

### Setup steps
1. **On EC2:** enable IP forwarding
   ```
   sudo sysctl -w net.ipv4.ip_forward=1
   echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf
   ```

2. **On EC2:** add iptables MASQUERADE + FORWARD
   ```
   sudo iptables -t nat -A POSTROUTING -s <VPC_CIDR> -o ens5 -j MASQUERADE
   sudo iptables -A FORWARD -i ens5 -j ACCEPT
   sudo iptables -A FORWARD -o ens5 -j ACCEPT
   ```

3. **On EC2:** persist iptables across reboots
   ```
   sudo apt-get install -y iptables-persistent
   sudo iptables-save | sudo tee /etc/iptables/rules.v4
   ```

4. **AWS:** disable source/dest check on EC2's ENI
   ```
   aws ec2 modify-network-interface-attribute --network-interface-id <ENI_ID> --no-source-dest-check
   ```

5. **AWS:** allow Lambda SG → EC2 SG (all protocols)
   ```
   aws ec2 authorize-security-group-ingress --group-id <EC2_SG> --source-group <LAMBDA_SG> -p -1
   ```

6. **AWS:** route Lambda subnet's 0.0.0.0/0 to EC2's ENI
   ```
   aws ec2 replace-route --route-table-id <LAMBDA_RT> --destination-cidr-block 0.0.0.0/0 --network-interface-id <ENI_ID>
   ```

7. **AWS:** ensure Lambda VpcConfig points to the private subnet (not the EC2's subnet)

## Verification
- Invoke Lambda with a test that hits an external service that logs source IP
- Source IP should match EC2's EIP
- Check `iptables -t nat -L POSTROUTING -n -v` — packet counter on MASQUERADE rule should increment

## Trade-offs
- EC2 reboot = brief Lambda outage during reboot
- EC2 carries Lambda's outbound traffic (low for batch jobs, watch for high-volume cases)
- iptables is per-instance — if EC2 is replaced, must re-apply
- Cost: $0 vs ~$32/mo for NAT Gateway

## Pitfalls
- Lambda must NOT be in the same subnet as the EC2 (routing loop on EC2's own outbound)
- Must be same AZ (ENI route targets are AZ-bound)
- Forgetting to disable source/dest check = AWS drops forwarded packets silently
- Forgetting SG ingress = packets reach ENI but get dropped at SG layer
