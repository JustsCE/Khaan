### PR022: og Brain — Safe Multi-Line Brain File Writing via Bash
- **Stage:** conscious
- **Learned:** 2026-04-07
- **Executions:** 3
- **Strength:** 1
- **Cue:** Need to write/rewrite a brain file that contains code blocks, backticks, markdown, or shell-special characters
- **Routine:**
  1. Write a Python script to a temp file: `cat > /tmp/script.py << 'PYEOF' ... PYEOF`
  2. Use raw strings (`r"""..."""`  or regular triple-quotes) for the content
  3. Run: `python /tmp/script.py`
  4. Verify: Read the file back to confirm no corruption
  - Gotcha: NEVER use `python -c "..."` — Bash interprets backticks and $vars inside double quotes
  - Gotcha: Single-quoted PYEOF delimiter prevents variable expansion in the heredoc itself
- **Prerequisites:** Python available on PATH
- **Verification:** Read the file, check for no shell output leaked into content
- **Reward:** Clean brain file with intact code blocks and markdown
