# Claude Code Hook Format Schema

## The Rule
Claude Code settings.json hooks MUST use the nested format with discriminated union on "type":
```json
"EventName": [{ "hooks": [{ "type": "command", "command": "..." }] }]
```

NEVER flatten to `{"command": "..."}` — Claude Code silently ignores malformed hook objects.

## Why
Commit b196db1 (2026-04-18) flattened hooks and broke ALL brain enforcement. Counter stopped, recall gate stopped, SLO stopped. Silent failure — took hours to diagnose.

## Key Details
- Schema uses Zod discriminatedUnion on "type" field
- Valid types: command, prompt, agent, http
- hooks array wrapper groups multiple hooks under one matcher
- Hooks loaded at session start — service restart required after changes

## Apply When
Editing settings.json hooks, debugging hook failures, cross-platform hook setup.
