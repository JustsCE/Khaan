The detached subprocess pattern (Popen, not subprocess.run) is the correct way to run long-running pipelines from hooks. Hooks have timeouts; subprocess.run blocks the hook return. Popen fires and forgets — the hook returns immediately with stale context, subprocess updates navigation files for subsequent turns.
