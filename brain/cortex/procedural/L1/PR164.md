# PR164: Project ID fallback in brain-tasks endpoint

## Problem
Dashboard task ID and project folder name can diverge (e.g., task moi9hen6dcn3 vs folder mohjt910knq1). Backend looked up projects/<task.id>/state.json — silent miss, no progress bar rendered.

## Fix (permanent)
Backend now builds a taskId-to-folder map by scanning all project state.json files before matching. Lookup chain: taskIdToFolder.get(task.id) || task.id.

## Location
viz/backend/src/index.ts — brain-tasks endpoint, project phase attachment block.

## Apply when
Any time the project engine or dashboard task creation changes — verify that task IDs and project folder names stay in sync, but the fallback will catch drift.
