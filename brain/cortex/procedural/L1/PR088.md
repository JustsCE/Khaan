# PR088: Auto-Sync Redundant When Remote Triggers Commit

## Problem
stop-sync.sh runs as a Stop hook on every session end. Remote triggers (WhatsApp Responder, Solo Time, Slack Triage, etc.) already commit and push their own work. The Stop hook then fires AGAIN, creating a second redundant commit. During active WhatsApp sessions, this produced 19+ auto-sync commits in 2 hours, each triggering a GitHub notification on user's phone.

## Solution
Disable stop-sync.sh from the EC2 settings.json Stop hook. Remote triggers handle their own persistence. stop-sync.sh is only needed for local interactive sessions where changes might not be committed.

## Apply When
Modifying EC2 settings.json, debugging git noise, or reviewing sync architecture.
