### PR031: og Brain — Forecast QA - Submit-Verify Workflow
- **Stage:** conscious
- **Learned:** 2026-04-08
- **Executions:** 5
- **Strength:** 1
- **Reinforced:** 1
- **Cue:** User submits forecast file via /submit_forecast
- **Routine:**
  1. submit-verify downloads file from Slack, uploads to S3 (with UTC timestamp in filename)
  2. Reads current_month from Forecast_Input.xlsx config
  3. Validates: shop files get full QA (actuals check, format match, coverage gaps). Shoptype files get partial QA. Region files auto-pass.
  4. If fail: sends error report to Slack AND deletes the failed file from S3
  5. If pass (shop): sends success summary with Prev FC vs New FC delta
  6. If pass (shoptype/region): silently proceeds to submit (no summary)
  7. On pass: invokes FORECAST-submit with entity_name in payload
- **Reward:** Automated quality gate
- **Errors:** SSM bot token lacks chat:write, must use response_url
