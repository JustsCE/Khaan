# PR070: Slack Puller Performance Fix

**Cue:** Slack puller taking 8+ minutes for delta pulls with few new messages.

**Root cause:** filt() uses Slack conversations.updated field which fires on read receipts, presence changes, typing indicators -- not just new messages. Passes 668/758 conversations even when only 4 have new content.

**Fix:** Before calling conversations.history, compare channel watermark (lastTs) against conversation updated timestamp. Skip if difference < threshold (e.g., 60s). Would cut API calls from ~668 to ~20-30.

**Alternative fixes:**
1. Only scrape tracked channels + DMs where updated >> watermark
2. Batch by priority: 5 tracked channels first, then filtered DMs
3. Use conversations.info per-channel to check latest msg ts before full history pull

**File:** viz/scripts/slack-puller.py, filt() function

**Verification:** Delta pull completes in <1 minute with <50 API calls
