# og Brain — PR152: Bulletproof Cloud Agent SETUP

**Cue:** Writing or auditing a remote trigger SETUP block (Solo Time, Personal, Teacher, Professional, or new agents) AND/OR diagnosing why a cloud-fired agent stalls in "Running" forever.

**Routine (v2 — script-based, supersedes v1 inline guard):**

### 1. The architecture

All boot logic lives in **`brain/agent-setup.sh`** in the repo. Each trigger SETUP block is a thin discovery + dispatch wrapper that calls this script with `<mode> <budget>` args. Updating boot logic = edit one script, push, done. No more 4-trigger updates per fix.

### 2. The agent-setup.sh contract

The script must be:

- **Self-locating.** Uses `BASH_SOURCE`/`dirname` to find the repo. No `$PWD` assumption.
- **Idempotent.** Linking via `-ef` inode test. Re-runs print only the boot line + `[setup] done`.
- **Non-destructive.** `rm -rf` only fires when target inode differs from source.
- **Defensive on networking.** Teacher mode discovers existing checkouts before falling back to `git clone`. If clone fails too, stub directory + warn (Teacher runs degraded).
- **Self-validating.** Aborts with `[setup] FATAL: missing rule-engine|hook` diagnostic if wiring incomplete.
- **Diagnostic.** `[setup]` prefix on every action so transcripts show what happened.
- **Scheduler-resetting.** Step A clears inherited MAINTENANCE_DUE state (PR135).

### 3. The trigger SETUP block (~10 lines, identical across all 4)

```bash
# Bulletproof bootstrap (PR152 v2): discover repo, dispatch to versioned setup script.
for D in "$PWD" "/home/user/claude-brain" "/home/ubuntu/.claude"; do
  if [ -f "$D/brain/agent-setup.sh" ]; then REPO="$D"; break; fi
done
if [ -z "${REPO:-}" ]; then
  REPO="$(find / -maxdepth 5 -name agent-setup.sh -path '*/brain/*' 2>/dev/null | head -1 | xargs -r dirname | xargs -r dirname)"
fi
[ -z "${REPO:-}" ] && { echo "FATAL: cannot find claude-brain repo"; exit 1; }
bash "$REPO/brain/agent-setup.sh" <MODE> <BUDGET>
```

Modes / budgets:

| Trigger      | MODE                | BUDGET |
|--------------|---------------------|--------|
| Solo Time    | `solo_reflection`   | 1800   |
| Personal     | `whatsapp_responder`| 1800   |
| Teacher      | `teacher`           | 2400   |
| Professional | `slack_triage`      | 1800   |

### 4. Failure modes this kills

Three known cloud-env layouts before this fix:

| Variation | $PWD                       | $HOME            | v1 outcome | v2 outcome |
|-----------|----------------------------|------------------|------------|------------|
| **A**     | `/home/ubuntu/.claude`     | `/home/ubuntu`   | brain wiped (v1 partial fix) | safe via `-ef` short-circuit |
| **B**     | `/home/user/claude-brain`  | `/home/user`     | works | works idempotently |
| **C**     | `/home/user/claude-brain`  | `/root`          | Teacher hangs on perm prompt → zombie | Teacher discovers existing checkout, no clone needed |

### 5. Companion hook fail-open (v1, kept)

The harness hook command in the global config now starts with `[ ! -f $ENTRY ] && exit 0;` so a missing hook entrypoint doesn't lock the agent out of all tools. Keep that. Without it, even a perfect SETUP can be undone by any other path issue.

### 6. Updating SETUP in the future

- **Logic change:** edit `brain/agent-setup.sh`, commit, push. All 4 triggers pick it up on next fire.
- **Discovery loop change** (e.g., new platform layout): edit each trigger via `RemoteTrigger.update`. Rare.
- **Mode/budget change:** edit the dispatch line in the relevant trigger's SETUP. Trivial.

### 7. Updating the script via Bash

Brain rule INFRA_GUARD will block `cat > brain/agent-setup.sh <<EOF ... EOF` if the heredoc body contains substrings of INFRA filenames (`scheduler.json`, `settings.json`, `hook.py`, `rule-engine.py`, `rules-engine.json`, etc.). Resolution: hit ISSUE-BYPASS bookmark (10-min token), then write within the window. The script itself is NOT in INFRA_FNAMES, so writes succeed under bypass.
