# PR082: WhatsApp Bridge Deploy and QR Pairing

Cue: Need to deploy or reconnect WhatsApp bridge on EC2

Routine:
1. Fix Baileys version: pass version [2, 3000, 1034074495] to makeWASocket
2. SCP updated files to ubuntu@goodhealth.lv:/opt/brain-viz/whatsapp-bridge/src/
3. Rebuild: docker compose build wa-bridge --no-cache && docker compose up -d wa-bridge
4. Get QR via docker exec: call /whatsapp/qr endpoint and pipe through qrcode-terminal
5. User scans QR in WhatsApp > Settings > Linked Devices
6. Auth persists in Docker volume wa-auth

Key facts:
- Server path: /opt/brain-viz (NOT a git repo, deploy via SCP)
- PEM key: Desktop/good_health_machine.pem
- printQRInTerminal DEPRECATED in Baileys 7.x, use qrcode-terminal + /whatsapp/qr
- QR expires in ~20 seconds

Reward: Connected WhatsApp bridge receiving live messages
Errors: Baileys 405 = stale protocol version. TS import of qrcode-terminal needs @ts-ignore.
