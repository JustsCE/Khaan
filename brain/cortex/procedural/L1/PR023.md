### PR023: og Brain — Cross-Machine Brain Portability
- **Stage:** conscious
- **Learned:** 2026-04-07
- **Executions:** 1
- **Strength:** 1
- **Cue:** Setting up the brain on a new machine, or troubleshooting why hooks/settings don't work after cloning
- **Routine:**
  1. Clone brain repo to ~/.claude/ on the new machine
  2. Fix absolute paths in settings.json hooks — replace C:/Users/old_user/ with the new machine's home path (or use ~ paths)
  3. Recreate settings.local.json (gitignored, machine-specific)
  4. Recreate ~/.credentials.json (gitignored: slack_token, Azure SQL)
  5. MCP HTTP servers (goodhealth.lv) work automatically — no setup needed
  6. MCP stdio servers need local binary installation
  - Gotcha: settings.json hooks use absolute Windows paths — won't work on Mac/Linux without path updates
  - Gotcha: settings.local.json is gitignored — must be recreated per machine
  - Gotcha: .message-counter and .last-brain-update are gitignored — brain cycle starts fresh
- **Prerequisites:** Git access to github.com/JustsCE/claude-brain, Claude Code installed
- **Verification:** Run /brain-diagnostic — all checks should pass
- **Reward:** Full brain operational on new machine with all hooks, rules, and recall working
