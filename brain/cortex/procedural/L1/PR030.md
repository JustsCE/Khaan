### PR030: og Brain — Running a Real Temporal Audit
- **Domain:** Brain / Maintenance
- **Learned:** 2026-04-08
- **Reinforced:** 1 times
- **Strength:** 1
- **Cue:** TEMPORAL_AUDIT rule fires (every 10 cycles), or /brain-retro detects audit debt
- **Routine:**
  1. Run: python brain/tracker.py audit
  2. Read brain/navigation/temporal-audit.json for full results
  3. Address HIGH alerts first:
     - Missing references: check if source entry was merged/deleted, update reference or remove
     - SLO violations: note for future optimization, no quick fix
  4. Address MEDIUM alerts:
     - Stale entries: read each flagged entry, verify still correct, update verified_at_cycle in recall-index.json
     - Orphans: add to recall-index.json under appropriate domain
     - Entity dangling refs: remove H### refs (hippocampus entries get consolidated)
  5. Note LOW alerts (no action needed unless pattern persists)
  6. Write ### Audit #N entry to brain/navigation/temporal-audit-log.md documenting what was checked
  7. The file_stale_cycles checker verifies this entry exists — cannot be bypassed
- **Gotcha:** DO NOT just reset the state file to clear the gate. The anti-tamper in file_stale_cycles detects baseline resets and restores the true count.
- **Gotcha:** Updating verified_at_cycle in recall-index.json requires knowing the entry is a dict, not a string.
- **Prerequisites:** tracker.py exists, thalamus.json accessible
- **Verification:** State file shows current_cycle:entry_count matching actual ### count in audit log
- **Apply when:** TEMPORAL_AUDIT blocks, brain health check, end-of-session maintenance
