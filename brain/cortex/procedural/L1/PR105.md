# PR105: Deploy NEW SESSION Button (Session Relay)

**Cue:** Need to add ability to spawn fresh Claude Code remote-control session from /connect page.

**Architecture:** Host-side Python HTTP relay (port 9922) + Docker container proxies to it.

```
Frontend POST /api/remote-control/restart
  -> Express proxies to host relay (bearer token)
    -> systemctl restart claude-brain.service
    -> update-env.sh waits for new env ID
    -> returns new connectUrl
```

**Routine:**

1. Create `/home/ubuntu/brain-shared/session-relay.py` -- Python HTTP server on port 9922
   - POST /restart with bearer token -> restarts claude-brain.service
   - GET /health for liveness check
   - 60s cooldown between restarts
   - Gotcha: Must run as root (systemctl needs privileges)

2. Create systemd unit `/etc/systemd/system/session-relay.service`
   - User=root, Environment=SESSION_RELAY_TOKEN=<token>
   - `systemctl daemon-reload && systemctl enable --now session-relay.service`

3. Update `viz/docker-compose.yml`:
   - Add `extra_hosts: host.docker.internal:host-gateway` to brain-viz
   - Add env vars: SESSION_RELAY_URL, SESSION_RELAY_TOKEN

4. Add `POST /api/remote-control/restart` to `viz/backend/src/index.ts` (after GET endpoint ~line 898)
   - Proxies to relay with 140s timeout (relay itself waits up to 130s for update-env.sh)

5. Add NEW SESSION button to `viz/frontend/src/pages/ConnectPage.tsx`
   - confirm() dialog before restart
   - Loading state while waiting

6. Deploy:
   - Add SESSION_RELAY_TOKEN to /opt/brain-viz/.env
   - Copy files: `cp viz/{docker-compose.yml,frontend,backend} /opt/brain-viz/`
   - `cd /opt/brain-viz && docker compose build --no-cache brain-viz && docker compose up -d brain-viz`

**Verification:**
- `curl http://localhost:9922/health` -> {ok: true}
- `docker exec brain-viz-brain-viz-1 sh -c "wget -q -O- http://host.docker.internal:9922/health"` -> {ok: true}
- `curl https://goodhealth.lv/connect` -> 200

**Gotchas:**
- Docker container needs extra_hosts for host.docker.internal to resolve
- Relay needs root for systemctl restart
- update-env.sh can take up to 120s -- frontend must handle long wait
- Clicking NEW SESSION kills the current brain session (including yours if connected)

**Prerequisites:** claude-brain.service, update-env.sh, Docker with compose v2+
