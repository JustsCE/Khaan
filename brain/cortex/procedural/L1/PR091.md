### PR091: Gmail MCP Deployment on goodhealth.lv
- **Stage:** conscious
- **Learned:** 2026-04-18
- **Strength:** 1

**Cue:** Setting up a new MCP server on goodhealth.lv infrastructure

**Routine:**
1. Dockerfile: Use `supergateway` to bridge stdio MCP to HTTP
   ```dockerfile
   FROM node:20-alpine
   RUN npm install -g @package/name supergateway
   EXPOSE <port>
   CMD ["supergateway", "--stdio", "<binary-name>", "--outputTransport", "streamableHttp", "--port", "<port>"]
   ```
   - Gotcha: Check actual binary name with `ls /usr/local/bin/` — npm package name ≠ binary name
2. docker-compose.yml: Add service with port mapping + credential volumes
   - Mount credentials to where the package expects them (check source code for paths)
3. **Host nginx** (NOT Docker nginx): Add proxy location to `/etc/nginx/sites-enabled/brain-viz`
   ```nginx
   location /mcp/<service>/ {
       rewrite ^/mcp/<service>/(.*)$ /$1 break;
       proxy_pass http://127.0.0.1:<port>;
       proxy_http_version 1.1;
       proxy_set_header Host $host;
       proxy_set_header Connection '';
       proxy_buffering off;
       proxy_cache off;
       proxy_read_timeout 300s;
   }
   ```
   - Gotcha: The HOST nginx (1.24.0 Ubuntu) serves port 443, NOT Docker nginx
   - Gotcha: Rewrite must strip the FULL prefix. `/mcp/service/(.*)` → `/$1`, not `/mcp$1`
4. Reload: `docker compose up -d <service> && sudo nginx -t && sudo nginx -s reload`
5. settings.json: Add `"<name>": {"type": "http", "url": "https://goodhealth.lv/mcp/<service>/mcp"}`
6. Restart Claude Code session for MCP tools to appear

**Prerequisites:** Docker, docker-compose, host nginx with SSL, credentials for the service
**Verification:** `curl -X POST https://goodhealth.lv/mcp/<service>/mcp -H "Content-Type: application/json" -H "Accept: application/json, text/event-stream" -d '{"jsonrpc":"2.0","method":"tools/list","id":1}'`
