# og Brain — Adding MCP Tools to Brain MCP Server

**Cue:** Need to expose a new capability to remote agents (they can't use curl)

**Routine:**
1. Edit `viz/mcp-servers/whatsapp/server.py`
2. Add tool definition to TOOLS list (name, description, inputSchema with properties/required)
3. Add handler in `handle_request()` function (elif tool_name == "new_tool": ...)
4. Handler calls backend at `http://brain-viz:3001/api/...` with `Authorization: Bearer changeme`
5. Return MCP format: `{"jsonrpc":"2.0","id":req_id,"result":{"content":[{"type":"text","text":json.dumps(result)}]}}`
6. Deploy: `scp server.py ubuntu@goodhealth.lv:/opt/brain-viz/mcp-servers/whatsapp/server.py`
7. Rebuild: `ssh ... "cd /opt/brain-viz && docker compose build --no-cache mcp-whatsapp && docker compose up -d mcp-whatsapp"`
8. Test: `curl -s https://goodhealth.lv/mcp/brain/mcp -X POST -H "Content-Type: application/json" -d '{"jsonrpc":"2.0","method":"tools/call","id":1,"params":{"name":"new_tool","arguments":{...}}}'`

**Gotcha:** All MCP connectors on triggers point to the SAME server (goodhealth.lv/mcp/whatsapp). Adding a tool here makes it available to ALL agents.

**Verification:** Tool appears in tools/list response and returns valid JSON.
