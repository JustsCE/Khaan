### PR044: Migrating AWS Lambda to Azure Functions
- **Stage:** conscious
- **Learned:** 2026-04-10
- **Strength:** 1

**Cue:** Need to migrate serverless functions from AWS Lambda to Azure Functions

**Routine:**
1. Download Lambda code: `python -m awscli lambda get-function --function-name NAME --profile personal --region eu-north-1 --query "Code.Location" --output text`
2. Create Azure Function App on REGULAR Consumption plan (not Flex):
   - `az functionapp create --resource-group inudat --name NAME --storage-account inudatbbb5 --consumption-plan-location northeurope --runtime python --runtime-version 3.12 --functions-version 4 --os-type Linux`
   - Gotcha: storage account must be in same resource group
3. Set app settings: `az functionapp config appsettings set --name NAME --resource-group inudat --settings "KEY=VALUE"`
4. Adapt code: boto3 -> azure.storage.blob
   - s3.get_object -> blob.download_blob()
   - s3.put_object -> blob.upload_blob(data, overwrite=True, content_settings=ContentSettings(...))
   - Gotcha: content_settings MUST be ContentSettings object, NOT dict
   - s3.list_objects_v2 -> container.list_blobs(name_starts_with=prefix)
   - Large CSV reads: stream via blob.download_blob().chunks()
   - Gotcha: chunk streaming splits multi-byte UTF-8 chars -- accumulate bytes, decode with trim
5. Handle 230s hard timeout (Consumption plan ignores host.json functionTimeout):
   - Use time budget (200s), checkpoint progress, make functions resumable
   - Stream large files line-by-line, never download all at once
6. Deploy: `func azure functionapp publish APP_NAME`
   - Remote build works on regular Consumption (NOT Flex)
   - Gotcha: Flex rejects ENABLE_ORYX_BUILD / SCM_DO_BUILD_DURING_DEPLOYMENT
7. Test: trigger timer via admin endpoint with master key
   - `curl -X POST "HOST/admin/functions/FUNC_NAME" -H "x-functions-key: MASTER_KEY" -d '{"input": ""}'`
   - Check logs: `az monitor app-insights query --app APP --resource-group RG --analytics-query "traces" 2>&1 | cat`
   - Gotcha: Azure CLI swallows output on Windows -- always pipe through `| cat`

**AWS Learnings:**
- Lambda Layers provide pre-built dependencies (requests, beautifulsoup4, etc.)
- Lambda roles need explicit inline policies for S3 access per path
- Lambda can run up to 15 min (vs Azure Consumption 230s)
- NAT-OPEN Lambda manages NAT gateway for VPC internet access

**Azure Learnings:**
- Flex Consumption = same 230s limit + no remote build. Use regular Consumption.
- Azure Functions Core Tools: `npm install -g azure-functions-core-tools@4`
- Python v2 programming model: single function_app.py with decorators
- Timer trigger cron: 6-field format "0 0 22 * * *"
- Manual timer trigger via /admin/functions/NAME endpoint with master key
- App Insights must be explicitly created and connected via app settings
- ContentSettings class required for blob uploads (not plain dict)
- Blob streaming chunks split UTF-8 multi-byte chars at 4MB boundaries
