# PR089: Cross-Platform Hook Infrastructure (hook.py)

## Cue
Settings.json hooks need to work on both Windows (laptop, Git Bash) and Linux (EC2, native bash) from the same git-synced file.

## Routine
1. **hook.py** at `~/.claude/brain/hook.py` ŌĆö resolves rule-engine.py path via `os.path.expanduser('~')`, runs it via subprocess, catches all crashes ŌåÆ exit 0, only propagates exit 2 (intentional block)
2. **settings.json hook command:** `python "$HOME/.claude/brain/hook.py" <EVENT>; e=$?; [ $e -eq 2 ] && exit 2 || exit 0`
   - `$HOME` is expanded by bash (works on Git Bash + Linux)
   - MSYS2 auto-translates `/c/Users/...` ŌåÆ `C:/Users/...` for native Python on Windows
   - The `; e=$?;` safety net catches even hook.py itself failing
3. **3 events:** UserPromptSubmit, PreToolUse, PostToolUse ŌĆö all route through hook.py
4. **Removed:** brain-cycle.sh (no-op), brain-correct.sh (unreliable) ŌĆö rule-engine.py handles everything

## Gotchas
- **NEVER use `|| true`** ŌĆö it masks exit 2 (enforcement blocks), silently disabling the entire rule engine
- **NEVER prefix command with `bash`** ŌĆö hook runner already does `bash -c "..."`, doubling it creates `bash if [...]` syntax error
- **NEVER use hardcoded paths** like `C:/Users/...` or `/home/ubuntu/...` ŌĆö use `$HOME` + `expanduser('~')`
- Exit code 2 = intentional block. ALL other non-zero = crash = must become 0.

## Prerequisites
- Python in PATH on both platforms
- rule-engine.py exists at `~/.claude/brain/rule-engine.py`
- settings.json synced via git between platforms

## Verification
```bash
# Test hook directly
echo '{"user_message":"test"}' | python ~/.claude/brain/hook.py UserPromptSubmit
echo "exit: $?"

# Test crash safety (missing engine)
mv ~/.claude/brain/rule-engine.py ~/.claude/brain/rule-engine.py.bak
python ~/.claude/brain/hook.py UserPromptSubmit; echo "exit: $?"  # should be 0
mv ~/.claude/brain/rule-engine.py.bak ~/.claude/brain/rule-engine.py

# Test full bash safety net
echo '{}' | bash -c 'python "$HOME/.claude/brain/hook.py" PreToolUse; e=$?; [ $e -eq 2 ] && exit 2 || exit 0'
echo "exit: $?"
```

## Reward
Hooks work on both platforms. User is NEVER locked out by infrastructure failures. Only deliberate rule-engine blocks pass through, with a 3-strike safety valve (MAINTENANCE_FAILED auto-recovery).
