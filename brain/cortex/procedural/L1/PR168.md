# PR168: Migration ordering - cognition before gates

**Cue:** building / replacing a gated cognitive system.

**Routine:**
1. Migrate / populate the data substrate first (cortex, indexes, config). No engines, no hooks.
2. Build the cognitive engines as runnable scripts. Verify each by hand.
3. Build the learning / synthesis engine. Verify cycles run by hand.
4. Build the gating skeleton (rule engine + binaries + flippers + dispatcher). All binaries default 0.
5. Wire LAST: harness wiring, hooks registration. Hooks start firing on registration.
6. Hook registration shipping is the trigger that makes the system live.

**Why this order:** if gates fire before engines exist, a gate could ask for an engine output to clear it; no engine exists; system deadlocks with no recovery. Build cognition first; every output a gate could ask for already exists by the time the gate fires.

**Prerequisites:** migration plan exists; tunable constants centralised.

**Verification:** after each phase, system in coherent state; after wire-in, every gated output already exists.

**Reward:** no deadlock window during bring-up.

**Source incident:** Brain v2 implementation order 2026-05-05. Initial order had skeleton before engines = deadlock window. Justs caught it. Reordered to cognition-first, gates-last.
