### PR041: og Brain — State Machine API Freeze (v3.1)
- **Stage:** conscious
- **Learned:** 2026-04-09
- **Executions:** 1
- **Strength:** 1
- **Cue:** Need to modify state-machine transitions or add new transition methods
- **Routine:**
  1. Frozen API: on_user_message(), begin_cycle(), commit_cycle(), fail_cycle()
  2. load()/save() are infrastructure only, save() asserts valid state before write
  3. Every transition has assert-based invariants (state precondition, counter monotonicity, nonce match)
  4. on_user_message handles: counter increment, NORMAL to MAINTENANCE_DUE (gap>=5), MAINTENANCE_FAILED to NORMAL recovery
  5. fail_cycle owns escalation counting exclusively, callers never manually increment escalation_count
  6. Legacy state-machine.json removed, single JSON file is source of truth
- **Verification:** All 4 functions have assert on entry state; save() asserts VALID_STATES; no legacy mirror writes
- **Reward:** No invalid state transitions possible. Double-increment bug eliminated.
- **Errors:** (none)
