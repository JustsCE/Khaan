### PR34: og Brain — Slack Conversation Pipeline (CACHE-FILTER-SCRAPE)
- **Date:** 2026-04-08
- **Strength:** 1
- **Stage:** conscious
- **Cue:** Need automated Slack data collection for dashboard/agents
- **Routine:**
  1. CACHE: conversations.list (all types, paginated)
  2. FILTER: fixed channels + active DMs/GDMs (check updated timestamp)
  3. SCRAPE: conversations.history with oldest=watermark per source
  4. Resolve user IDs via users.info (persist cache!)
  5. Append to slack-conversations.json, dedupe, trim >20 weeks
  6. Update watermarks, git commit+push
- **Prerequisites:** SLACK_BOT_TOKEN, scopes: channels/groups/im/mpim:history+read, users:read
- **Gotchas:**
  - First run: use 2-week window to avoid timeout
  - Persist user cache in state file (otherwise 100+ extra API calls per run)
  - Git push conflict: use git pull --rebase before push
  - EEST=4 chars, regex must use [A-Z](2,4) not E[ES]T
  - xoxp token works but slower rate limits than xoxb
- **Files:** viz/scripts/slack-puller.py, .github/workflows/slack-puller.yml
- **Result:** 3608 msgs from 128 sources (5ch + 70dm + 53gdm)
