# PR097: Boot Gate Safety — Staleness Auto-Clear Pattern

**Cue:** Any tool-blocking gate that uses a sentinel file MUST have a staleness escape hatch.

**Problem:** Creating `.agent-mode` mid-session blocks all Bash for the interactive user. No UserPromptSubmit fires between sentinel creation and next PreToolUse, so the boot gate never completes. User is locked out.

**Pattern:**
1. Check sentinel file mtime in the gate
2. If sentinel > threshold (10 min for boot gate), auto-delete and fall through
3. Allow Read/Glob/Grep through the gate always (so user can investigate)
4. The existing session timeout (TEACHER_BUDGET) acts as a secondary safety net

**Apply when:** Building any new hook-based gate that blocks tools based on file presence. Always ask: "What happens if this file gets stuck?"
