### PR139: og Brain — Dashboard Hot-Deploy on EC2 (local Docker, bypass GH Actions)
- **Stage:** conscious
- **Learned:** 2026-04-24
- **Executions:** 8+
- **Strength:** 1
- **Cue:** Viz changes need to ship but GH Actions `deploy-viz.yml` SSH step fails (EC2 → EC2 SSH loopback blocked)
- **Routine:**
  1. `cp -r /home/ubuntu/.claude/viz/frontend /home/ubuntu/.claude/viz/backend /opt/brain-viz/`
  2. `cd /opt/brain-viz && docker compose build --no-cache brain-viz`
  3. `docker compose up -d brain-viz` (NOT `up -d` alone — avoids touching nginx/certbot/MCP containers)
  4. `sleep 3 && curl -s https://goodhealth.lv/api/health` to verify
- **Prerequisites:** Files must be committed first (`git push`) so GH webhook triggers brain sync. Deploy-viz.yml will still try-and-fail on SSH but that doesn't matter.
- **Verification:** `curl https://goodhealth.lv/api/health` returns `{"ok": true}`
- **Reward:** Dashboard updates live without waiting for CI retries
- **Gotchas:**
  - `cp` to `/opt/brain-viz/..` fails with permission denied — cp to `/opt/brain-viz/` directly with subdir targets
  - The GH Actions run will show as "failed" (SSH error) — ignore it, the local deploy is what matters
