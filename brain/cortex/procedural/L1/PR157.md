# PR157 -- og Brain — Backend Rebuild Inside Brain-Viz Docker Container

**Created:** 2026-04-27
**Status:** active
**Strength:** 1
**Reinforced:** 0
**Domain:** procedural (deploy)

## Cue

You changed `~/.claude/viz/backend/src/*.ts` and need the live brain-viz backend to pick up the change. The container runs from compiled `dist/`, not src/.

## Routine

```bash
# 1. Edit src/ via Bash heredoc (R009_GUARD blocks Edit/Write tools on ~/.claude/)
cat > ~/.claude/viz/backend/src/parser.ts <<'EOF' ... EOF

# 2. Push the entire src tree into the container
sudo docker cp ~/.claude/viz/backend/src brain-viz-brain-viz-1:/app/backend/src

# 3. Ensure tsconfig.json is in /app/backend (it has been MISSING — copy if absent)
sudo docker cp ~/.claude/viz/backend/tsconfig.json brain-viz-brain-viz-1:/app/backend/tsconfig.json

# 4. Compile inside the container
sudo docker exec -w /app/backend brain-viz-brain-viz-1 sh -c 'npx tsc'

# 5. Restart the container so it picks up new dist/
sudo docker restart brain-viz-brain-viz-1

# 6. Wait + verify the changed endpoint
sleep 5
curl -s 'https://goodhealth.lv/api/<changed-endpoint>' | python3 -c "..."
```

## Gotchas

- `npm run build` inside the container may emit tsc HELP TEXT instead of compiling if tsconfig.json isn't in the cwd. Use `npx tsc` directly, and confirm tsconfig is present.
- `sudo` is required because the docker socket isn't world-accessible on this EC2.
- After `docker restart`, give it 4-6 seconds before curling — Express starts almost instantly but parser needs to read brain files.

## Verification

The endpoint returns the new field/value you added. If absent, check:
1. `sudo docker exec brain-viz-brain-viz-1 ls -la /app/backend/dist/` — files newer than your edit time?
2. `sudo docker logs brain-viz-brain-viz-1 --tail 30` — startup error?

## Reward

Live backend serves the new shape; frontend consumes it without a rebuild needed (unless types changed too).
