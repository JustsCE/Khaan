# og Brain — QA Protocol: Cross-checking Doc vs Calculations — INCLUDING Images

When asked to verify that a document's numbers match underlying calculations, do NOT stop at table-text verification. Embedded images (bar charts, boxplots, forest plots) carry their own statistical annotations and must be independently verified.

## Procedure

1. Extract embedded images from the docx using zipfile and save to /tmp/imgs/.
2. Use multimodal Read tool on each PNG to visually parse the annotated statistics (chi-squared / U / r_rb / p values embedded in chart titles).
3. Compare against computed values from the JSON results.
4. Report per-image: chart label, annotated value, computed value, match/mismatch.

## Why

Images are often generated in a separate pipeline from the tables — they can drift. Also, percentages shown on bar charts are derived from the underlying counts but rendered as images, so they need verification that the labels reflect the data correctly.

Justs (2026-04-23, after initial cross-check): "DID YOU CHECK THE IMAGES? DID YOU VERIFY THE NUMBERS THERE MATCH?" — the table text was verified, the images were not. Trust but verify the images too.
