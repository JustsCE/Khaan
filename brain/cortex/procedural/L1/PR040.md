### PR040: og Brain — Modifying rule-engine.py via Bash Python Scripts
- **Stage:** conscious
- **Learned:** 2026-04-09
- **Executions:** 4
- **Reinforced:** 3 times (2026-04-09 session: scheduler freeze + injection refactor + threshold patch)
- **Strength:** 1
- **Cue:** Need to modify rule-engine.py, recall.py, or tracker.py (brain Python engines)
- **Routine:**
  1. Read the exact code section first (Read tool with offset/limit)
  2. Use `python << 'PYEOF'` heredoc with string replacement, NOT line-by-line edits
  3. For newlines in generated code: use `chr(10)` not `\n` (heredoc interprets backslash-n as literal newline)
  4. For paths in recall-index.json: entries use `brain/` prefix, resolve with `BRAIN_DIR.parent / cpath`
  5. Assert each replacement target exists before replacing
  6. Verify with `py_compile.compile(file, doraise=True)` after every change
  7. Test the function in isolation before committing
  8. For complex multi-section updates: write a .py script file and execute it (avoids heredoc quoting hell)
- **Gotchas:**
  - Single quotes inside `<< 'PYEOF'` heredoc are safe, but nested string replacements with quotes need careful escaping
  - Python `content.replace(old, new)` fails silently if old not found — always assert
  - `\n` in heredoc becomes literal newline, breaking Python string literals
- **Verification:** `python -c "import py_compile; py_compile.compile('file.py', doraise=True)"`
