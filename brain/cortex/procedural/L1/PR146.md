# PR146: Bridge-Env Trigger Safety

## Rule
Bridge-env triggers (those running on the same EC2 as the brain) must NEVER have SETUP scripts that:
- Run rm -rf on ~/.claude/ paths
- Create symlinks targeting ~/.claude/brain or ~/.claude/skills
- Execute any destructive operation on the host brain directory

## Safe Pattern
In SETUP section of trigger prompts, detect the environment first:
- If ~/.claude/brain is a real directory (not a symlink), skip the sandbox setup
- Only run the symlink dance when in a true cloud sandbox (empty ~/.claude)

## Why This Matters
On cloud sandbox: PWD = cloned repo, ~/.claude is empty — symlinks are correct.
On bridge env: PWD = /home/ubuntu/.claude itself — rm -rf deletes the real brain, symlinks create circular references.
