# og Brain — PR114: python-docx Workflow for Academic Tables

## Cue
Need precise table control in DOCX: merged row-header cells, custom borders (top/bottom rule only), centered headers with bold font, Latvian characters.

## Routine
1. Install: `pip install --break-system-packages python-docx` (Linux bypasses PEP 668)
2. Import pattern:
   ```python
   from docx import Document
   from docx.shared import Pt, Cm
   from docx.enum.table import WD_ALIGN_VERTICAL
   from docx.enum.text import WD_ALIGN_PARAGRAPH, WD_BREAK
   from docx.oxml.ns import qn
   from docx.oxml import OxmlElement
   ```
3. Set document-level font: `doc.styles["Normal"].font.name = "Times New Roman"; .size = Pt(12)`
4. Custom cell borders via raw XML — `python-docx` has no high-level API:
   ```python
   def set_cell_border(cell, **kwargs):
       tcPr = cell._tc.get_or_add_tcPr()
       tcBorders = tcPr.find(qn("w:tcBorders")) or OxmlElement("w:tcBorders")
       # set top/bottom/left/right sub-elements with {"val":"single","sz":"12"}
   ```
5. Merge row header across multiple data rows: `row_a.cells[0].merge(row_b.cells[0])`
6. Page break: `paragraph.add_run().add_break(WD_BREAK.PAGE)` (NOT `add_break(6)` — use enum)
7. Condensed cell margins (critical for "compact row height"):
   ```python
   tcMar = OxmlElement("w:tcMar")
   for edge, val in [("top",20),("bottom",20),("left",80),("right",80)]:
       e = OxmlElement(f"w:{edge}")
       e.set(qn("w:w"), str(val)); e.set(qn("w:type"), "dxa")
       tcMar.append(e)
   ```

## Gotchas
- Latvian characters work via UTF-8 literal in strings (e.g. `"Ķermeņa"`), but complex f-strings with embedded Unicode fail with `SyntaxError: unterminated string literal`. Workaround: use `\u0137` escape sequences or write the script to disk as a file (not inline bash heredoc).
- `boxplot(labels=...)` is deprecated in matplotlib 3.9; use `tick_labels=...` instead.
- `chi2_contingency` defaults to `correction=True` for 2×2 tables (Yates' correction) — this clamps small differences to 0 falsely. Always pass `correction=False` for modern practice.
- Cell text must be set AFTER clearing: `cell.text = ""; p = cell.paragraphs[0]; p.add_run(text)`.

## Verification
- Open DOCX in Word/LibreOffice → tables render with merged cells + tight padding
- `pandoc output.docx -o preview.pdf` converts correctly as final sanity check
- Size: 50–100 KB for text-only docs; 500 KB–1.5 MB with charts (200dpi PNG embedded)

## Reward
Professional academic-quality Word tables matching journal standards. Directly pasteable into a larger Word document.
