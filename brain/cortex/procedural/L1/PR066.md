# PR066 -- og Brain — Vericity Deploy Pipeline
- Created: 2026-04-13
- Source: brain-retro session 2026-04-13
- Strength: 3
- Reinforced: 0
- Tags: vericity, deployment, docker, azure, acr
- Domain: vericity

## Cue
Deploying code changes to vericity.ai

## Routine
1. git add + commit + push to GitHub
2. Build: az acr build --registry vericityacr --image vericity:TAG --file Dockerfile . --no-logs
   - Uses two-tier: Dockerfile.base (heavy deps) + Dockerfile (app code)
   - Base rebuild only when requirements.txt/package.json change
   - App build: ~110s on ACR (vs 235s before split)
   - Context upload from Windows: ~13 min (main bottleneck)
3. Deploy: az containerapp update --name vericity-app --resource-group vericity-rg --image vericityacr.azurecr.io/vericity:TAG
   - CRITICAL: wait for build to appear in az acr task list-runs BEFORE deploying
   - Deploy before build completes = MANIFEST_UNKNOWN error
4. Helper script: bash scripts/deploy.sh TAG (or --base for base rebuild)

## Gotchas
- az acr build log streaming CRASHES on Windows with Unicode error. Always use --no-logs
- Context upload takes 13+ min from Windows. Build itself is only ~2 min
- If deploy fails with MANIFEST_UNKNOWN, the build hasn't finished yet. Check az acr task list-runs
- Multiple concurrent az acr build commands can queue and interfere

## Verification
- az acr task list-runs --registry vericityacr --top 1 (check Succeeded + correct tag)
- az containerapp show --name vericity-app -o json | check container image tag
