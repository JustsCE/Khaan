# PR167: Cross-doc consistency verification before implementation

**Cue:** finishing a multi-doc design pass; about to declare implementation-ready.

**Routine:**
1. List every numeric constant and named binary referenced across the docs.
2. For each constant, grep all docs. Differences = blocker.
3. For each named binary, grep all docs. Same name, same threshold.
4. List every artifact path. Standardize to one shape.
5. List every cross-engine reference. Verify both ends agree on schema.
6. Per engine, verify dispatcher / hook list matches what other engines expect.
7. Per engine, verify anti-lockout coverage matches the system pattern.
8. Generate a checklist with severity tags. Trace each finding to a specific file/line.
9. Verify the checklist against actual files, not memory.

**Prerequisites:** all docs in one tree; no TBD lines that contradict implemented claims.

**Verification:** every constant has the same value across docs; every binary has one canonical name; cross-engine paths consistent across producer + consumer; checklist claims grep-verifiable.

**Reward:** implementation can begin without runtime divergence.

**Source incident:** Brain v2 reconciliation 2026-05-05. Initial checklist found 4 blockers + 9 major gaps; verification pass surfaced 3 more (binary naming, undercount, anti-lockout placement). Verification is not optional.
