# Deploying future-week error suppression in the Forecast Lambda

To suppress misleading error values for future weeks in the Forecast Lambda, gate the computation on the current ISO week derived from `datetime.date.today()`. Any week beyond the current cutoff should be silenced rather than displayed, preventing the report from surfacing incomplete or nonsensical figures. This logic belongs in the Lambda's core calculation path so that it applies uniformly each time the function executes.

When deploying iterative fixes to the Picanova AWS Lambda, follow a clean sequence: adjust the logic locally, verify the ISO-week boundary behaves correctly for edge cases (year boundaries, partial weeks), then push to `main` and deploy. Expect multiple refinement passes — the Summary 2 columns, YoY% formatting, and error thresholds each required their own iteration in practice. Treat each pass as a discrete deploy-and-verify cycle rather than batching all changes, since the Lambda's output is user-facing and regressions surface immediately in the report.

The underlying principle: any time-dependent suppression should derive its cutoff from a live clock call (`datetime.date.today()`), never from a hardcoded or cached value, so that each invocation self-corrects without redeployment.
