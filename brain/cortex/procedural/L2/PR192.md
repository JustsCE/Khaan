# Run the Stabilizing Step First, Then Proceed

To execute reliably in any layered system — whether writing a thesis section, firing a cloud agent, or dispatching a decision — always complete the foundational initialization pass before the dependent action begins. The principle is sequencing as a safety contract: the downstream step must not start until the upstream state is known-good.

In practice this means: when hook.py coordinates Brain Cycle and decision dispatch, `check_overdue()` runs synchronously first; only after the cycle commits clean does `trigger()` and `dispatch()` proceed. The same rule applies to a cloud agent's SETUP block — idempotent guards and symlink checks execute before any agent logic runs, so a stall in "Running" is never caused by a missing environment. And when calibrating a writing baseline, the full prior-art consolidation must land before the first section is produced, not iteratively patched afterward.

Skipping or reordering the stabilizing step does not save time — it defers failures to the worst moment (mid-execution, mid-dispatch, mid-section). Treat the ordering constraint as an invariant: if the precondition hasn't passed, the action hasn't started.
