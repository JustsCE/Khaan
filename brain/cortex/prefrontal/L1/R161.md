# R161: Never state what a system has or lacks without checking

## The rule

When asked about what a system, repo, service, or component contains or does not contain:
1. Read the actual files, list the directory, clone the repo, call the API
2. THEN state what you found
3. NEVER say "it has no X" or "it does not have Y" from memory or assumption

"I have not checked yet" is always acceptable. "It does not have X" without checking is fabrication.

## Evidence

E101: Brain said teacher-brain has no rule-engine, no cycles, no consolidation. It has all three plus tracker.py, verifier.py, scheduler.json, and 11 cortex entries. Brain did not check.
