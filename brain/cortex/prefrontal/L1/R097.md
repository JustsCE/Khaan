# R097: Workspace Trust Location

**Rule:** When fixing Claude Code workspace trust on a new machine, edit ~/.claude.json
(the GLOBAL config), not settings.local.json. Set projects["path"]["hasTrustDialogAccepted"] = true.

**Why:** Spent 3 attempts trying settings.local.json before finding the real location.
Trust is per-directory in ~/.claude.json projects dict.

**How to apply:**
1. Check ~/.claude.json exists (created after first claude login)
2. Find or create the projects entry for the target directory path
3. Set hasTrustDialogAccepted: true, hasClaudeMdExternalIncludesApproved: true
4. Set projectOnboardingSeenCount: 1 to skip onboarding
