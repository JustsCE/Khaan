When a system is behaving unexpectedly, read the implementation file before forming any hypothesis. Do not say 'the problem is X' without citing the specific line number where X occurs. If tools are blocked, ask the user to paste the relevant 20 lines. Theorizing before reading is a fabrication, not a diagnosis.
