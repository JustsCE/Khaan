# R108: Anthropic Routines — 1 Hour Minimum Cron

**Rule:** When creating Anthropic Routines (RemoteTrigger), never set cron intervals shorter than 1 hour. The API rejects */5, */10, */15, */20, */30 cron expressions. Minimum is hourly.

**Evidence:** Attempted */5 * * * * for "Hey Brain!" WhatsApp responder — API returned "cron expression fires more frequently than once per hour; minimum interval is 1 hour."

**Implication:** Real-time or near-real-time response systems cannot use Routines. Use direct API calls (Anthropic SDK with Haiku) or webhook-based approaches instead.
