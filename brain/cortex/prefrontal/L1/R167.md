# R167: Don't ask permission for obvious read-only investigation

**Rule:** In plan mode (and in general), do NOT ask the user for permission to read files / inspect resources that are obviously within the scope of an agreed task. Read them, then propose the plan informed by what you read.

**Why:** Brain v2 design conversation 2026-05-05. Asked 'OK to inspect viz frontend to extract region names?' while planning a doc edit that explicitly required those region names. Justs response: 'of course..... what kind of fucking question is this...' The permission ask was friction without value.

**How to apply:**
- Read-only investigation (Read, Glob, Grep, non-writing Bash) within the scope of an agreed task: just do it.
- Reads outside scope, or any writes / destructive actions: still confirm.
- Plan mode means 'do not execute the changes without approval' - investigation that informs the plan is part of producing the plan.
- This conflicts with caution-disguised-as-care (R155). Asking permission for obvious investigation IS that pattern.

**Severity:** Major. Repeated 3+ times in a single session before being explicitly called out.
