# R114: Recall Gate Enforcement Rules

## Rule
When recall gate activates (active-recall.json has required entries):
1. Confidence tiers determine blocking: high (>=5.0) = immediate, medium (3.5-5.0) = progressive, low (<3.5) = pass
2. Safe tools ALWAYS pass: Read, Glob, Grep, WebSearch, WebFetch, Skill, Agent, AskUserQuestion
3. 2h TTL auto-expire prevents cross-session stale state
4. deny_count tracks progressive enforcement for medium tier
