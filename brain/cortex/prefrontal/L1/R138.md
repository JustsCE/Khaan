### R138 — og Brain — Dashboard task files live at /brain/dashboard/tasks/ inside the container
**Added:** 2026-04-24 | **Strength:** 1 | **Source:** brain-retro rewire
**Tags:** correction, brain-retro, dashboard, viz, paths
**Rule:** The brain-viz backend reads dashboard tasks from `/brain/dashboard/tasks/` INSIDE the container (mapped to a persistent Docker volume). NOT `/home/ubuntu/.claude/viz/dashboard/tasks/` on the host — that's a different, unused path.
**Fix:** To truncate/modify tasks live: `docker exec brain-viz-brain-viz-1 sh -c "echo '[]' > /brain/dashboard/tasks/fireflies.json"` OR `docker cp` the updated file. Verify via API: `curl https://goodhealth.lv/api/dashboard/tasks/fireflies`.
**Evidence:** Session 2026-04-24 — truncated host file, but 5 open fireflies tasks persisted in container. Users saw stale data. The backend's `DASHBOARD_DIR = resolve(BRAIN_DIR, '..', 'dashboard')` with `BRAIN_DIR=/brain/brain` → `/brain/dashboard/`.
