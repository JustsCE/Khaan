# R141: Brain/viz runs FROM the EC2 — deploy is in-place, not over SSH

**Rule:** When asked to deploy viz to goodhealth.lv, NEVER suggest `bash viz/deploy.sh` or anything requiring `PEM_KEY`. The brain process already runs ON the EC2 instance (ip-172-31-29-157, goodhealth.lv). Deploy means: copy/rsync files into `/opt/brain-viz/` locally and run `docker compose build --no-cache && docker compose up -d` in place. SSH/SCP/PEM are for laptop→EC2 pushes, which is not our topology.

**Why:** `viz/deploy.sh` is authored for a developer pushing from their laptop; its first action is `if [ ! -f "$PEM_KEY" ]; then exit 1`. Suggesting it in brain's own context adds a non-existent blocker and contradicts `project_ec2_migration.md` (brain lives on the EC2).

**How to apply:**
- Source of truth: `/home/ubuntu/.claude/viz/` (git-tracked)
- Deploy target: `/opt/brain-viz/` (what docker-compose uses)
- Commands in order:
  1. `rsync -a --delete /home/ubuntu/.claude/viz/{Dockerfile,docker-compose.yml,nginx.conf,frontend,backend} /opt/brain-viz/`
  2. `cd /opt/brain-viz && docker compose build --no-cache && docker compose up -d`
  3. Verify: `docker compose ps` + `curl -sI https://goodhealth.lv`
- Keep `viz/deploy.sh` as-is for future laptop-originated deploys; just don't invoke it from here.
