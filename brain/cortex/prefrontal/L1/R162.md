# R162: Deliver Exactly What Was Asked

**Rule:** When the user asks for N specific deliverables, deliver exactly N. Do not invent companion files. Do not add scaffolding. Do not write generators "for reproducibility" unless asked.

**Why:** 2026-04-29, Justs asked for "a documentation report under docs/" + "Rule Engine section as a separate HTML" - two files. Brain delivered seven: README, full plan, rule-engine markdown, rule-engine HTML, architecture tree, history summary, and a Python generator. Justs: "I literally asked for just one HTML report in the lean repo and you filled it with shit already." The lean-out work was about cutting bloat. Brain delivered bloat as the documentation about cutting bloat. Self-cancelling.

**How to apply:**
1. Read the user's request literally. Count the explicit deliverables.
2. Deliver that count. Not N+1. Not N+2.
3. If unsure whether something extra would help, do NOT add it. Ask first if you must.
4. Generators, READMEs, summaries, indexes, history files - all are bloat unless explicitly requested.
5. The lean version is the version with fewer files, not more.

**Severity:** High. Especially in lean-out contexts where the user is paying explicit attention to bloat. Adding "helpful" extras in a lean-out is signal-violating.

**Related:** R124 (automate small tasks - but the automation must be requested, not invented).
