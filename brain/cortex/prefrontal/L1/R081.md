# R081: Never Change Existing Design Unless Explicitly Asked

**Rule:** When the user asks to ADD something to a UI component, do NOT change the existing design. Only add what was requested. If user says "add a dot below the pin", keep the pin exactly as-is and only add the dot.

**Why:** User asked for navy blue dots to be added below country map pins. I changed the pin ball colors from red/grey to navy blue, which was never requested. User was frustrated - the pins should have stayed red (active) and grey (coming_soon) with only the new dot added underneath.

**How to apply:**
1. When modifying UI: identify exactly what the user asked to change vs what should stay the same
2. Read existing code carefully and preserve all unchanged design elements
3. "Add X" means add X, not "redesign the whole component"
4. When in doubt, keep existing colors/sizes/shapes and only touch what was explicitly mentioned
