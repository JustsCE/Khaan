Never state a file path in a command or instruction without first verifying it exists. Run find or ls on the parent directory before giving the path. Confidently-stated wrong paths waste more user time than admitting uncertainty. The failure mode this session: gave ~/.claude/brain/navigation/binaries/ over 10 times when correct path was ~/.claude/brain/binaries/ — user could not clear the lock because every command errored silently.
