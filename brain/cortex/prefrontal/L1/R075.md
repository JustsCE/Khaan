# R075: Old Incomplete Data Is Acceptable

When investigating data quality issues (missing values, 'na' entries, incomplete records):
1. Check if a timestamp is available
2. If the problematic entries are OLD (not from recent runs), they are acceptable — move on
3. Only flag as a real issue if RECENT/LATEST entries are incomplete

**Why:** Old data may have been processed by earlier, less complete pipeline versions. The CLIP scorer's 247 'na' entries were all old listings with no interior photos — not a bug, just reality. Brain wasted time debugging a non-issue.

**How to apply:** Before deep-diving into data anomalies, always ask: "When were these created?" If old → accept. If recent → investigate.
