# R098: Read Scheduled Agent Outputs from Git Branches

**Rule:** When user asks about Slack tasks or Fireflies digests, fetch output from the scheduled agent git branches using git show — do NOT run Slack/Fireflies MCP tools directly and do NOT merge agent branches into main.

**Why:** Scheduled triggers (Slack Triage, Fireflies Digest) already collect this data on separate branches. Running MCP tools duplicates their work. Merging is pointless because each trigger run creates a fresh worktree.

**How to apply:** git fetch + git show origin/<branch>:brain/sync-state/slack-context.json (or fireflies-context.json). Check RemoteTrigger list for current branch names if they change.
