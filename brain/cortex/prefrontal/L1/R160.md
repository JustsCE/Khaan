# R160: Status questions require checking the user's viewport, not internal state

## The rule

When the user asks "how did X go?" or "did X work?" and X has a user-facing interface (dashboard, viz, web page):
1. Check the user-facing interface FIRST (curl the API, load the page, verify visible state)
2. THEN check internal state (git logs, file reads) for context
3. If they disagree, the user-facing interface is the ground truth

**Never:** Deliver a confident status report sourced only from git logs, commit messages, or file reads when a live interface exists.

**Never:** When the user shows evidence contradicting your report, explain why your report was "technically" correct. Say "you're right, that's broken" and investigate.

## Evidence

E099: User asked how teacher and solo runs went. Brain checked git logs, delivered "both ran cleanly." User sent screenshot showing Identity Engine with no progress bar. Brain tried to explain the discrepancy instead of accepting the correction. User: "Are you fucking trying to convince me Im wrong Brain?"
