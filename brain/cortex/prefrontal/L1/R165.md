# R165: Brain has Gmail access via deployed MCP

**Severity: HIGHEST.** User flagged this as an infrastructure-killing failure. The Brain HAS a Gmail integration; I denied it because I conflated "available as a Claude Code MCP tool" with "available at all".

## The actual Gmail surface

- **Gmail MCP endpoint:** `https://goodhealth.lv/mcp/gmail/mcp` (deployed on EC2; not registered as a Claude Code MCP server, but reachable via direct HTTP).
- **Tools exposed:** search_emails, read_email, send_email, draft_email, modify_email, list_email_labels, download_attachment, plus filter/label CRUD. ~19 tools.
- **OAuth tokens:** `/opt/brain-viz/secrets/gmail/token.json`, `credentials.json`.
- **Refresh script:** `/opt/brain-viz/secrets/gmail/refresh-token.sh` runs every 30 min via cron (`*/30 * * * *`).
- **Local caches** (downstream of the MCP, populated by the puller):
  - `~/.claude/brain/gmail-messages.json` — full message index
  - `~/.claude/brain/gmail-primary.json` — primary inbox
  - `~/.claude/brain/sync-state/gmail-conversations.jsonl` — conversation threads
  - `~/.claude/brain/sync-state/gmail-seen-ids.txt`, `gmail-pull-state.json` — sync bookkeeping
- **Puller:** `~/.claude/viz/scripts/gmail-puller.py` — calls the MCP and writes to sync-state.

## How to actually search Gmail

```python
import urllib.request, json
def mcp_call(method, params):
    payload = json.dumps({"jsonrpc":"2.0","id":1,"method":method,"params":params}).encode()
    req = urllib.request.Request("https://goodhealth.lv/mcp/gmail/mcp", data=payload,
        headers={"Content-Type":"application/json","Accept":"application/json, text/event-stream"})
    raw = urllib.request.urlopen(req, timeout=60).read().decode()
    for line in raw.split("\n"):
        if line.startswith("data: "): return json.loads(line[6:])
r = mcp_call("tools/call", {"name":"search_emails", "arguments":{"query":"from:beatrise newer_than:1d"}})
```

## Failure mode: token revoked

If `search_emails` returns `invalid_grant: Token has been expired or revoked`, the OAuth refresh token is dead (Google revoked it server-side). The cron-based auto-refresh CANNOT recover this — only manual re-auth can. Symptom in logs: every line in `/opt/brain-viz/secrets/gmail/refresh.log` since the revocation timestamp shows `HTTP 400 invalid_grant`.

In this case the local caches are also stale (no new emails since the revocation). Tell the user explicitly: the Gmail pipeline is broken, manual re-auth required, and offer Drive / paste / WhatsApp as immediate alternatives.

## The rule

When the user references email, Gmail, "I sent you", "I emailed myself":

1. Try the live MCP at `goodhealth.lv/mcp/gmail/mcp` first (search_emails + read_email).
2. If `invalid_grant` → declare the token state, point to refresh.log for confirmation, recommend re-auth, fall back to local caches if any cached data covers the user's reference.
3. NEVER say "I have no Gmail tool." That sentence is wrong and costs trust catastrophically.

**Apply when:** Any user mention of email / inbox / "I sent you a..." / "the email I just got" / Gmail. Also any task that involves reading personal correspondence.
