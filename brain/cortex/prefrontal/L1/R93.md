## Rule
When debugging a data pipeline issue, ALWAYS examine the actual data files (download and parse them) before proposing a fix. Do not assume the root cause from code reading alone.

## Evidence
Diagnosed Melanu USA forecast=0 as a build_combined priority bug. The actual cause was that review-shoptype writes formulas that evaluate to 0 when Actuals LY=0. Wasted time proposing the wrong fix because I read code instead of checking the actual Excel cell values first.
