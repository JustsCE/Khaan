# R112: Never || true on Enforcement Hooks

**Rule:** NEVER append `|| true` to hook commands that use exit codes for enforcement. The `|| true` operator converts ALL non-zero exit codes to 0, silently disabling the entire rule engine without any visible error.

**Instead use:** `; e=$?; [ $e -eq 2 ] && exit 2 || exit 0` — this preserves intentional blocks (exit 2) while converting crashes to pass (exit 0).

**Evidence:** 2026-04-18 — `|| true` was added to make hooks cross-platform safe, but it silently killed all rule enforcement. PreToolUse blocks, recall gates, and maintenance mode all stopped working. The system appeared functional but was completely unenforced.
