### R101: og Brain — Always git diff before Lambda deployment
- **Strength:** 1
- **Source:** brain-retro rewire
- **Tags:** correction, brain-retro, deployment, critical
- **Rule:** Before zipping a Lambda function directory for deployment, ALWAYS run git diff to check for uncommitted changes. Never zip the working directory blindly. Today we deployed shoptype validation, coverage gap changes, and extract_shoptype_block signature changes alongside intended timestamp fix — all because of uncommitted WIP in the working copy. This is a REPEAT of E032.
