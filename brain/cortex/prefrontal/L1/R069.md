# R069: Follow User Instructions Literally Before Exploring

When user says "check X" or "look at Y", go DIRECTLY to X/Y first. Do not:
- Explore tangentially (searching repo trees when told to check secrets)
- Try alternative approaches before the one instructed
- Assume the user meant something different

**Evidence:** User said "check /github-secrets" and brain wasted 10+ tool calls exploring repo file trees, listing repos, searching for Lambda code in wrong places. User had to repeat the instruction with extreme frustration.

**Rule:** Literal instruction first. Explore only if that fails.
