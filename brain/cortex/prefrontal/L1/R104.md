### R104: og Brain — Use date-based split for actuals/forecast boundary, not week-based
- **Strength:** 1
- **Source:** brain-retro rewire
- **Tags:** correction, brain-retro, forecast, report
- **Rule:** When combining actuals and forecast data in reports, split by DATE (actuals through last_actual_date, forecast after), never by ISO week number. Week-based split causes either a gap (> cw excludes partial week forecast) or a spike (>= cw double-counts actuals that exist in both sources for the same week).
- **Evidence:** 2026-04-17 — CW16 showed dip with > cw, spike with >= cw. Fixed by using fcst.date > last_actual_date.
