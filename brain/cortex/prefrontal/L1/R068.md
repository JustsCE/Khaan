# R068: Brain Engine Edits via Bash Python Scripts Only

When modifying rule-engine.py, recall.py, tracker.py, or any brain Python engine:

1. Read the exact code section first (Read tool with offset/limit)
2. Use python heredoc with string replacement, NOT line-by-line edits
3. For newlines in generated code: use chr(10) not backslash-n (heredoc interprets it)
4. Assert each replacement target exists before replacing
5. Verify with py_compile after every change
6. For complex multi-section updates: write a .py script file and execute it
