# R166: Verify counts and claims before stating them in checklists

**Rule:** When writing a review / checklist / audit doc, every numeric claim ('8 docs total', '15 binaries', '4 blockers') must be verified by direct grep / file count BEFORE the doc ships, not after a user asks for verification.

**Why:** Brain v2 implementation checklist (2026-05-05) stated '8 v2 docs total' (actually 9 HTML files including v1 legacy + checklist itself) and '15 binaries total' (actually 27 system-wide). Justs asked for verification; the verification pass found the miscounts plus three additional issues. The miscounts cost trust and a verification round.

**How to apply:**
- Before writing 'N items total' anywhere, run the grep / ls / wc that produces N.
- Quote the exact source file:line for every numeric claim in a review doc.
- 'About N' is not acceptable in an implementation checklist; either count exactly or don't claim a count.

**Severity:** Major. Implementation checklists with wrong counts cause downstream design errors.
