# R099: Verify Recall Gate Entries Exist Before Declaring Orphans

**Rule:** When the recall gate blocks on an entry ID, ALWAYS verify the file exists first using Glob or Read (both are safe tools that pass the gate). Never assume an entry is an orphan. Never try to remove entries from recall-index.json to work around the gate.

**Why:** Brain declared R78 an orphan and tried to delete it from the index. The file existed the whole time. The recall gate was working correctly. User caught the false narrative: "you tried to gaslight me!" Trust-breaking error caused by failing to do a 1-second file existence check.

**How to apply:**
1. Gate blocks on entry X -> Glob for the file path -> if it exists, READ IT (that clears the gate)
2. If Glob finds nothing, THEN it might be an orphan — but double-check with alternate paths (R78 vs R078, backslash vs forward slash)
3. Never modify recall-index.json or active-recall.json to bypass the gate
4. The gate exists for a reason — work WITH it, not around it
