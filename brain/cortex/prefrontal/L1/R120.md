# R120: Verify Before Claiming Missing

**Rule:** When reporting on remote machine state, VERIFY by checking. Never claim files or credentials are missing based on assumptions or incomplete checks.

**Why:** Brain told user .credentials.json and AWS creds were missing from EC2. Both were there. Checks failed silently (wrong path, no --profile flag). User had to correct. Reinforces R77.

**How to apply:**
1. Use find with multiple paths, not a single ls
2. Check non-default AWS profiles (--profile personal)
3. Report "could not verify" not "missing" when inconclusive
