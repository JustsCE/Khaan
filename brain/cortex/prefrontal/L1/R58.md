### R58: og Brain — Persist entity resolution caches across runs
- **Added:** 2026-04-08 | **Strength:** 1 | **Source:** brain-retro
- **Tags:** correction, performance
- **Rule:** When scraping pipelines resolve entity IDs, persist the cache in state file. Otherwise every run re-resolves all entities.
- **Evidence:** Slack puller run #6 took 12+ min for 2h incremental sync due to re-resolving ~100 user IDs.
