# R129: Chi² Validity — Three Checks, No Yates

**Rule:** When running `scipy.stats.chi2_contingency()` for validity-gated analysis (e.g. per-Robson stratification), ALWAYS:

1. Require `ct.shape[0] ≥ 2` AND `ct.shape[1] ≥ 2` (at least 2 categories × 2 groups)
2. Require `ct.values.min() ≥ 5` (Cochran's rule)
3. Pass `correction=False` explicitly to disable Yates' continuity correction

**Why:** Justs caught TWO bugs in one review:
- *Bug A* — I marked 1×2 tables as "valid" because min_cell check passed. Result: fake χ²=0.00, p=1.000 for 5 Robson-1 variables (Holestāze, Preeklampsija, Dzemdību tips, Ķeizargrieziens kategorija all had zero Adipozi variance).
- *Bug B* — Even after fixing shape, Attiecību statuss in Robson 1 still showed χ²=0.00 because Yates' correction clamped all |O-E|=0.3 to 0.

**How to apply:**
1. Code-level: `compute_per_robson_chi()` in build_continuation.py and build_results_docx.py must have all three guards
2. Code review: whenever chi² appears in any new analysis, grep for `correction=False` presence
3. Result review: any row showing "χ²=0.00, p=1.000 exactly" is a red flag — investigate shape + correction, not data

**Severity:** High — false-zero results are methodologically unacceptable in a published paper. Single wrong result undermines reviewer trust in the entire analysis.

**Related:** R128 (verify artifact before reporting — applies here: don't trust log "valid=36" without checking what's in the results table).
