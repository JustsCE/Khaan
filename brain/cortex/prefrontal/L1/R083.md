# R083: Check Data Before Diagnosing

**Rule:** When a field shows unexpected values, check the ACTUAL data in source files before assuming code is broken.

**Evidence:** Spent multiple iterations fixing CLIP scorer code when the real issues were: (1) img_keys not in img_rank.csv, (2) images not downloaded yet. Code was fine - data had gaps.
