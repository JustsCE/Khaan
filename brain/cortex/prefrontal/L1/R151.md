### R151: og Brain — iOS/Cloud Sessions Need Explicit Allow Entries
- **Strength:** 1
- **Source:** brain-retro 2026-04-26 (cp prompt incident, parallels agent-mode incident earlier same day)
- **Date:** 2026-04-26
- **Tags:** correction, brain-retro, ios, permissions, recurring-pattern
- **Rule:** When the user is connected via the iOS Claude.ai app (or any cloud-deployed session including scheduled agents), commands NOT explicitly in `permissions.allow` will trigger a permission prompt — even if `defaultMode: bypassPermissions` is set on the EC2 user config file (settings.json). The bypass mode applies only to local EC2-initiated sessions, not the cloud overlay.
- **Why:** Same root cause hit twice in one morning. First: scheduled triage agents stuck on `rm -f ~/.claude/brain/.agent-mode` (fixed by adding to allow list via bypass token). Second: iOS user prompted to allow `cp ~/.claude/viz/whatsapp-bridge/src/index.ts /opt/brain-viz/...` during wa-bridge deploy — same pattern, different command.
- **How to apply:**
  1. **Anticipate, dont react.** When proposing a Bash command involving `cp`, `mv`, `rm`, `mkdir`, `chmod`, `chown`, `rsync`, `docker compose ...` etc, assume the user on iOS WILL get prompted unless the pattern is in `permissions.allow`.
  2. **Batch the additions.** Each iOS prompt = one user friction event. When you are about to issue a class of commands, propose adding the whole class to allow at once via a bypass token, not one command at a time.
  3. **Common patterns to allow** (consider proactively):
     - `Bash(cp:*)`, `Bash(mv:*)`, `Bash(mkdir:*)`, `Bash(rsync:*)`
     - Specific safe deletions: `Bash(rm -f ~/.claude/brain/.agent-mode:*)` (already added)
     - Container ops with explicit service: `Bash(docker compose build wa-bridge:*)`, `Bash(docker compose up -d wa-bridge:*)` (R146-compatible)
  4. **Do NOT blanket-allow `Bash(rm:*)`** — too broad, defeats the safety purpose.
  5. The fix path is the same as agent-mode: user issues bypass token via Lambda → brain writes to settings.json → log INFRA_BYPASS_USED.
- **Mechanical:** No (judgment-based; encoded as prefrontal rule for retrieval before iOS-prompting commands).
- **Cross-references:** R148 (external-script loophole — same auth model), R150 (no scope creep — dont add commands that arent actually used), the agent-mode allow patterns added 2026-04-26 morning.
