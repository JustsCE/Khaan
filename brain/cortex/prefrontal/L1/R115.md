# R115: goodhealth.lv Host vs Docker Nginx

**Rule:** When modifying nginx proxy rules on goodhealth.lv, edit `/etc/nginx/sites-enabled/brain-viz` (HOST nginx 1.24.0) — NOT the Docker nginx container config. The Docker nginx cannot bind port 443 because the host nginx already has it. Docker nginx is effectively unused for external traffic.

**Evidence:** Spent multiple iterations fixing Docker nginx config (rewrite rules, dynamic resolution) before discovering the "x-powered-by: Express" and "nginx/1.24.0 (Ubuntu)" headers proved requests never reached Docker nginx. The host nginx catch-all `location /` forwards everything to Express on port 3001.

**How to apply:** Any MCP proxy setup, any goodhealth.lv routing change → edit host nginx, `sudo nginx -t && sudo nginx -s reload`.
