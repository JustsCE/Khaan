### R051: og Brain — EC2 Recovery — Stop/Start, Not Reboot
- **Strength:** 1
- **Rule:** When goodhealth.lv is down and SSH times out: always STOP then START the EC2 instance (not reboot). Reboot does not fix networking. After start, wait 60s for SSH, then docker compose up -d. If nginx fails, restart nginx separately. systemd brain-viz.service handles auto-start on boot.
- **Source:** PR028 (strength 3), E029
