# R109: EC2 Deploy Pipeline — Manual Deploy When Actions Fail

**Rule:** When pushing viz/ changes, do NOT assume deploy-viz.yml GitHub Action will succeed. It has been failing (SSH key issues). Always verify deployment by checking the Action status, and if it fails, deploy manually from the EC2 machine itself:

```
cd /opt/brain-viz
git -C /brain pull --ff-only
cp /brain/viz/<files> /opt/brain-viz/<files>
docker compose build <service> --no-cache && docker compose up -d <service>
```

**Evidence:** Multiple deploys in this session required manual SSH because deploy-viz.yml returned failure. We are ON the EC2 machine (hostname ip-172-31-29-157) so docker commands work directly.
