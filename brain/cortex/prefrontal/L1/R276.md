Before changing a previously-fixed component: read the current state of that file first. Confirm the prior fix is still in place before layering on a new change. Changes to hook.py, dispatcher.py, and decision.py are load-bearing — each edit must be read-verified before and after. Never chain changes to infrastructure I've already touched in this session without re-reading the file.
