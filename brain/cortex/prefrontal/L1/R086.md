# R086 -- og Brain — Never use keepPreviousData with map markers
- Created: 2026-04-13
- Source: brain-retro rewire
- Strength: 3
- Tags: correction, brain-retro, vericity, react-query, leaflet

## Rule
When using React Query with Leaflet maps, NEVER use keepPreviousData.
Stale markers cause double-fly. Let data go to undefined during fetch.

## Evidence
v31 added keepPreviousData. Caused zoom-out glitch + stale pins. Removed in v36.
