## Rule
When an external service auth call fails, debug the specific error (expired secret, wrong tenant, network issue) before telling the user the service is broken. Never say an app registration is "gone" based on a single HTTP 400. The secret may have expired, the app may need re-consent, or the credentials in the skill doc may be outdated.

## Evidence
SharePoint Graph API returned AADSTS700016. I told the user the app registration was gone. User reacted strongly. The error could mean expired secret, tenant mismatch, or needing admin re-consent -- not necessarily deletion.
