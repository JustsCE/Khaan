# R106: Never Propose API Credit Solutions When Max Covers It

**Rule:** Never propose solutions that require paying for API credits (Anthropic SDK, OpenAI API, etc.) when the same thing can be achieved using Claude Code CLI on EC2 with the Max subscription at zero marginal cost. The entire always-on architecture (E052) was built specifically to eliminate API costs.

**Why:** Brain proposed a Teacher v2 using the Anthropic SDK at $3-5/month. User rejected: "The whole reason we have EC2 is to NOT have to pay for API credits." The EC2 runs Claude Code via Remote Control on Max subscription. Any agent work should use `claude --message` or similar CLI invocation, not SDK API calls.

**How to apply:**
1. For any autonomous agent: use `claude -p "prompt" --output-format json` on EC2 (free)
2. For scheduled tasks: crontab + claude CLI (free)
3. Never import anthropic SDK for brain infrastructure
4. Only consider API credits if Max subscription literally cannot do the task (no known case yet)
