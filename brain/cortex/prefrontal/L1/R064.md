### R064: og Brain — Use chr(10) for Newlines in Heredoc-Generated Python
- **Strength:** 1
- **Rule:** When generating Python code via Bash heredoc (`python << 'PYEOF'`), NEVER use `\n` inside string literals. Use `chr(10)` instead. The heredoc interpreter converts `\n` to a literal newline, breaking the Python string.
- **Evidence:** Phase A deployment: `text.split("\n")` became `text.split("
")` (literal newline inside quotes = SyntaxError). Fixed by replacing with `text.split(chr(10))`.
- **Exception:** `\n` is safe inside raw strings `r"..."` for regex patterns (not inside heredoc though).
