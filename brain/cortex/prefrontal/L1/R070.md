# R070: Azure Blob ContentSettings Must Be Object

When calling blob.upload_blob() with content_settings, always use:
```python
from azure.storage.blob import ContentSettings
blob.upload_blob(data, content_settings=ContentSettings(content_type="text/csv"))
```

NEVER pass a plain dict like `{"content_type": "..."}`. It causes: `AttributeError: 'dict' object has no attribute 'cache_control'`

**Evidence:** Scraping function scraped 4042 URLs successfully then crashed on blob upload because content_settings was a dict. Required a redeploy to fix.
