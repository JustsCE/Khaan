### R028: og Brain — Safe Hook Deployment Protocol
- **Strength:** 1
- **Source:** brain-retro rewire (E012, E059)
- **Tags:** correction, brain-retro, hooks, cross-platform
- **Rule:** When modifying settings.json hooks: (1) Use `$HOME` variable + Python `os.path.expanduser('~')` for paths — NEVER hardcode platform-specific absolute paths (breaks cross-platform). (2) Test each hook script independently via Bash BEFORE writing settings.json. (3) Use the exit-code filter pattern: `; e=$?; [ $e -eq 2 ] && exit 2 || exit 0` — never `|| true`. (4) User MUST start a new session to load changes (hooks cached at session start). (5) Add one hook at a time if uncertain. (6) The hook.py wrapper is the canonical entry point — all hooks route through it.
- **Evidence:** E012 — relative paths bricked Claude 20+ min. E059 — three successive cross-platform "fixes" each introduced a new lockout mode. The hook.py wrapper + bash safety net pattern finally solved it.
