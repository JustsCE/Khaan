# R126: Never force scheduler state transitions manually

**Rule:** Never write `sched['state'] = 'MAINTENANCE_DUE'` to force a brain cycle. If the scheduler is in NORMAL, that means no cycle is needed. Trust the scheduler's gap counter.

**Evidence:** 2026-04-21 — Brain manually set state to MAINTENANCE_DUE when gap was only 4 (threshold is 5). This caused a cascade: recall gate fired during MAINTENANCE_DUE, 3 blocked attempts triggered MAINTENANCE_FAILED escalation, then had to recover. The forced cycle was unnecessary and delayed the actual task by several minutes.

**Exception:** Only if Justs explicitly says "run a brain cycle now" AND the scheduler is in NORMAL.
