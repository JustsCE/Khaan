# R133: Verify Before Interpreting Classification Labels

**Rule:** When a dataset uses named groups (Robson, treatment arm, stage, etc.), VERIFY the actual coding via value_counts and cross-tabulations on the raw data BEFORE assuming standard definitions apply. Non-standard conventions are common in clinical datasets.

**Why:** In Beatrise's thesis dataset, I assumed Robson 2A1 = induced-with-any-outcome and 2A2 = pre-labour planned CS (the ISO Robson definition). Actually: 2A1 = successful induction (all vaginal, n=176), 2A2 = failed induction (all emergency CS, n=119). The dataset repurposed the 2A sub-split as a success/failure split. My initial LR setup silently failed for 2A1 because outcome had zero variance. Nearly shipped wrong analysis.

**How to apply:**
1. Before running any grouped analysis, check value_counts of the grouping column AND key outcome columns within each group. Pseudocode: for each group, print sub[outcome_col].value_counts().
2. Compare to the definition/standard — any collapsing or re-mapping?
3. If a group has zero variance in the outcome, that is a RED FLAG — investigate the coding, do not just skip the group.
4. Flag non-standard conventions prominently in any deliverable (methods section + footnote in tables).

**Severity:** Critical — wrong assumption = wrong analysis shipped with false confidence.
