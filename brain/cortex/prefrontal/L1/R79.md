# R79: Data Labels Mean Raw Codes, Not Formatted UI

**Rule:** When the user asks to "add data labels" or "show the mapping" for debugging/adjustment purposes, they want the RAW identifiers (codes, IDs, keys) — NOT formatted UI with prices, counts, and styling. The purpose is to see the underlying data mapping so they can adjust it in the backend.

**Why:** User asked to see district cad_group codes on the city map to verify the mapping. Brain instead showed formatted UI labels with district names, listing counts, and avg prices — the opposite of what was needed. User wanted the raw cadastre codes (e.g., "0100008") to verify which polygons map to which districts.

**How to apply:**
1. "Show the mapping" / "add data labels" = show raw identifiers, not formatted display
2. When the user says "so I can adjust it", they're in debug/admin mode — show raw data
3. If unsure whether they want raw or formatted, ask
4. Consider creating a separate admin/debug view rather than modifying the public-facing page

**Severity:** Medium — wasted a deploy cycle showing wrong information
