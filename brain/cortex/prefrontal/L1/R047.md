### R047: og Brain — Verify External Dependencies Before Building Infrastructure Around Them
- **Strength:** 1
- **Rule:** When building a pipeline that depends on an external service (MCP tools, API endpoints, sandbox network), verify the dependency works FIRST with a minimal test. Do NOT build the full delivery infrastructure (dashboard, webhook ingestion, frontend page) and THEN discover the dependency is broken.
- **Exception:** None. Always test the critical dependency first.
- **Evidence:** Session 3 (2026-04-07): Built dashboard page, backend endpoints, git delivery pipeline, deployed to EC2 — then discovered RemoteTrigger sandbox cant load Slack MCP tools or reach goodhealth.lv. All infrastructure wasted until MCP issue is resolved.
