# R144: GitHub fine-grained PAT — rotate everywhere, set yearly

## The 4 places the BRAIN PAT lives

Every rotation MUST update all four atomically:

1. **EC2 host clone**: `~/.claude/.git/config` — embedded as `https://x-access-token:PAT@github.com/...`
2. **Container clone**: `/brain/.git/config` — same embedded URL (used by brain-viz container's git ops)
3. **brain-viz env**: `/opt/brain-viz/.env` `GITHUB_PAT=...` — used by `ensureGitAuth()` in viz/backend/src/index.ts
4. **GitHub Actions secret**: `BRAIN_PAT` on `JustsCE/claude-brain` repo — used by Notify Brain Viz on Changes, Deploy Brain Viz, Context Builder, Slack Puller, Fireflies Puller, Slack Task Processor, Teacher Agent, EC2 Recovery workflows (8 active workflows depend on it)

## Rotation script (idempotent — copy when token regenerated)

```bash
PAT='github_pat_NEW_TOKEN_HERE'
cd ~/.claude && git remote set-url origin "https://x-access-token:${PAT}@github.com/JustsCE/claude-brain.git"
cd /brain && git remote set-url origin "https://x-access-token:${PAT}@github.com/JustsCE/claude-brain.git"
sudo sed -i.bak "s|^GITHUB_PAT=.*|GITHUB_PAT=${PAT}|" /opt/brain-viz/.env
echo -n "$PAT" | GH_TOKEN="$PAT" gh secret set BRAIN_PAT --repo JustsCE/claude-brain --body -
echo "$PAT" | gh auth login --with-token   # so future gh calls don't need GH_TOKEN= prefix
cd /opt/brain-viz && docker compose up -d --force-recreate brain-viz
```

## Default expiry

Always set fine-grained PATs to **1 year** (the GitHub max). Default offered in the regenerate UI is 30 days — that means you do this dance every month. Always change to "Custom..." → 1 year. Email reminders fire 7 days before expiry, plenty of warning.

## Why same-week as gmail (R143)

Both incidents are the same anti-pattern: secret with short expiry, no automation, scattered across N places. R143 covers OAuth refresh_tokens (Production-mode = no expiry). R144 covers GitHub PATs (1-year max). Together they kill the rotation-Whac-A-Mole that's burned at least 4 hours of frustration in 2 weeks.
