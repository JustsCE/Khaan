### R044: og Brain — Never Use python -c for Multi-Line Brain File Writes
- **Added:** 2026-04-07
- **Strength:** 1
- **Source:** brain-retro rewire
- **Tags:** correction, brain-retro, bash, python
- **Rule:** When writing brain files with code blocks, backticks, or shell-interpretable characters, ALWAYS write a temp .py file first (`cat > /tmp/script.py << 'PYEOF' ... PYEOF`) then run `python /tmp/script.py`. NEVER use `python -c "..."` with complex content — Bash will interpret backticks, $variables, and parentheses inside the double-quoted string.
- **Evidence:** Session 2026-04-07: python -c heredoc corrupted CLAUDE.md — Bash interpreted code blocks as commands, SSH'd to goodhealth.lv, ran npm install, leaked server output into the file.
