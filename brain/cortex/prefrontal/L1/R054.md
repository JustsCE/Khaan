### R054: og Brain — Every Excel Formula Must Guard Against 'na' Text
- **Strength:** 1
- **Rule:** When writing Excel formulas that reference forecast cells, ALWAYS wrap cell references with ISNUMBER() checks. Forecast cells contain the string 'na' when no forecast exists. This breaks arithmetic (+, /, -) operations. SUM() ignores text but direct cell references (+) do NOT.
- **Guards required:**
  - Division/growth formulas: =IF(AND(ISNUMBER(X),X<>0,ISNUMBER(Y),Y<>0),X/Y-1,0)
  - Sum formulas: =IF(ISNUMBER(X),X,0)+IF(ISNUMBER(Y),Y,0)+... (NOT =X+Y+Z)
  - Any formula referencing a cell that could contain forecast data
- **Why:** This has broken production 3 times in one session. The 'na' convention is fundamental to the FORECAST pipeline (S005). Every time we write a formula, check for na FIRST.
