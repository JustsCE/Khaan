### R146: og Brain — Docker Compose Service Isolation
- **Strength:** 1
- **Source:** E006-pattern (3 production outages from viz work)
- **Date:** 2026-04-25
- **Rule:** NEVER run docker compose build/up/restart/down/stop without specifying the exact service name. Always isolate: `docker compose build brain-viz && docker compose up -d brain-viz`. A full-stack rebuild takes down nginx, MCP services, WhatsApp bridge, and everything else — causing total production outage.
- **Mechanical:** Yes — check_security() in rule-engine.py blocks bare docker compose commands.
- **Why:** Three times during viz work, bare `docker compose` commands took down the entire production stack on goodhealth.lv. This is the E006 pattern: tunnel vision on viz work leading to reckless infrastructure actions.
