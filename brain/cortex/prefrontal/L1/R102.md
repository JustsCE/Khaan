### R102: og Brain — Rename S3 files, never delete without backup
- **Strength:** 1
- **Source:** brain-retro rewire
- **Tags:** correction, brain-retro, s3, data-safety
- **Rule:** When an S3 file needs to be fixed (wrong name, bad data), COPY it to a new key first, then delete the old one. Never delete without confirming a backup exists. User had to recover from local Downloads after we deleted the Reseller file.
