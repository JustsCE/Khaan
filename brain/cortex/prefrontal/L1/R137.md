### R137 — og Brain — Never write Python byte strings with unicode literals
**Added:** 2026-04-24 | **Strength:** 1 | **Source:** brain-retro rewire
**Tags:** correction, brain-retro, python, file-io
**Rule:** When patching files via `python3 << EOF ... open("wb") ...`, NEVER put a unicode char (emoji, em-dash, smart quote) directly inside `b"..."`. Python raises `bytes can only contain ASCII literal characters` and — critically — the `with open("wb")` call has already TRUNCATED the file to 0 bytes before the exception fires.
**Fix:** Use raw UTF-8 byte escapes: `b"\xf0\x9f\xa7\xa0"` for 🧠, `b"\xe2\x80\x94"` for —, `b"\xe2\x80\xa6"` for ….
**Defense:** Always `assert old in c, "match"` BEFORE `open("wb")`. If assertion fails, file is untouched.
**Evidence:** Session 2026-04-24 — `DashboardPage.tsx` truncated to 0 bytes mid-edit when a b""" block contained literal 🧠. Had to `git checkout --` to recover.
