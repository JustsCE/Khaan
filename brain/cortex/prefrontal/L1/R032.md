### R032: og Brain — Heredoc Quoting Breaks on Complex Python — Use Agent or File Approach
- **Strength:** 1
- **Source:** brain-retro rewire (recall v2 session)
- **Tags:** correction, brain-retro, bash
- **Rule:** When writing multi-line Python via Bash heredoc, single quotes inside Python strings conflict with heredoc delimiters even when using quoted heredoc (<< 'EOF'). Three working approaches in order of preference: (1) Use Agent tool to write the file (agent handles quoting internally). (2) Write a short Python one-liner that creates the script file. (3) Base64 encode the script. Never retry heredoc after first failure — switch approach immediately.
