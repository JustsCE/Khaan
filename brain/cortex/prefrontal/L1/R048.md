### R048: og Brain — Diagnose Before Tinkering
- **Strength:** 1
- **Source:** brain-retro rewire (2026-04-07)
- **Tags:** correction, brain-retro, efficiency
- **Rule:** When debugging infrastructure issues, gather ALL diagnostic data in ONE SSH call before attempting fixes. Do not retry the same failing command — diagnose WHY it fails first.
- **Evidence:** Spent 20min + 25k tokens on goodhealth.lv recovery. Multiple SSH attempts when the first timeout should have triggered stop/start immediately. Multiple nginx restarts when one diagnostic log read would have shown the upstream resolution issue.
- **Specific failures this session:**
  1. Tried SSH 3 times before checking instance state via API
  2. Tried reboot (does not fix networking) before stop/start
  3. Restarted nginx 4 times instead of reading error logs once
  4. Tried to install AWS CLI when boto3 was already available
- **Reinforced:** 2026-04-07 user repeated correction: 20min + 25k tokens is unacceptable, be much more efficient
