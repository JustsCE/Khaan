# R132: GitHub-first Deliverable Output

**Rule:** When delivering any generated artifact (docx, PDF, code, JSON), save it to `brain/artifacts/<topic>/`, commit + push to the `claude-brain` repo, and return the raw GitHub URL. Do NOT propose emailing the file.

**Why:** Justs explicitly corrected (2026-04-23): "dont email save to github and give me link". Email adds friction — Gmail attachments expire, can't easily re-download, require a separate client to open. GitHub URLs are permanent, versioned, and direct-downloadable.

**How to apply:**
1. Save artifact to `/home/ubuntu/.claude/brain/artifacts/<topic>/filename.ext`
2. `cd ~/.claude && git add ... && git commit -m "..." && git push`
3. Return BOTH URLs:
   - Raw (direct download): `https://github.com/JustsCE/claude-brain/raw/main/<path>`
   - Blob (view on GitHub): `https://github.com/JustsCE/claude-brain/blob/main/<path>`
4. Never email. Never attach. Never upload to a third-party.

**Exception:** None known. Even "email this to Beatrise" means save to GitHub first, then Justs forwards manually.
