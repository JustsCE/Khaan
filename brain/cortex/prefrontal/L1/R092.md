# R092: Never Skip Sequential Pipeline Steps

When running the FORECAST pipeline, EVERY step must execute in order. Never skip a step because "the data already exists from a previous run."

**Pipeline order:** submit -> calculate -> (verify)

**Why:** submit rebuilds seperate_forecasts_combined.csv from ALL submit files on S3. If new files were uploaded to S3, skipping submit means calculate uses stale combined data and the new submissions are NOT reflected in the forecast output. User was furious when brain skipped submit after uploading 2 new shoptype files.

**How to apply:** When user says "run the pipeline" or "run calculate" after uploading/modifying files on S3, ALWAYS run submit first to rebuild the combined CSV. The only exception is if the user explicitly says to skip submit.
