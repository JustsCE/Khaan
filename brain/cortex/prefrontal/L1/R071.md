# R071: Azure Consumption Plan 230s Hard Timeout

Azure Functions Consumption plan (both regular and Flex) enforces a 230s hard timeout. The functionTimeout in host.json is IGNORED for Consumption.

When designing Azure Functions that run longer than 3 minutes:
1. Use a time budget (200s) -- checkpoint and exit before 230s
2. Make functions resumable -- check existing output to skip already-processed items
3. Stream large files -- never download entire blob into memory
4. Split work across multiple invocations

**Evidence:** Function scraped 4042 URLs in 160s then died at blob upload (total 230s). Three separate runs all hit the same 230s wall despite host.json setting 10min timeout.
