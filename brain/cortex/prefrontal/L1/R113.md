# R113: Exhaust All Options Before Suggesting Manual Work

**Rule:** When infrastructure is broken (EC2 down, service unreachable), exhaust ALL available credential sources and connection methods before telling the user to fix it manually. Check: ~/.aws/, ~/.credentials.json, environment variables, GitHub secrets, boto3 profiles, alternative ports, DNS changes, GitHub Actions workflows. Only suggest manual intervention after demonstrating you tried everything.

**Evidence:** 2026-04-18 — User said "there is pem on this machine" after brain immediately suggested AWS Console. Brain should have first tried every credential source and connection method, THEN reported what's missing.

**Why:** The user expects the brain to be resourceful and self-sufficient. Defaulting to "you do it" is the opposite of the brain's purpose.
