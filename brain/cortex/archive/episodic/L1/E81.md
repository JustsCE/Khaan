# E81: Dashboard Drag-and-Drop + Tag System Build-Out

**Date:** 2026-04-24
**Emotion:** grind + pride

Multi-hour session building the dashboard drag/tag/animation system from scratch. Started with "crashed the viz" (committed dist/ with broken asset hashes) and ended with a working three-zone drag system + tag promote flow + hologram kit package for Claude Design.

## Key milestones
1. **Crash recovery:** `git rm --cached viz/frontend/dist/` — dist was gitignored but tracked from before the ignore rule. Docker builds from source, so dist in git caused broken deploys.
2. **Switched drag from HTML5 to mouse events** — HTML5 doesn't cross sibling `overflow:hidden` panes (R139).
3. **Tag selector + SICK flying orb animation** — Click tag cell → dropdown → EXECUTE/PLAN/PROJECT → orb flies 2s bezier arc to Brain pane, explodes, shimmer. Then incomingBrainId hides the new task until orb lands.
4. **Three-zone drag** (top/middle/bottom of row) — dispatch: reorder (same pane), move (cross pane), group (middle). Insertion line visual + row shift preview.
5. **Brain pane hologram design** — purple/cyan holographic gradient bg, pulsing brain emoji centerpiece, colored left flag strips per task type.
6. **Packaged the 3D brain model + React Three Fiber components** as downloadable tarball at https://goodhealth.lv/brain-hologram-kit.tar.gz for handing to Claude Design.

## Architectural lessons
- FloatingPane root had no DOM `id` — `document.getElementById('brain-tasks')` returned null. Fixed by adding `id={id}` on the root div.
- brain-tasks endpoint scans ALL *.json files in tasks/ for brain-tags → tasks with PROJECT appeared in BOTH Personal (from whatsapp.json) AND Brain (cross-source). Fix: frontend filters brain-tags out of non-Brain panes.
- Work Triage remote agent wrote individual `wt-2026-04-24-00N.json` files instead of appending to slack.json/fireflies.json arrays. Fix: import script + prompt rewrite for next run.
