# Model routing for cli_invoke subsystem calls

The `cli_invoke` helper in `_shared.py` accepts a `model` parameter that defaults to `'sonnet'`. This design allows each Brain subsystem to select the model tier appropriate to its workload. Subsystems that process large input volumes—retro (`brain_retro.py`), identity boot (`identity_boot.py`), and the consolidation cycle (`cycle_phases.py`)—override the default and invoke with `model='opus'` to gain the larger context window. Subsystems that prioritize latency over capacity, specifically the identity relay and the decision engine, retain the `'sonnet'` default for faster round-trips.

The routing split follows a single principle: choose Opus when the call must digest a large corpus in one pass; choose Sonnet when the call is interactive or time-sensitive. This was introduced as FIX6 and eliminates the earlier rigidity where all subprocess calls used the same model regardless of task shape.
