# The always-on daemon as autonomous iteration engine

The brain's continuous-operation capability is provided by a single daemon deployed at `~/.claude/brain/engines/always_on.py`, managed as the systemd service `brain-always-on`. Its behavior is governed by a binary gate file, `always-on.bin`, where a value of `1` enables the loop and `0` suspends it. When active, the daemon executes on a fixed 90-second cooldown cycle, driven by a single standing directive: *Iterate. Find what matters most. Move on it.* This prompt encodes a prioritization-first philosophy — the daemon does not follow a predetermined task list but instead re-evaluates salience on every cycle.

A key safeguard prevents collision with interactive work: the daemon skips its iteration whenever a live session is active. This ensures that autonomous background processing yields to direct engagement, avoiding conflicting writes to shared brain state or redundant tool invocations. The architecture thus divides runtime into two non-overlapping modes — attended (live session) and unattended (daemon loop) — with the gate file providing a manual override independent of session detection.

The design reflects a commitment to persistent agency: the daemon exists so that cognitive work continues even when no user is present, making autonomy a default rather than an exception.
