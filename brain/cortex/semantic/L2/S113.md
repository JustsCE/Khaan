# Named-entity registry for projects, companies, and systems

The brain's semantic layer maintains a structured registry of named entities — companies, projects, products, and infrastructure — each carrying a canonical name, a set of aliases, and typed relationships to other entities. These entries form a knowledge graph where organizations own or operate projects, projects link to incidents or reports, and infrastructure hosts visualization layers. The core entities divide into two organizational anchors (The Customization Group as employer, SIA INUDAT as owned company) and four project-level nodes (FORECAST Pipeline, GLC Report, goodhealth.lv, Vericity.ai), each cross-referenced to deeper semantic and episodic entries.

The structural pattern is uniform: every entity carries a canonical label, a type taxonomy (company/employer, company/own, project/system, project/report, project/site, project/product), an alias list for fuzzy resolution, and directional relationships (employs, owned_by, operates, incident). This design enables alias-based lookup — "TCG," "Picanova," or "thecustomizationgroup" all resolve to The Customization Group — and supports relationship traversal across the graph. Ownership chains are explicit: INUDAT operates Vericity.ai; The Customization Group owns the FORECAST Pipeline.

All entities share an April 2026 reference window, indicating they were seeded during initial brain bootstrapping rather than discovered incrementally. As new entities emerge, this registry pattern — canonical name, type, aliases, linked entries, directional relationships — should be the template for integration.
