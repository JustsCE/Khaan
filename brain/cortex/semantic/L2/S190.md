# Model routing for cli_invoke subprocess calls

The `cli_invoke` helper in `_shared.py` accepts a `model` parameter with a default of `'sonnet'`. This design lets each Brain subsystem choose the Claude model that best fits its workload. Subsystems that process large input volumes—retro (`brain_retro.py`), identity boot (`identity_boot.py`), and the consolidation cycle (`cycle_phases.py`)—override the default and call Opus for its greater context capacity. Latency-sensitive subsystems, specifically the identity relay and the decision engine, remain on Sonnet where faster round-trip time matters more than raw input headroom.

The routing split follows a simple heuristic: prefer Opus when the subprocess must digest a large corpus in a single pass, and prefer Sonnet when the subprocess runs on the critical path of user-facing responsiveness. Because the default is Sonnet, any new caller inherits the fast path automatically; only callers with a demonstrated need for Opus should opt in explicitly.
