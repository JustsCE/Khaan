# Real Riga Transactions Pipeline Architecture and Schema

The Real Riga transactions pipeline is an S3-to-Excel transformation workflow with no persistent database layer. Raw data lives as four CSV files spanning 2022–2025, totaling approximately 38 MB, stored in S3. A Lambda function (`real-riga-transactions-transform`) reads these CSVs and produces a consolidated `Transactions.xlsx` output. There is no `dbo.transactions` table or equivalent relational store — the pipeline is entirely file-based.

The transaction schema captures Riga residential property sales: `transaction_date`, `price`, `area_apt` (apartment area), `rooms`, `room_address`, `building_cadastre_nr`, `apt_cadastre_nr`, and `property_cadastre` identifiers linking records to Latvia's cadastral registry. Building-level details are present, but geographic coordinates are absent — meaning spatial analysis or map-based visualization would require a separate geocoding step against the cadastre numbers.

The design implies an analytics or reporting use case rather than a transactional one: Excel output, no database persistence, and a batch-oriented Lambda trigger. The cadastre number fields serve as the primary linkage mechanism to external property registries, making them the most structurally important columns beyond price and date.
