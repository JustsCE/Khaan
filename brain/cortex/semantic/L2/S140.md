# Model routing policy for Brain subsystem CLI invocations

The `cli_invoke` helper in `_shared.py` accepts an explicit `model` parameter, defaulting to Sonnet when unspecified. This routing exists because not all Brain phases have identical compute profiles: retrospectives and cycle phases operate on large transcripts—sometimes exceeding 100 KB—that reliably cause Sonnet to time out, while lighter-weight components finish well within Sonnet's latency budget.

The operational rule is straightforward. Retro (`brain_retro.py`), identity boot, and `cycle_phases` route to Opus, which handles long-context synthesis without timing out. Identity relay and the decision engine remain on Sonnet, where sub-second responsiveness matters more than deep reasoning over large payloads. This split was driven by a concrete failure: a 298-entry, 283 KB session transcript caused `brain_retro.py` to time out on Sonnet, prompting the addition of the `model` parameter (FIX6) and the subsequent Opus migration for heavy phases.

The principle generalises: any Brain phase whose input regularly scales with session length should default to Opus; phases that run on fixed-size or small inputs should prefer Sonnet for speed. When adding new phases to the cycle, the model choice should follow input-size expectations rather than defaulting blindly.
