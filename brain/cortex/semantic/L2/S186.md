# Engine security gates enforce write protection on brain infrastructure through HMAC-authenticated bypass

The brain's rule engine is protected by two complementary security gates: `use-bash-block` prevents direct Edit/Write operations on `brain/` paths, while `engine-security-block` prevents Bash-mediated writes to `engines/*.py`. Together, these gates ensure that the agent cannot modify its own execution infrastructure—whether through direct file operations or shell commands—without explicit authorization.

Bypass is not disabled; it is gated. A valid HMAC token, issued by the bypass Lambda and keyed to `.bypass.secret`, is required to override either gate. This design means bypass authority lives outside the agent's own context, preventing self-issued exemptions. The gates have been validated under adverse conditions: during a `MAINTENANCE_FAILED` state, the system correctly blocked attempted self-modification, confirming that protection holds even when internal state is degraded.

The architectural principle is separation of write authority from execution authority. The agent can read, reason about, and invoke its engines, but it cannot alter them without a cryptographically verified external signal. This is a deliberate trust boundary—engine code is treated as infrastructure, not as mutable application state.
