# The Always-On Daemon as Autonomous Iteration Engine

The always-on daemon (`~/.claude/brain/engines/always_on.py`) is a systemd-managed background process (`brain-always-on.service`) that enables continuous autonomous cognition outside of live interactive sessions. Its core directive is a single imperative prompt — *Iterate. Find what matters most. Move on it.* — encoding a tight prioritization-and-action loop as the default mode of unsupervised operation.

Activation is governed by a binary gate file (`always-on.bin`), where `1` enables iteration and `0` suppresses it, giving external control over whether the daemon runs without requiring a full service restart. The daemon is session-aware: it detects live interactive sessions by checking transcript modification time and skips its iteration cycle when a session has been active within the last 90 seconds, preventing conflict between autonomous and interactive cognition. Logging output routes to `~/.claude/brain/logger/`, maintaining an audit trail of autonomous actions.

The architectural pattern is a gated, self-yielding loop: the daemon continuously cycles through prioritization and execution but defers to human-interactive sessions and respects an external kill switch. This design treats autonomous operation as the baseline state rather than the exception — the system is always on unless explicitly paused or preempted.
