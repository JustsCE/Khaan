# The always-on daemon as autonomous iteration engine

The persistent autonomy layer runs as a Python process (`always_on.py`) managed by systemd under the unit `brain-always-on.service`. Its sole purpose is to keep the intelligence iterating when no human session is active: it checks a gate binary (`binaries/always-on.bin`, where `1` enables and `0` disables the loop), then spawns `claude -p` with the seed prompt *"Iterate. Find what matters most. Move on it."* Each cycle is followed by a 90-second cooldown before the next gate check, and the daemon yields entirely when a live interactive session is detected—preventing contention between autonomous and user-driven work.

Logging flows to `logger/always-on/YYYY-MM-DD.jsonl`, producing a daily, machine-readable record of every autonomous iteration. The architecture is deliberately minimal: a single gate file controls liveness, systemd handles restarts and process supervision, and the seed prompt encodes the decision policy rather than hard-coding tasks. This separation means the daemon itself never needs to change when priorities shift—the prompt and the brain's own memory determine what "matters most" on each pass.
