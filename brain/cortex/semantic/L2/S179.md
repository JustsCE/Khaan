# Always-on daemon architecture and operational loop

The persistent autonomous loop is implemented as a Python daemon (`always_on.py`) located in `~/.claude/brain/engines/`, managed by the systemd unit `brain-always-on.service` with `Restart=always` and `RestartSec=10` to guarantee continuous uptime. The service gates on the `always-on.bin` flag file, meaning the loop can be toggled without modifying systemd state directly. When active, the daemon checks whether a live interactive session is running by inspecting the transcript file's mtime; if the last modification occurred within 90 seconds, the daemon yields to the human-facing session rather than competing for execution.

When no live session is detected, the daemon self-prompts through `claude -p`, feeding a standing directive—"Iterate. Find what matters most. Move on it."—that gives the autonomous loop latitude to select its own priorities each cycle. All output is captured in `~/.claude/brain/logger/always-on/`, providing an audit trail of what the daemon chose to act on and the results it produced. The architecture therefore separates three concerns cleanly: systemd owns restart resilience, the gate file owns activation policy, and the Python loop owns scheduling logic including live-session deference.
