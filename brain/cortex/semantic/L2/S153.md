# Infinite recursion in _recursive_median_split when clusters converge at median threshold

The `_recursive_median_split` function in `cycle_phases.py` contains a latent infinite-recursion bug triggered when all nodes in a component cluster at or near the median similarity threshold. Under this condition, the split produces a subcomponent identical in size to the input, so the next recursive call re-enters with the same data, never terminating. The failure mode is silent until it hits Python's recursion limit and crashes the consolidation cycle.

Two independent safeguards are necessary. First, a depth guard that force-truncates recursion beyond a fixed ceiling (depth greater than 20). Second, a progress check that compares the size of each resulting subcomponent against the size of the eligible set passed in: if `len(sub) >= len(eligible)`, the split made no progress and must be force-truncated rather than recursed into. Neither guard alone is sufficient—depth alone permits expensive no-op recursion up to the cap, and progress alone fails to protect against pathological chains of marginal reductions.
