# Sonnet context-window timeout threshold for brain-retro transcripts

The `brain_retro.py` retrospective pass fails under Sonnet when the session transcript exceeds approximately 283 KB (around 298 entries). At that payload size, Sonnet cannot complete extraction within the 300-second timeout, causing the retro to silently abort or error out. Two mitigations exist: switching the retro model to Opus, which handles larger contexts within the same timeout, and truncating the transcript to the last 30–50 entries before submission. Truncation serves as a fallback that keeps Sonnet viable for most sessions while sacrificing older context that is typically less actionable for retrospective extraction.

The practical boundary is that any session producing fewer than roughly 280 KB of transcript can run retro on Sonnet without issue. Beyond that threshold, the system must either escalate to Opus or shed older entries. Because retro primarily surfaces patterns from the most recent work, tail-truncation loses relatively little signal compared to the cost of a full timeout failure.
