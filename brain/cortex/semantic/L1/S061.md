# S061 -- og Brain — Ship real or don't ship: dark-launch pattern rejected

Justs rejects "dark-launched with cron disabled" as a shipping pattern for new infra.

## Why
- If trigger is off, code provides zero value
- Sits in repo as debt, future session either resurrects it unreviewed or trips over it
- Violates charter: complete the thing or don't add the thing
- E068: autonomy-engine v1 shipped this way, user demanded full revert

## How to apply
When designing a new subsystem:
- Either wire it live (with kill switch like a pause file, not a disabled cron), or
- Keep it as a design doc only (DESIGN.md + SKILL.md, no executable state files)
- Never commit runtime code paired with "trigger is off" comments

The right staging is: design -> design review -> wire live with an off-switch -> observe -> promote. Not: design + ship code + leave trigger off indefinitely.
