### S013: og Brain — Claude Code Permission Scoping
- **Domain:** Infrastructure / Claude
- **Learned:** 2026-03-31
- **Reinforced:** 0 times
- **Strength:** 0
- **Facts:**
  - settings.local.json Edit permission only auto-allows within the current project directory
  - Edits to files outside project dir (e.g., ~/.claude/ when project dir is different) still prompt
  - Workaround: use Bash tool for cross-directory file writes (Bash covers all paths)
  - Best practice: when editing brain files, work from within brain repo as project dir
  - This has caused user frustration TWICE (2026-03-31)
- **Apply when:** Any session that needs to edit brain files from a non-brain project
