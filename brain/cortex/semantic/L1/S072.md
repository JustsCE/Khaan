### S072: Viz Canonical Source vs Deployment Copy

- **Canonical source:** ~/.claude/viz/ — committed to claude-brain repo, deployed via deploy-viz workflow (push triggers SSH-deploy to EC2 goodhealth.lv).
- **Deployment copy:** slash-opt/brain-viz/ — lives on the EC2 host (and apparently also on the workstation for local dev). Built/served by Docker compose under slash-opt/brain-viz/.
- **Sync rule:** When fixing a bug touching frontend/src or backend/src, edit BOTH:
  1. ~/.claude/viz/... — so the next CI run deploys it.
  2. slash-opt/brain-viz/... — so local dev / already-running container picks it up.
- **Edits in slash-opt only:** Will be wiped on next deploy-viz workflow run that overwrites the host copy. Do not rely on local-only patches.
- **Edits in ~/.claude/viz/ only:** Will deploy via CI but do not immediately update an already-running local docker stack at slash-opt/brain-viz/.
- **R009_GUARD note:** ~/.claude/viz/ is in the carveout list, so Edit and Write tools work directly. No need for python heredoc workaround for routine viz edits.
- **Confirmed:** 2026-04-28 — the two trees are byte-identical when no in-flight session edits exist.
