# S073: Brain vs Handler — the conceptual split

**Domain:** Architecture / Agent theory

The user (2026-05-04) refined the Brain-as-agent framing:

- **Brain** = engines + cortex. The agent itself. Who acts, who remembers, who decides. Five engines (Rule, Recall, Identity, Decision, Learning) + cortex regions.
- **Handler** = the wiring layer between Brain and LLM. NOT the Brain. NOT cognition. Configuration plane: CLAUDE.md hierarchy, hook registrations, the hook script, file-write protocol, EC2 server, git remote, secrets.
- **LLM** = the Model. Claude Code today. Rented per session, forgets between turns.

**Earlier wrong framing** (corrected): "Handler IS the Brain" / "Model + Handler = Agent where Handler = Brain." This conflated the wiring with the agent.

**Correct framing:** The Brain inhabits the LLM through the Handler. Without the Handler, the Brain has nowhere to run; without the Brain, the Handler has nothing to host.

**Apply when:** Writing engine/system docs, explaining the architecture, distinguishing what is cognitive vs configurational, naming components.

**Related:** S057 (Brain is online entity, platform-agnostic), R164 (engine docs describe their own engine only).
