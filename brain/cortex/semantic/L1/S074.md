# S074: CLAUDE.md memory system — authoritative loading facts

**Domain:** Claude Code / Memory system
**Source:** code.claude.com/docs/en/memory (fetched 2026-05-04)

## What auto-loads

- Managed policy CLAUDE.md (macOS, Linux, Windows paths) — every session, NOT excludable
- Project CLAUDE.md or .claude/CLAUDE.md — when in that project
- User ~/.claude/CLAUDE.md — every session
- Local <project>/CLAUDE.local.md — gitignored, per-project
- Subdirectory CLAUDE.md — on demand when files in that dir are read
- Project .claude/rules/*.md — every session unless YAML paths: scopes
- User ~/.claude/rules/*.md — every session
- Auto-memory MEMORY.md — first 200 lines or 25KB at session start

## Critical facts

1. **Loaded in full regardless of length.** No token cap. Anthropic recommends <200 lines per file for adherence quality only.
2. **Delivered as USER MESSAGE after system prompt** — NOT as system prompt. Only --append-system-prompt reaches the actual system prompt.
3. **AGENTS.md is NOT auto-loaded.** Need CLAUDE.md to import via @AGENTS.md.
4. **Load order:** root → cwd. CLAUDE.local.md appended after CLAUDE.md within a dir. Closer to cwd = read last.
5. **@path imports** recursive, max 5 hops. Relative, absolute, and ~/ paths allowed.
6. **/compact survival:** only project-root CLAUDE.md re-injected. Nested reload when files in those dirs are read.
7. **Block-level HTML comments stripped** before injection. Comments inside code blocks preserved.
8. **MEMORY.md cap:** first 200 lines or 25KB. Topic files load on-demand only.
