- **Domain:** Vericity / Data Engineering
- **Learned:** 2026-04-02
- **Reinforced:** 0 times

- **Facts:**
  - Raw data: dbo.listings_ss (daily scrapes from ss.lv). Columns: street_name, building_name, building_type, price, area, rooms, extras, etc.
  - Geocoding: dbo.address_data (street_name, house_nr, Cadastre_nr, LAT, LON) — joined via cleaned street name + house_nr
  - Street mapping: dbo.ss_streetnames (ss_name → address_name) — resolves ss.lv naming vs canonical names
  - Fact table: dbo.listings_fact — rebuilt from listings_ss + address_data + ss_streetnames via INSERT SQL
  - District assignment: CASE WHEN on LEFT(Cadastre_nr, 7) — cadastre code → district name (15 zones)
  - Extras parsing: 'lifts%' → elevator, '%Balkons%/%Lodžija%/%Terase%' → balcony, '%Parkošanas vieta%' → parking
  - price_unit: 'sell' or 'month' (EXPLICIT values, not NULL-based)
  - House Type = House Series = building_type (same concept, one column)
  - Building type names in DB have NO dots/commas: "P kara", "Jaun", "Renov" etc. User prefers this format.
  - 8% of listings fail geocode join (unmatched street names need ss_streetnames entries)
  - Admin page at /admin allows: rebuild fact table, edit district mapping, manage street mappings
