# og Brain — scipy.stats.chi2_contingency — Yates Correction Gotcha

scipy.stats.chi2_contingency(ct) applies Yates' continuity correction by default (correction=True) on 2x2 tables. This systematically lowers chi-squared compared to classic Pearson.

## Symptoms
- 2x2 test output chi-squared is ~5-20 percent lower than expected
- p-values are slightly higher (conservative)
- 3x2 / 3x3 / larger tables are unaffected (Yates only applies to 2x2)
- Cramer's V = sqrt(chi-squared/n) correspondingly lower

## Convention
- For n >= 40 per cell (typical medical studies with hundreds of patients), Yates is over-conservative.
- Modern medical statistics convention: NO Yates correction (Pearson chi-squared).
- To disable: chi2_contingency(ct, correction=False).

## When this matters
When re-implementing someone else's statistical analysis and comparing outputs — silent default can cause apparent "discrepancies" that are actually just Yates-on vs Yates-off. Always check both before concluding the reference is wrong.

## Concrete example (Beatrise thesis, Robson 2A, Dzemdibu tips)
- Yates=True:  chi-squared = 8.96, p = 0.003, V = 0.174
- Yates=False: chi-squared = 9.69, p = 0.002, V = 0.181
