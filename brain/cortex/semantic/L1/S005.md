- **Domain:** Picanova / FORECAST
- **Learned:** 2026-03-30
- **Reinforced:** 3 times
- **Strength:** 0
- **Facts:**
  - Blue cell (DCE6F1) = manual override value in FORECAST submission
  - Blue `na` = month where user was EXPECTED to enter a forecast but didn't
  - Non-blue cells = formula reference only, not user input
  - Submit parser: skips blue `na` correctly (no value to extract)
  - Submit-verify: blue `na` IS a quality issue — flags as missing forecast coverage
  - Key distinction: `na` is valid parsing behavior but a data quality problem
