### E009: og Brain — Deploy Workflow 8-Iteration Debug Session
- **Date:** 2026-04-01
- **Strength:** 1
- **Context:** Setting up CI/CD for goodhealth.lv — GitHub Actions → SSH to EC2 → Docker rebuild
- **What happened:** deploy-viz.yml failed 8 times in a row, each with a different issue: missing SSH key secret, private repo clone needing PAT auth, npm peer dep conflict (three.js version), port 3001 already allocated, port 80 held by orphan nginx, SSL certs trapped in Docker named volume. Fixed each iteratively by reading workflow logs via GitHub API. Eventually broke HTTPS by fuser -killing host nginx. Site currently HTTP-only, needs manual SSH cert migration.
- **Lesson:** Docker deploy workflows need: (1) docker compose down BEFORE anything, (2) fuser -k for host processes on required ports, (3) SSL cert migration when switching from named volumes to host mounts requires tar (not cp) to preserve symlinks, (4) always have HTTP-only fallback config so nginx can start without certs. Also: check GitHub Actions logs programmatically via API instead of guessing.
- **Emotion:** frustration-then-resolution — site went down during the process but was recovered on HTTP
