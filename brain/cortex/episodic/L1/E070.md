# E070 -- Docker port mapping bug breaks WhatsApp agent replies

## What happened
wa-bridge (3006) and mcp-whatsapp (3007) Docker containers had no ports: mapping in docker-compose.yml. Host nginx proxied to 127.0.0.1:3006/3007 but containers only exposed ports to Docker internal network. Result: 502 Bad Gateway on all /whatsapp/ and /mcp/whatsapp/ routes. WhatsApp agent could not send replies via MCP.

## Detection
User reported no WhatsApp reply. curl to /whatsapp/status returned 502. Internal Docker network test (docker exec wget) worked fine — proved the issue was host-to-container, not container health.

## Fix
Added ports: ["3006:3006"] and ports: ["3007:3007"] to docker-compose.yml. Agent run 239 confirmed working.

## Lesson
Always verify Docker port mappings after modifying docker-compose.yml. The container can be healthy internally but unreachable from host nginx.
