# E080: Viz Crash from Committed dist/ with Broken Asset Hashes

**Incident (Apr 24):** Brain session ran `npm run build` which regenerated dist/ with new Vite hashes. Committed index.html pointing to new bundle hash but deleted old bundles without adding new ones. deploy-viz.yml auto-deployed broken build to goodhealth.lv — blank page, entire viz down.

**Root cause:** dist/ was tracked in git despite being in .gitignore (files added before ignore rule existed). Docker already runs `npm run build` in multi-stage build, so dist/ in git is redundant and dangerous.

**Fix:** `git rm -r --cached viz/frontend/dist/` — removed from tracking, .gitignore now effective. 3D model assets (brain.glb etc) safe in public/.

**Lesson:** Never commit build output when the deploy pipeline builds from source. Static assets belong in public/, not dist/.
