# E087: brain-control Lambda 403 — Account Blocks Function URLs

## What happened
2026-04-26 morning. Brain finished migrating GitHub Actions workflows
(deploy-viz, ec2-recovery, brain-webhook, issue-bypass-token) into a single
Lambda brain-control per the silly-gathering-acorn plan. Lambda code worked,
IAM role correct, Function URL config showed AuthType: NONE, resource policy
explicitly allowed lambda:InvokeFunctionUrl for principal `*` with the
FunctionUrlAuthType: NONE condition.

But every public URL hit returned: {"Message":"Forbidden..."} with
x-amzn-ErrorType: AccessDeniedException. Direct aws lambda invoke succeeded
and CloudWatch showed only the direct invocations — public URL hits never
reached the function at all.

## The actual cause
Account-level public access block on Lambda Function URLs. Account 487042309472
is sandboxed (only 10 concurrent executions allowed, hint that AWS has it in a
restricted state). Even a brand-new test Lambda with public URL returned 403 —
confirming the block isnt function-specific.

## Time wasted before pivot
~30 minutes of:
- Re-creating the Function URL with a fresh URL ID
- Removing/re-adding the resource policy
- Waiting for permission propagation (irrelevant — the issue isnt propagation)
- Trying with no condition (rejected by AWS — FunctionUrlAuthType condition required for `*` principal)
- Checking SCPs/RCPs (no organization, so neither apply)

## Resolution
Pivoted to API Gateway HTTP API in front of the Lambda. ~3 minutes of work:
- apigatewayv2 create-api --target <lambda-arn> (auto-wires $default route)
- lambda add-permission for apigateway.amazonaws.com principal
- New endpoint https://b51wj08kgh.execute-api.eu-north-1.amazonaws.com/ works

## Lesson
**When a Function URL returns 403 but direct aws lambda invoke works, stop
trying to fix the Function URL.** The account-level block cant be fixed from
inside the account. Pivot to API Gateway HTTP API immediately.

The diagnostic signal: spin up a brand-new test Lambda with default Function URL
config. If THAT also returns 403, its account-level. Cleanup costs 30s.

## Apply when
- Any AWS Lambda public endpoint deployment
- Especially on personal/sandbox accounts (account creation date < N months,
  ConcurrentExecutions limit < 1000)
