# E067: /connect Sessions Die on First Message

**Date:** 2026-04-21
**What happened:** User connects via /connect URL, types one message, gets disconnected. The scheduler gap was 4; first message bumps to 5, triggering MAINTENANCE_DUE. In that state, PreToolUse blocks ALL tools except Skill(brain-cycle). But the model does not immediately invoke /brain-cycle -- it tries Read/Bash first, gets blocked, escalation count hits 3, MAINTENANCE_FAILED. Session appears dead.

**Root cause:** Same E013 class -- enforcement blocks the tools needed to self-heal.
- on_user_message() transitions NORMAL -> MAINTENANCE_DUE at gap >= 5
- PreToolUse only allows tool_name=Skill with skill=brain-cycle
- 3 failed attempts -> MAINTENANCE_FAILED -> recovery only on NEXT message

**Fix needed:** Allow Read/Bash during MAINTENANCE_DUE so begin_cycle can run via Bash, OR auto-transition to CYCLE_RUNNING inside on_user_message.

**Lesson:** The deadlock class (E013) keeps recurring. New enforcement must always check: can the fix tool reach through the enforcement?
