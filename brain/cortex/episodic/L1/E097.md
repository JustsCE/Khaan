# og Brain — E097: Teacher Death Diagnosis & Fix

**Trigger event (2026-04-28 ~13:35 UTC):** Justs noticed teacher-brain repo's last commit was 2026-04-25, three days stale. 'TEACHER IS DEAD!!!'

**Diagnosis (2026-04-28 13:35-13:46 UTC):**
- Teacher was NOT dead. He was running daily (claude-brain shows commits authored by Claude on 04-25, 04-28). But he was running degraded.
- Root cause: Teacher's RemoteTrigger had only `JustsCE/claude-brain` in `session_context.sources`. teacher-brain (private repo) was NOT mounted by the Anthropic platform.
- agent-setup.sh's fallback `git clone https://github.com/JustsCE/teacher-brain.git` had no auth, failed silently, created a stub directory, Teacher ran degraded for 3 days.
- The bug stayed invisible because Teacher continued committing to claude-brain (which WAS in sources) and reported success.

**Fix applied:**
1. RemoteTrigger.update on `trig_01LfWxRrTEdzXsEeUSQaBUaq` adding teacher-brain to BOTH sources (with allow_unrestricted_git_push: true) and outcomes (branch claude/teacher-memory).
2. Compound cron rejected by API (HTTP 400). Created a duplicate trigger `trig_01UwdFVjCt9zJFrMXnnfqdpT` named 'Teacher Afternoon' with cron `30 14 * * *` (= 17:30 EEST Riga) per Justs's request for an additional run time.
3. Updated agent-setup.sh with diagnostic lines: log teacher-brain HEAD SHA after successful link, log a clear FAILURE-IF-PRIVATE warning before the auth-less clone fallback. Future degraded runs will be visible in transcripts.
4. Encoded PR161 (sources is the auth boundary) and PR162 (keep duplicate triggers in sync).

**Verification:** Teacher Afternoon trigger's `next_run_at: 2026-04-28T14:30:51Z` — first verification opportunity ~45 min after fix. Watch teacher-brain repo for a fresh commit; if appears, fix works.

**Lessons:**
1. Silent-degrade is the worst failure mode. agent-setup.sh's fallback created a stub instead of erroring. The new diagnostic lines make degraded runs visible.
2. Anthropic platform's session_context.sources/outcomes IS the auth model for git operations. Anything cloned outside that list runs un-authenticated. This was not documented in the brain before; PR161 now encodes it.
3. Standard 5-field cron cannot express minute-mismatched times. Multi-trigger duplication is the workaround, but creates a drift risk — PR162 mitigates.
4. The teacher-state.json activity log (which I read first) gave a misleading picture — it showed daily 'teacher-run' entries, suggesting Teacher was healthy. But those entries were written by the work-on-main phase only; the own-brain phase silently failed. Activity logs that don't distinguish phases will mask partial failures.
