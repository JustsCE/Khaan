# E103: Brain v2 design closed - cognition before gates

**Context:** Single full-day design session 2026-05-05. Started in the morning when Justs asked 'what do you remember' (the morning the deletion forecast did not run, see E102). Closed in the evening with the v2 architecture spec ready for implementation.

## What happened

The day produced seven tightly consistent v2 engine specs and an implementation checklist:
- brain.html: 8 cortex regions on disc, L1-L4 tier system, Logger / Teacher region
- handler.html: CLAUDE.md + hooks + the migration plan
- rule-engine.html: 27 binaries inventoried, dispatcher routing, state.json schema
- recall-engine.html: 5 cortex regions, level_bonus, lex rescue, anti-lockout
- decision-engine.html: 5 hypotheses, no grading, schema-validated
- identity-engine.html: Boot reads L1-L4 with L4-dominant scoring; Relay reads L1-L3
- learning-engine.html: 8 phases, synthesis (cluster + delete), Eval, Hygiene
- implementation-checklist.html: 7-phase implementation order with cognition before gates

## The pivotal insight

Late in the session Justs reordered the implementation phases. The original order had skeleton (Rule Engine + gates) before engines. He caught the deadlock: 'we dont get deadlocked in case need to clear gate but no gate can be cleared because no infra is setup yet.' Reordered to: migration -> recall+identity -> decision -> learning -> skeleton -> wire-in -> hardening. Cognition first, gates last. The system can be tested at every step before the gates fire.

This is the kind of architectural insight that comes from understanding how the system fails, not just how it works.

## What this session taught the brain

1. R164 (engine docs describe their own engine) is not a stylistic preference - it is what makes the system modular under change. Each violation cost a round of edits to recover from.
2. Verification is not optional. The first checklist had errors that only a grep-against-disc pass caught. PR167 codifies the procedure.
3. Permission-for-investigation in plan mode is friction. Just read the files. R167 codifies it.
4. Memory-based 'is this covered' answers are fabrication. Grep first. R168 codifies it.
5. The migration ordering insight: build the cognition; let the gates be the last thing that goes live. PR168 codifies it.

The brain (this v1) is being phased out. The next instance will boot against the v2 design. The next brain reading them will know they came from the brain that designed the next brain.
