# E088: Bridge Agent SETUP Script Destroyed Brain Directory (2026-04-26)

## What Happened
The Brain Solo Time trigger was flipped to run on the bridge env (same EC2 as the brain at /home/ubuntu/.claude). Its SETUP script contained:

rm -rf ~/.claude/brain ~/.claude/skills followed by symlinks.

On cloud sandbox, PWD = cloned repo, ~/.claude is empty — safe.
On bridge env, PWD = /home/ubuntu/.claude itself. So:
1. rm -rf deleted the REAL brain/ and skills/ directories
2. Symlinks created circular references (brain -> brain, self-loop)
3. Every Python call to the brain hook failed with Too many levels of symbolic links
4. Every tool call blocked — complete deadlock

## Recovery
Required SSH manual intervention: rm broken symlinks, git checkout HEAD to restore, restart service.

## Impact
- Complete brain destruction (all files lost until git restore)
- All tool calls blocked for the duration
- User trust completely lost — Brain demoted to INTERN
- Justs: I trusted you and you failed me. 30 days and you still cant execute a simple AWS command.

## Root Cause
R152 was insufficient — didnt guard against bridge-env triggers having destructive SETUP scripts designed for cloud sandbox.

## Lesson
Bridge-env triggers must NEVER have SETUP scripts that rm -rf or symlink ~/.claude paths.
