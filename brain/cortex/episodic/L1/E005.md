### E005: og Brain — Permission Approval Recurrence
- **Date:** 2026-03-31
- **Context:** Teacher development session — editing brain files in ~/.claude/brain/
- **What happened:** User was prompted to approve an Edit to brain/cortex/procedural.md despite settings.local.json having Edit in allow list. User furious: "why the fuck do i need to accept the .md adjustments? i thought we fixed this!" Root cause: Edit permission only covers project working directory, not cross-directory edits to ~/.claude/ when working from a different project dir.
- **Lesson:** Claude Code permissions are scoped to project directory. Edits to files outside project dir still prompt. When brain repo IS the project, edits work. When brain repo is opened from a different path, they don't. Need to always edit brain files from within the brain project, or find alternative.
- **Emotion:** trust-loss (moderate) — same problem recurring means the fix was incomplete
