# E072: Dashboard Zombie Bug — 45-min Debug Hunt

**Date:** 2026-04-21
**Context:** Justs reported stale Slack/Fireflies tasks kept reappearing on dashboard after clearing. Had tried multiple times before this session.

**What happened:**
1. Added ACTIVE_SOURCES filter to webhook handler — agentSources=[] confirmed, yet files repopulated
2. Added saveTasks debug logging with stack traces — showed 0 calls, yet 10KB files appeared
3. Checked git history, stashes, submodules, file watchers, process list — all dead ends
4. Finally discovered: `git log --all --oneline -- dashboard/` inside container revealed commit 0d2d5ef that added task files to repo root
5. `git show HEAD:dashboard/tasks/slack.json` confirmed files were in git HEAD
6. `git reset --hard origin/main` was restoring them on every webhook — the JS code was innocent

**Key insight:** When a Docker volume mounts a git repo, `git reset --hard` operates at the filesystem level. It doesn't care about your application code's filters — it restores tracked files before your code even runs.

**Emotion:** frustration → satisfaction. Justs said "Jesus that took you long but good job."
