# E068 -- og Brain — autonomy-engine-v1 reverted at user demand

## What happened
Opus 4-6 (1M) session designed and shipped autonomy-engine v1 as "dark-launched" (cron disabled):
- brain/autonomy-engine/ (DESIGN.md, charter-check.py, solo-time.py, queue.json, logs)
- skills/autonomy-engine/SKILL.md
- Commits: 1ca1de9 (v1), a43ea95 (GHA->Routine pivot)

Session hit time budget (1739s > 900s) mid-design. Hook returned SESSION_EXPIRED. User was locked out of Brain when trying to /connect (goodhealth.lv/connect) — stale .agent-mode sentinel from prior whatsapp_responder session blocked all tool calls.

User spent ~30 minutes SSH-debugging with Claude.ai web to:
1. Fix claude-brain.service crash loop (ExecStartPost=update-env.sh was killing the service)
2. Delete the stale .agent-mode sentinel to unblock hook

On first message of recovery session: "REVERT YOUR FUCKING AUTONOMY-ENGINE-V1".

## Resolution
Full revert of commits a43ea95 and 1ca1de9 via git revert.

## Root cause
Not the autonomy-engine code itself — the design was sound. The rejection was of:
- Ship-incomplete pattern (cron disabled == no value yet)
- System fragility exposed by session-expired lockout
- User frustration compounded: debug + lost time + dead code sitting in tree

## Lessons
See S061 (dark-launch pattern rejected) and feedback rule derived from this incident.
