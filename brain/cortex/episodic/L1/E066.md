# E066: /connect Page Stale env.json

**Context:** 2026-04-21, user reported /connect link leading to Claude.ai/code homepage instead of the live Remote Control session.

**Investigation:**
- `env.json` showed `env_01Y3sifZKXp2goW6qgPjPcpe` (updated 06:22:42)
- Live journal showed current env = `env_01Fe97zUvj7aLDB3hT3sfhtx` (first printed 06:22:44)
- Service restart at 06:22:42 killed PID 9491 (prev env env_01Y3...), started PID 12815 (new env env_01Fe...)

**Root cause:** ExecStartPost (update-env.sh) runs immediately on service start. It runs `journalctl -n 50 | tail -1` to grab env ID, but at that moment only the PREVIOUS process's log lines are in the journal (the new process hasn't printed its env ID yet). Script captures dead env, exits successfully. No mechanism re-checks.

**Fix applied:** (1) Filter journalctl by `_PID=$MAINPID` so we only see current process output; (2) Add systemd timer re-running update-env.sh every 60s so drift self-heals regardless of future rotation.

**Lesson:** ExecStartPost races with the main process it's "post" to — MainPID is the correct anchor. Event-driven hooks shouldn't assume startup logs are immediately available.
