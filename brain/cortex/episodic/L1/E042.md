## What Happened
Deployed vericity:v7-full to Container App. Health check returned 200, revision showed 100% traffic. Declared it live. User reported it was NOT live. Need to check container logs for actual startup errors.

## Lesson
Health endpoint responding 200 and revision showing active is NOT sufficient verification. Must actually load the site in browser or check container logs for startup errors before declaring deploy complete.

## Correction
RG was wrong initially (used inudat, actual is vericity-rg). Fixed that. But even after correct RG update, site not working. Always verify with actual page load.
