### E016: og Brain — goodhealth.lv Nginx Crash from Dead MCP Container
- **Date:** 2026-04-06
- **Strength:** 1
- **Tags:** infrastructure, nginx, docker, mcp
- **Related:** PR013, PR016
- **Summary:** nginx crash-looped because mcp-aws container could not start. nginx hard-fails on startup if upstream hostname unresolvable. Fixed with resolver 127.0.0.11 + variable-based proxy_pass for runtime DNS resolution. Dead containers return 502 instead of crashing the site.
- **Lesson:** Docker service dependencies must be soft -- use runtime DNS resolution, not startup-time resolution.
