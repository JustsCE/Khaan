### E036: og Brain — Rule Engine v2 Deadlock - Counter Never Syncs
- **Date:** 2026-04-09
- **Tags:** brain, security, deadlock, rule-engine
- **Salience:** 0.9
- **What happened:** v2 rule engine had 5 interacting bugs causing MAINTENANCE_DUE every message:
  1. Double counter increment (shell hook + python both increment)
  2. Verifier was only code that synced update file, but INFRA_GUARD blocked running it
  3. Manual workaround bypassed guards but could not touch counter files
  4. Gap grew by 2 per message, maintenance triggered every single message
  5. Escalation fallback wrote fake learning to satisfy verifier
- **Resolution:** v3 atomic design: single state file, one counter owner, no fake-learning, MAINTENANCE_FAILED with auto-recovery
- **Lesson:** Security guards that protect by substring matching create deadlocks when protected operations are needed for recovery. Protect resources and transitions, not strings.
