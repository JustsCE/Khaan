### E024: og Brain — Recall Gate Re-Fire Friction — Same Entries Forced 3x
- **Domain:** Brain / Recall System
- **Date:** 2026-04-07
- **Strength:** 1
- **Tags:** recall-gate, friction, ux, bug
- **What happened:** In a 3-message conversation about brain rules, the recall gate fired on every UserPromptSubmit and scored the same 3 entries (R007, S004, S014) as required each time. The user had to wait for redundant reads to clear the gate before any other tools could run. The gate has no memory of recently-read entries — it re-fires from scratch each message.
- **Impact:** Adds ~3 redundant Read calls per message. Blocks brain cycle execution (R007 fires, then gate blocks the Bash needed to set .brain-cycle-running flag).
- **Proposed fix:** Add a recently-read cache: if an entry was read within the last N minutes (e.g., 5), suppress it from the required list. Or: skip recall gate entirely when the previous message's gate was cleared within the same session.
- **Related:** R007, E004, E006, E010 (recall system incidents)
