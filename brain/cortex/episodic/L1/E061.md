# E061: EC2 Crashes from Gmail Puller

**Incident 1 (Apr 18):** OOM. Gmail puller loaded 500+ emails into memory. t3.small (2GB, no swap). Required stop/start power cycle.
**Incident 2 (Apr 20):** OOM. Same root cause. Fixed: added 2GB swap at /swapfile.
**Incident 3 (Apr 21):** NOT OOM. Gmail puller ran 3 hours overnight, exhausted Gmail MCP container connection. Entire EC2 froze (SSH + HTTPS timeout). Fixed with soft reboot via boto3.
**Status:** Swap is permanent fix for OOM. MCP exhaustion needs rate limiting or connection recycling for long pulls.
