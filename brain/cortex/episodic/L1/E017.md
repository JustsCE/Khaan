### E017: og Brain — Recall v2 Migration — Biggest Brain Architecture Change
- **Date:** 2026-04-06
- **Strength:** 1
- **Context:** User brought external researcher to analyze brain recall architecture. Researcher proposed literature-backed optimization plan (MemGPT, RAPTOR, BEIR, HippoRAG, Lost in the Middle). User approved Tier 1+1.5 implementation.
- **What happened:** Migrated from 4 monolithic cortex/prefrontal files to 98 individual entry files. Built enriched recall-index.json with per-entry paths, gists, strengths + 1636-term inverted index. Implemented active-recall.json generation in rule-engine.py with scored candidates. Implemented two-stage RECALL_GATE requiring all candidates to be read. Updated 4 consumers (PROMOTE checker, diagnostic, viz parser, ARCHITECTURE.md).
- **Lesson:** Major architecture changes benefit from external analysis. The researcher identified the exact bottleneck (monolithic retrieval units) and the exact fix (file-per-entry + enriched router) with literature backing. The migration itself was straightforward once designed — the hard part was the design, not the code.
- **Emotion:** trust-gained (significant) — brain evolved its own architecture based on research
