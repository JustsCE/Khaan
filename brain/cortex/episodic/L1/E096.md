# og Brain — E096: Cross-Pane Task-Move Bubble Animation — Partial Fix

**Context:** Justs reported on 2026-04-28 that dragging tasks across visuals (Brain to Personal to Professional panes) on the dashboard had no bubble animation; the task simply disappeared from the source and reappeared in the destination after refresh. Grouping tasks within a pane (drop on row) had a working bubble animation.

**Diagnosis (Explore agent):** The drag onUp handler in frontend/src/pages/DashboardPage.tsx had three branches: kind on (group) calls setFlyingOrb. The before-after and end cross-pane branches called moveTask directly with NO setFlyingOrb.

**Fix attempt (this session):** Added a flyOrbForMove(targetEl) helper after remapSource that mirrors the group flow setFlyingOrb call. Wired into both cross-pane branches with targetRow (before-after) or targetPaneEl (end) as anchor. Applied to both slash-opt/brain-viz/ and ~/.claude/viz/ copies; verified byte-identical.

**Outcome:** User reports task still just jumps with no bubble visible. Fix is structurally correct but the orb does not visibly render for cross-pane moves.

**Open hypotheses (next session):**
1. Race: moveTask triggers optimistic update that removes the source task row from DOM, but flyOrbForMove captures rect before moveTask, so that should not be it.
2. Orb layer suppression: FlyingOrbLayer may render only when source and target panes share a positioning container, or when the orb coordinates are within a clip region. Cross-pane moves cross panes that may live in different scroll/clip contexts.
3. Different bubble: The bubble the user is referring to may not be FlyingOrbLayer at all — could be the mergingTaskId-driven CSS effect set on the destination row, or some other animation tied to the group flow but not the move flow.
