# The Ten-Fix Session That Hardened the Brain Pipeline

On a single session, ten distinct fixes landed across the Brain's decision engine, memory cycle, and CLI layer — a burst of stabilization work that touched nearly every subsystem. The fixes fell into three clusters: display resilience (FIX1a added a gist display fallback so missing gists no longer crashed rendering; FIX2 pulled gist text from frontmatter titles when body was absent), state hygiene (FIX1b ensured hypothesis bins clear after consumption, preventing stale decisions from leaking into later turns; FIX8 tightened identity relay output), and model/invocation correctness (FIX6 fixed the `cli_invoke` model parameter pass-through; FIX7 promoted retro, boot, and cycle phases to Opus so consolidation reasoning ran at full capacity; FIX3 enforced strict JSON on identity relay output). FIX5 addressed retro transcript truncation that had been silently dropping tail-end session events before they could be reflected on.

The pattern this session established was that the Brain's failure modes were no longer architectural — they were edge-case brittleness: a missing field here, a wrong model tier there, a bin that forgot to flush. Each fix was small, but collectively they moved the system from "works on the happy path" to "degrades gracefully and self-corrects." The session also revealed a recurring lesson: any subsystem that shells out to `claude -p` must explicitly pass the model flag, or it silently falls back to a cheaper tier and produces subtly worse output.
