Overview

The GEO Controlling (GLC) Power BI report shows NGO Controlling P&L lines for material, shipping, and labor costs. For months where LucaNet actuals are not yet loaded, the model uses coefficient targets — percentage-of-revenue quotas that approximate the real costs. Once LucaNet actuals arrive, the coefficients for those months are updated to reflect the real quota.

The coefficient update runs on the same day as the LucaNet load (~22nd–23rd of the month) and does not depend on the overnight ETL run — it can be done immediately after the SP runs.

Reference source: Alisa Lobodenkov's Global Reporting file → period-shift adjusted tab. Always confirm the file is up to date with Alisa before using the benchmarks.

When to Run

Coefficient targets are updated on two occasions:

  1. After the monthly LucaNet load — update the loaded month's coefficient to reflect the actual quota (or set the absolute cost directly for labor).
  2. Proactively for future months — populate dummy targets for months not yet in LucaNet so GLC always shows plausible estimates rather than stale coefficients.

The update can be done same day as the LucaNet load — it does not wait for the overnight ETL run.

Step-by-Step Process

1. Get current GEO Controlling data

  1. Open the GEO Controlling BI report (BI SharePoint — standard report location).
  2. Copy the current state to clipboard.
  3. Open GLC Alerts Controlling Excel (BI SharePoint → Power BI alerts folder).
  4. Go to the GLC export tab → paste → Text to Columns → Delimited by comma.

2. Compare current coefficients vs. Alisa's targets

  1. Open Alisa's Global Reporting file → go to the period-shift adjusted tab.
  2. Confirm Alisa has updated the file for the month you are loading (ask her directly if unsure).
  3. Compare current GLC coefficient values against the benchmark targets for material, shipping, and labor.

3. Update coefficients in the model

  • For months with real LucaNet values: leave absolute costs as-is; set the coefficient to reflect the actual quota from LucaNet. For labor: use absolute amounts as the primary check (see Labor Costs doc).
  • For future months without real data: populate dummy targets (material ~16%, shipping ~20%, labor ~9%).

Labor coefficient warning: The Excel formula for the labor coefficient is not fully optimized. Watch for accidental absolute-vs-percentage confusion — a coefficient of 3,000,000 instead of 3.0 is a real failure mode. Double-check labor rows after every update.

The SP_CM2 stored procedure handles the mechanics of applying these coefficients at the shop/day level — see the LucaNet Load Process doc for how it distributes costs internally.

Validation

After updating coefficients and waiting for the overnight ETL run (or asking Davis to rerun manually):

GCS alerts Slack channel

Alerts fire if daily values drop below historical minimums defined in the Anchors tab of GLC Alerts Controlling Excel. After a coefficient update, verify no false alerts are triggering. If thresholds are outdated (anchors not updated recently), update the Anchors tab from SQL or GEO Controlling and re-check.

Data Quality tab in GEO Controlling BI report

Shows actual vs. target cost quotas per position and per affiliate. Use this to confirm the coefficient update landed correctly:

  • Material / shipping: compare % quota against Alisa's benchmarks.
  • Labor: compare absolute amounts for months with LucaNet data.
  • Small deviations (~0.5–2% per affiliate) are normal; flag anything larger.
