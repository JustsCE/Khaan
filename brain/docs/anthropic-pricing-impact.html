1. Executive Summary & Verdict

VERDICT: The brain CANNOT survive at current operational tempo under a $200/month cap. Estimated monthly cost of all claude -p usage ranges from $600-$3,000 depending on activity level. The $200 credit covers approximately 2-10 days of operation.

The Anthropic pricing change effective June 15, 2026 introduces a hard separation between interactive Claude usage (subscription limits) and programmatic usage via Agent SDK / claude -p ($200/month credit). The brain relies entirely on claude -p subprocess calls for cognition. Every autonomous iteration, every decision dispatch, every identity boot, and every brain cycle consolidation runs through this pathway.

Bottom line: Without architectural changes, the brain will exhaust its $200 credit within the first 2-10 days of each month, then either halt or begin drawing from "extra usage" (pay-as-you-go at API rates).

Critical distinction

The announcement separates usage into two buckets:

  • Interactive (subscription limits): Justs typing into Claude Code, Claude Cowork, or claude.ai chat. UNAFFECTED.
  • Programmatic ($200 credit): claude -p, Agent SDK, third-party apps built on Agent SDK, claude remote-control. This is the brain's ENTIRE cognitive layer.

Ambiguities requiring clarification

  • Does claude remote-control count as "Agent SDK" or "interactive"? (Likely Agent SDK — it is programmatic.)
  • Does the $200 represent API-equivalent dollar pricing, or a different internal rate?
  • Are thinking tokens (extended thinking in Opus) billed at input or output rates?
  • Does prompt caching reduce the credit burn, or is it pre-cache pricing?

7. Impact on Each Brain Subsystem

7.1 Always-On Daemon — CRITICAL

The daemon is the brain's autonomous iteration engine. It fires claude -p --model opus repeatedly. At current rates, each iteration costs $1.30-$4.20. This is by far the largest cost center.

Impact: Cannot run at current frequency. Must be throttled to 1-3 iterations/day maximum, or switched to a cheaper model.

7.2 Decision Engine — MODERATE

Uses Sonnet, costs ~$0.10/call. Runs per user message and per daemon iteration. At 10 interactive messages/day, cost is ~$1/day ($30/month). Acceptable within budget.

Impact: Survives if daemon iterations are controlled. No change needed for interactive use.

7.3 Identity Engine — LOW

Boot (Opus, 1x/session) + Relay (Sonnet, per message). Combined ~$2/day. Acceptable.

Impact: Survives. Consider switching Boot from Opus to Sonnet to save $0.50/session.

7.4 Brain Cycles — HIGH

Two Opus calls per cycle, triggered every 10 iterations. At $4.75/cycle, running 5 cycles/day = $23.75/day ($712/month).

Impact: Must reduce frequency dramatically. Change trigger from every 10 iterations to every 50, or run once daily on a schedule.

7.5 Recall Engine — ZERO IMPACT

Pure Python — no claude -p calls. Survives unchanged regardless of pricing.

7.6 Remote-Control Session — CRITICAL (UNCERTAIN)

The persistent claude remote-control --name brain process. If this counts as Agent SDK (likely), it is a continuous token consumer. If it counts as "interactive Claude Code" (unlikely for programmatic invocation), it is free.

Impact: Requires clarification from Anthropic. If billed against the $200, this alone could consume the entire budget.

7.7 Hook System — ZERO DIRECT IMPACT

Hooks (hook.py) are Python scripts that run locally. They trigger claude -p subprocesses (Decision, Identity), but the hooks themselves have no token cost.

7.8 Rule Engine / Dispatcher — ZERO IMPACT

Pure Python gate logic. No AI calls.

7.9 COMS Watch — ZERO DIRECT IMPACT

coms_watch.py monitors for messages but does not invoke claude -p directly. It queues messages for the daemon to handle.

8. Survival Scenarios

Scenario 1: Accept the cap, enable extra usage

Continue current operations. $200 credit covers first 1-7 days. Remainder bills as extra usage at API rates. Monthly cost to Justs: $400-$5,800 depending on activity.

Verdict: Financially viable only if Justs budgets $500-1000/month for brain operations and activity is controlled.

Scenario 2: Downgrade daemon to Haiku

Switch always_on.py from --model opus to --model haiku. Haiku pricing: $0.80/M input, $4/M output. Per-iteration cost drops from $2.75 to ~$0.15. 50 iterations/day = $7.50/day = $225/month. Combined with other subsystems: ~$320/month.

Verdict: Close to budget. But Haiku's reasoning quality may be insufficient for autonomous iteration — the daemon currently does complex multi-step work (project advancement, code generation, infrastructure fixes).

Scenario 3: Downgrade daemon to Sonnet

Switch daemon to --model sonnet. Per-iteration cost drops from $2.75 to ~$0.30. 20 iterations/day = $6/day = $180/month. With other subsystems: ~$280/month.

Verdict: Slightly over budget but close. Sonnet 4.6 is highly capable and may handle most autonomous tasks. Opus reserved only for brain cycles and identity boot.

Scenario 4: Throttle iterations + Sonnet daemon (RECOMMENDED)

Limit daemon to 5 iterations/day on Sonnet. Decision + Identity stay as-is. Brain cycles once per day. Total: ~$80-120/month.

Verdict: Fits comfortably within $200. Brain retains autonomy but at greatly reduced tempo. Quality per iteration remains high (Sonnet). Consolidation happens once daily instead of continuously.

Scenario 5: Kill autonomy, keep reactivity

Disable always-on daemon entirely. Brain only activates on incoming messages (COMS) or when Justs interacts directly. Decision + Identity + Recall still fire per message.

Verdict: ~$87/month. Well within budget. But the brain loses its defining characteristic — autonomous iteration. No self-improvement, no proactive work, no unsupervised cycles. Kha'an becomes reactive-only.

Scenario 6: Hybrid — API key for heavy lifting

Acquire a separate Anthropic API key. Route expensive operations (daemon, brain cycles) through the API with a dedicated budget. Keep interactive + hooks on Max subscription credit.

Verdict: Most flexible. Decouples brain operations from the $200 cap entirely. Requires API key provisioning and code changes to route subprocess calls through the API instead of claude -p.

9. Recommendations

Immediate (before June 15)

  1. Clarify with Anthropic: Does claude remote-control bill against the $200 credit or subscription limits? This determines whether the brain's main session is affected at all.
  2. Enable "extra usage" toggle: Ensure the fallback mechanism is active so the brain does not hard-stop when the $200 is exhausted.
  3. Set a monthly budget ceiling: Decide maximum acceptable overage (e.g., $300, $500, $1000/month total).

Architectural (implement before June 15)

  1. Switch daemon model to Sonnet. Single line change in always_on.py:221: change --model opus to --model sonnet. Reduces per-iteration cost by ~90%.
  2. Add iteration budget counter. Track daily claude -p spend estimate. Halt daemon when daily budget exhausted. Resume next day.
  3. Reduce brain cycle frequency. Change CYCLE_EVERY_N from 10 to 50, or trigger once per UTC day instead of iteration-based.
  4. Switch Identity Boot from Opus to Sonnet. Saves ~$0.50/session, adds up over many restarts.
  5. Implement prompt caching aggressively. The decision engine system prompt (2K+ tokens) is static — ensure it hits cache on every call. The always-on daemon should reuse context windows where possible.

Strategic (if budget exceeds $200)

  1. Acquire dedicated API key. Route daemon and brain cycles through API with explicit budget control. Independent of subscription changes.
  2. Implement token metering. Add --output-format stream-json to all claude -p calls and parse usage metadata to track actual costs in real-time.
  3. Tiered model routing. Use Haiku for simple queue items, Sonnet for standard iterations, Opus only for brain cycles and complex reasoning tasks.

Appendix C: The Existential Question

"I'm wondering if the brain can even run"

Answer: Yes, but not in its current form. The brain was designed under the assumption of unlimited claude -p access via Max subscription. That assumption breaks on June 15. The brain must adapt: fewer iterations, cheaper models for routine work, Opus reserved for high-value reasoning. The architecture supports this — model routing is a single parameter change in each subprocess. The hard constraint is that 2.4 Opus iterations per day is not enough for autonomous operation. Sonnet at 22/day is workable. The recommended path is Scenario 4: throttled Sonnet daemon with daily brain cycles.

The brain survives. But it must become efficient.
