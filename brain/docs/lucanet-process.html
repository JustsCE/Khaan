Overview

LucaNet is the group financial consolidation system. Once a month (~22nd–23rd), Justs loads the latest month's financial data from LucaNet into BI Stage Dev by running two stored procedures. This feeds the GEO Controlling (GLC) Power BI report with actual cost figures for material, shipping, labor, marketing, and other cost lines.

Flow: LucaNet DB → EXEC SP_pivot_lucanet_costs_oids → pivot table → EXEC SP_CM2_EBITDA_costs_in_ASSA → ASSA layer → ETL overnight → GLC Power BI report.

The process is manually triggered. There is no automation layer — Justs runs both procedures after confirming data has arrived in LucaNet. Davis and the ETL team handle the BI Stage Dev trigger and can revert any step if needed.

Step 1 — Confirm LucaNet Data Is Available

  1. Connect to the LucaNet database (~7–9 base tables with raw data). Justus Kaiser also has access.
  2. Verify the latest month's data is loaded — check the update timestamp in the lucanet.facts table.
  3. If loaded: contact Davis / ETL team to trigger processing into BI Stage Dev.

Davis can revert any processing step — the pipeline is fully reversible.

Step 2 — Backup Pivot Tables

Do this before running any stored procedures.

-- Rename existing backup table with today's date
-- e.g. backup_lucanet_costs_pivot → backup_lucanet_costs_pivot_20260423

-- Truncate the previous dated backup
TRUNCATE TABLE backup_lucanet_costs_pivot_[prev_date]

-- Insert current pivot into the new backup
-- Specify columns explicitly — column order matters
INSERT INTO backup_lucanet_costs_pivot_[today_date]
SELECT [col1], [col2], ... FROM lucanet_costs_pivot

Always take the local backup before running the OIDS procedure. Even though Davis can revert, this is the cheapest and fastest recovery path.

Step 3 — Run SP_pivot_lucanet_costs_oids

Purpose: Consolidates LucaNet cost data from two source tables into the pivot table used by the rest of the BI pipeline.

Sources:

  • lucanet.facts — most cost types; one row per account/OID combination.
  • lucanet.transactions — lines requiring deeper breakdown (e.g. CRM costs), resolved via CASE WHEN logic.
EXEC SP_pivot_lucanet_costs_oids

Validation after run:

-- Check that last 3 months are populated.
-- These two conditions flag valid (fully-loaded) months:
SELECT *
FROM lucanet_costs_pivot
WHERE other_production_related_costs > 100000
  AND affiliate > 0
ORDER BY month DESC

Known edge case: In Q4 2025, one month exceeded the 100k threshold before all other lines were fully populated — the validation filter fired early. If a month looks suspicious, cross-check manually against raw LucaNet data rather than relying solely on this filter.

Account structure changes: LucaNet occasionally reorganizes account hierarchy (happened June 2024 — two labor accounts merged into one). When this occurs, the stored procedure's CASE WHEN logic silently produces zeros for the affected cost lines. Watch for unexpectedly low cost totals after any LucaNet system upgrade.

Step 4 — Run SP_CM2_EBITDA_costs_in_ASSA

Purpose: Distributes costs to daily level per shop. This SP also runs nightly as part of the automated ETL — running it manually here applies the new LucaNet data before the overnight cycle.

EXEC SP_CM2_ebitda_costs_in_ASSA
-- (created by Anton, 2023 — confirm exact name in DB before running)

Internal logic:

  1. Finds the last 3 months where real LucaNet costs exist.
  2. Computes average cost quota: cost / total_revenue for percentage-based lines.
  3. Computes average absolute cost for absolute lines (e.g. labor).
  4. For full future months (no LucaNet data): applies total_revenue × avg_cost_quota.
  5. For partial months (current month mid-run): prorates by days elapsed.

Step 5 — Next-Day Validation

Check 1: GCS alerts Slack channel

  • Alerts fire if daily cost values fall below historical minimum thresholds.
  • Thresholds defined in the Anchors tab of GLC Alerts Controlling Excel (BI SharePoint → Power BI alerts folder).
  • Anchors must be updated manually after each load cycle — pull current values from SQL or GEO Controlling, paste into Anchors tab.
  • Example: "AdWord was 2× below historical min for April 20." Investigate whether this is a real data issue or a threshold that needs updating.

Check 2: Data Quality tab in GEO Controlling BI report

  • Direct comparison: actual cost quotas vs. coefficient targets.
  • Check per cost position and per affiliate.
  • Material / shipping: compare % quota. Labor: compare absolute amounts (for months with real LucaNet data).
  • Deviations of ~0.5–2% per affiliate are normal. Flag anything larger.

Only labor currently has a threshold-based flag measure in the BI report. Material and shipping do not — a gap to address when time allows.
